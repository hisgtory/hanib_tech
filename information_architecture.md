# Information Architecture (완성)

<hl>정보를 어떻게 정리할까요? 바로 IA에요!</hl>

마트에 가면 라면이 어디 있는지 알죠?
앱도 마찬가지에요.

사용자가 원하는 걸 쉽게 찾을 수 있도록
정보를 체계적으로 정리하는 것.

그게 바로 Information Architecture에요!

## IA는 보이지 않는 디자인

화려한 UI는 아니지만
사용자 경험의 뿌리가 되는 중요한 작업이에요.

IA가 잘못되면?

- "이 기능이 어디 있지?"
- "설정에서 찾아야 하나? 프로필에서 찾아야 하나?"
- "아니 대체 어디에 있는 거야!"

사용자가 헤매기 시작해요.

## IA의 기본 구성 요소

1. **카테고리(Category)**
   비슷한 것들을 묶어서 그룹화
   예: 쇼핑앱에서 "패션", "전자제품", "식품"

2. **레이블(Label)**
   사용자가 이해하기 쉽게 이름 붙이기
   예: "마이페이지" vs "내 정보"

3. **내비게이션(Navigation)**
   사용자가 원하는 곳으로 갈 수 있는 길
   예: 탭 바, 사이드 메뉴, 브레드크럼

4. **검색(Search)**
   찾고 싶은 걸 빠르게 찾을 수 있는 기능

## IA 만들기 실전 팁

**카드 소팅(Card Sorting)**을 해보세요!

1. 모든 기능/콘텐츠를 카드에 적어요
2. 사용자에게 비슷한 것끼리 묶어달라고 해요
3. 각 그룹에 이름을 붙여달라고 해요

예상과 다른 결과가 나올 거예요!

"아, 사용자들은 이걸 여기서 찾는구나!"

## IA의 대표적인 패턴들

**1. 계층 구조(Hierarchical)**
트리 구조로 위에서 아래로
예: 대부분의 웹사이트

**2. 순차적 구조(Sequential)**
단계별로 진행
예: 회원가입, 온보딩

**3. 네트워크 구조(Network)**
여러 경로로 접근 가능
예: 위키피디아

**4. 태그 기반(Tag-based)**
여러 카테고리에 속할 수 있음
예: 블로그, 유튜브

## 좋은 IA의 특징

✅ 3번의 클릭 안에 원하는 걸 찾을 수 있어요
✅ 메뉴 이름만 보고도 무엇이 있을지 예상 가능해요
✅ 비슷한 기능들이 함께 모여 있어요
✅ 내가 어디에 있는지 항상 알 수 있어요
✅ 검색했을 때 원하는 결과가 나와요

❌ 나쁜 IA의 특징

"기타" 카테고리가 너무 커요
같은 기능이 여러 곳에 흩어져 있어요
메뉴 이름이 모호해요 ("설정" vs "환경설정" vs "옵션")
너무 깊은 계층 구조 (5단계 이상)

## Episode 1-10. IA가 존재하지 않는 앱이 망하는 이유

### 🎯 IA란 무엇인가?

Information Architecture(정보 구조)는 사용자가 원하는 정보나 기능을 쉽게 찾을 수 있도록 체계적으로 조직하는 설계 체계입니다.

**쉽게 말하면 앱의 '목차'나 '지도' 같은 거예요!**

백화점에 층별 안내도가 있듯이, 앱도 사용자가 원하는 기능을 빠르게 찾을 수 있도록 정리하는 것이 IA입니다.

### 🚨 IA가 없으면 생기는 문제

**사용자가 길을 잃어요:**
- '어디서 뭘 해야 하지?' - 사용자가 길을 잃고 방황합니다
- '이 기능이 여기 있었어?' - 중요한 기능을 찾지 못합니다
- '너무 복잡해!' - 사용자가 앱을 삭제합니다 😱

### 🎨 IA 만드는 방법

**1. 카드 소팅:** 기능들을 카테고리별로 분류하기
**2. 계층 구조 만들기:** 메인 > 서브 > 상세
**3. 네비게이션 설계:** 탭바, 햄버거 메뉴, 검색
**4. 사용자 테스트:** '찾기 쉽나요?' 확인하기

### 📱 좋은 IA 예시: 인스타그램

인스타그램은 하단에 5개의 아이콘이 있고, 각각의 역할이 명확합니다:
- 홈: 피드 보기
- 검색: 콘텐츠 및 사용자 검색
- 릴스: 짧은 비디오 콘텐츠
- 쇼핑: 쇼핑 기능
- 프로필: 내 계정 및 설정

**진짜 한눈에 보이고 찾기 쉬워요!**

### 💡 IA 설계 꿀팁

**프로가 알려주는 핵심 원칙:**
- **3 클릭 룰:** 모든 기능은 3번 이내 터치로 도달
- **일관성:** 비슷한 기능은 같은 위치에
- **우선순위:** 자주 쓰는 기능은 눈에 잘 띠게
- **명확한 라벨:** '더보기'보다는 '설정'

**이게 IA 설계의 핵심이에요!**

### 🔥 실패 사례

어떤 앱은 '더보기' 안에 '기타' 안에 '추가 옵션' 안에 중요한 기능이...

**"그런 앱은 바로 삭제...! 🗑️"**

이렇게 되면 사용자가 죽다 깨어도 못 찾아요!

## Episode 1-11. 화면 설계가 바뀌는 이유, 내가 IA를 그리지 않아서다

### 🏢 IA의 기본 4가지 요소

**1. 조직 체계(Organization Systems)**
   콘텐츠를 어떻게 분류하고 구조화할 것인가

**2. 네비게이션 시스템(Navigation Systems)**
   사용자가 정보 사이를 어떻게 이동할 것인가

**3. 레이블링 시스템(Labeling Systems)**
   정보를 어떤 이름으로 표현할 것인가

**4. 검색 시스템(Search Systems)**
   사용자가 정보를 어떻게 검색할 수 있게 할 것인가

### 😱 IA 없이 시작할 때 생기는 문제들

**첫번째: 기능이 추가될 때마다 전체 구조가 흔들려요**
- 처음엔 간단한 프로필 화면이었는데
- 나중에 '팔로우 기능'을 추가하려니 어디에 넣을지 막막하죠
- 결국 임시방편으로 어딘가에 끼워 넣고, 나중에 또 수정하는 악순환

**두번째: 사용자 플로우가 꿈어요**
- A화면에서 B화면으로 갔다가 다시 돌아올 때 어디로 가야 할지 명확하지 않아요
- "장바구니에서 상품 상세를 봤다가 뒤로 가기 하면 장바구니로 가나요, 홈으로 가나요?"

**세번째: 같은 기능이 여러 곳에 중복되거나 만대로 필요한 곳에 없어요**
- '주문 취소' 기능이 주문 내역에도 있고, 고객센터에도 있고, 마이페이지에도 있다면?
- 개발 리소스도 낭비되고 사용자도 헷갈려요

### 🛠️ IA 설계 실전 4단계

**1단계: 콘텐츠 감사 (Content Audit)**
먼저 서비스에 포함될 모든 콘텐츠와 기능을 목록화합니다.

**2단계: 카드 소팅 (Card Sorting)**
실제 사용자 5-10명에게 각 기능이나 콘텐츠를 카드로 만들어 주고, 자연스럽게 그룹핑하도록 합니다.

**3단계: 사이트맵 작성**
카드 소팅 결과를 바탕으로 전체 서비스의 계층 구조를 시각화합니다.

**4단계: 네비게이션 설계**
사이트맵을 바탕으로 사용자가 서비스를 탐색하는 방법을 설계합니다.

### ⚠️ IA 설계 시 자주 하는 실수들

**1. 내부 용어로 레이블링하기**
- '품목', 'SKU', '에스케이유' 같은 단어는 사용자가 이해하기 어려워요
- '상품', '제품' 같은 일반적인 용어를 사용해야 해요

**2. 모든 것을 홈에 넣으려고 하기**
- '이벤트', '신상품', '추천상품', '인기상품', '할인상품'...
- 이 모두를 홈에 놓으면 사용자는 무엇에 집중해야 할지 몰라요

**3. 사용자 테스트 없이 진행하기**
- 팀 내부에서만 결정한 IA는 실제 사용자의 니즈를 반영하지 못해요

### 🏆 결론

**IA는 서비스의 도대와 같습니다.**

보이지 않지만 모든 것을 지탱합니다. 아무리 아름다운 디자인이라도 IA가 엉망이면 사용자는 길을 잃고 떠나가버립니다.

<hl>기억하세요. 화면 설계가 자꾸 바뀐다면, 그건 당신이 IA를 그리지 않았기 때문입니다!</hl>

다음에 프로젝트를 시작할 때는 화면 디자인부터 하지 말고, IA부터 그려보세요. 처음엔 시간이 더 걸리는 것 같아도, 결국 전체 프로젝트 시간을 단축시키고 더 나은 사용자 경험을 만들 수 있을 겁니다.
