개발 용어 및 컨셉 정리
이 문서는 Miro에서 제공된 용어들을 주제별로 정리한 것입니다. 각 섹션은 개발 프로세스, 협업, 기술, 디자인, 비즈니스 등 다양한 측면을 다루며, 간단한 설명과 함께 용어들을 나열합니다.

1. 개발 프로세스 및 방법론
개발 프로젝트를 체계적으로 진행하고 관리하기 위한 용어들입니다.

애자일: 유연하고 반복적인 개발 방법론.
스프린트: 애자일에서 일정 기간 동안 수행하는 작업 단위.
스크럼: 팀 기반 협업을 강조하는 애자일 방법론.
킥오프: 프로젝트나 스프린트의 시작을 알리는 회의.
회고: 작업 종료 후 반성 및 개선을 위한 회의.
회고 템플릿: 회고를 구조화하기 위한 양식.
온보딩: 신입 팀원이 적응하는 과정.
PRD (Product Requirements Document): 제품 요구사항 문서.
지라: 프로젝트 관리 및 이슈 추적 도구.
백로그: 처리해야 할 작업 목록.
OKR (Objectives and Key Results): 목표와 결과를 설정하는 프레임워크.
스케일 업: 시스템의 처리 능력을 키우는 것.
스테이징 서버: 배포 전 테스트용 서버.
QA (Quality Assurance): 품질 보증 과정.
QA 시트: QA를 위한 문서.
A/B 테스트: 두 버전을 비교하는 테스트.
PoC (Proof of Concept): 개념 증명 프로토타입.
Feasibility Check: 기술적 실현 가능성 확인.
기술 부채: 단기 해결책으로 생기는 장기 문제.
리팩터링: 코드 구조 개선 작업.
레거시 코드: 유지보수가 어려운 오래된 코드.
빌드: 코드를 실행 가능한 앱으로 변환.
배포: 앱을 사용자에게 제공하는 과정.
버그/디버깅: 소프트웨어 오류와 수정 과정.
앱스토어 심사와 리젝: 앱 출시를 위한 검토 과정.
2. 협업 및 커뮤니케이션
팀원 간 소통과 협업을 위한 용어들입니다.

협업: 여러 사람이 함께 작업.
컨센서스: 팀 내 합의.
얼라인: 목표나 이해를 일치시킴.
팔로업: 진행 상황 점검.
커피챗: 비공식 대화로 정보 공유.
원온원: 일대일 회의.
라포: 신뢰와 친밀감 형성.
슬랙 이모지: 슬랙에서 사용하는 이모지.
캘린더: 일정 관리 도구.
인비: 초대(invite).
페어 프로그래밍: 두 개발자가 함께 코딩.
코드 리뷰: 다른 개발자의 코드 검토.
이슈: 해결해야 할 문제.
어싸인: 작업 할당.
Conflict: 코드 충돌.
Commit: 코드 변경 저장.
Push: 변경 사항을 원격에 업로드.
Pull: 원격 변경 사항을 로컬로 가져옴.
Merge: 코드 변경을 하나로 통합.
싱크: 팀 간 정보 동기화.
3. 개발 도구 및 기술
개발에 사용되는 도구와 기술 용어들입니다.

Git/GitHub: 버전 관리 및 코드 호스팅.
IDE (Integrated Development Environment): 통합 개발 환경.
VPN: 가상 사설망.
방화벽: 네트워크 보안 시스템.
내부망: 조직 내 네트워크.
오픈소스: 공개된 소스 코드.
라이브러리: 재사용 가능한 코드 모음.
프레임워크: 개발 구조.
SDK (Software Development Kit): 개발 도구 모음.
API (Application Programming Interface): 프로그램 간 인터페이스.
프로토콜: 통신 규약.
REST API: 웹 서비스 API 설계.
GraphQL: 데이터 쿼리 언어.
gRPC: 고성능 RPC 프레임워크.
HTTP: 웹 데이터 전송 프로토콜.
웹소캣: 실시간 통신 프로토콜.
JSON: 경량 데이터 형식.
XML: 마크업 언어.
SQL: 관계형 데이터베이스 언어.
NoSQL: 비관계형 데이터베이스.
데이터베이스: 데이터 저장 시스템.
RDBMS: 관계형 데이터베이스 관리 시스템.
캐시 서버: 빠른 데이터 제공 서버.
서버 소프트웨어: 서버에서 실행되는 소프트웨어.
하드웨어 서버 (데이터 센터, IDC): 물리적 서버.
AWS: 아마존 클라우드 플랫폼.
DevOps: 개발과 운영 통합.
클라이언트: 서비스 요청 프로그램.
프론트엔드: 사용자 인터페이스.
백엔드: 서버 측 로직.
서버 개발자: 백엔드 개발자.
프론트엔드 서버: 프론트엔드 서빙 서버.
크로스 플랫폼: 다중 플랫폼 지원.
리눅스: 오픈소스 OS.
Flutter: 크로스 플랫폼 프레임워크.
React Native: 크로스 플랫폼 프레임워크.
Swift: iOS 개발 언어.
코틀린: 안드로이드 개발 언어.
자바: 범용 언어.
파이썬: 범용 언어.
Node.js: 자바스크립트 런타임.
HTML: 웹 마크업 언어.
CSS: 웹 스타일링 언어.
Javascript: 웹 개발 언어.
Typescript: 자바스크립트 상위 언어.
UNIX: 운영체제.
M2M 테이블: 다대다 관계 테이블.
상태 코드 (404, ...): HTTP 응답 코드.
4. 디자인 및 사용자 경험
UI/UX 설계와 관련된 용어들입니다.

디자인 시스템: 일관된 UI 가이드라인.
Figma: 협업 디자인 툴.
오토 레이아웃: 자동 UI 배치.
와이어 프레임: UI 설계도.
페르소나: 가상 사용자 프로필.
유저 스토리: 사용자 요구사항 설명.
유저 리서치: 사용자 조사.
유저 인터뷰: 사용자 대화.
사용자 테스트 (Usability Test): 제품 사용성 테스트.
정성적/정량적: 질적/양적 데이터.
반응형/적응형: 화면 크기에 맞춘 디자인.
5. 비즈니스 및 성과
비즈니스 목표와 성과 측정 용어들입니다.

PMF (Product-Market Fit): 시장 적합성.
LTV/CAC: 고객 생애 가치와 획득 비용 비율.
피벗: 사업 방향 전환.
리텐션: 사용자 유지율.
코호트: 특정 사용자 그룹.
유저 세그먼트: 기준별 사용자 분류.
성과: 목표 달성 정도.
러닝쉐어: 학습 공유.
레슨런: 교훈 정리.
6. 기타
분류되지 않은 다양한 용어들입니다.

IoT (Internet of Things): 사물 인터넷.
메타버스 기능: 가상 세계 기능.
유료 프로그래밍 언어: 라이선스 비용 언어.
코드와 프로그램: 코드와 실행 프로그램.
웹 개발과 앱 개발의 차이: 개발 방식 차이.
