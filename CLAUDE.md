## 📚 hanib_tech 콘텐츠 제작 가이드

### 콘텐츠 스타일 특징
1. **일상 언어 사용**: "마트에 가면 라면이 어디 있는지 알죠?"
2. **실무 상황 예시**: 구체적인 대화 예시와 시나리오 제시
3. **단계별 설명**: 복잡한 개념을 단계별로 분해
4. **실전 팁 제공**: 바로 적용 가능한 실무 노하우
5. **감정적 공감**: "자존심이 상하지 않는 방법" 등 감정적 측면 고려

### HSEPC 모델 적용 사례

**Hook (훅)**: 
- "이 기능이 어디 있지?"
- "너무 복잡해!" - 사용자가 앱을 삭제합니다 😱

**Storyline (스토리라인)**:
- 요리 만들고 "맛이 어때?" 물어보는 것 같은 비유
- 마트에서 라면 찾는 경험을 IA에 비유

**Escalation (점진적 복잡성)**:
- 기본 개념 → 실전 적용 → 고급 기법 순서로 전개

**Payoff (핵심 통찰)**:
- "화면 설계가 자꾸 바뀐다면, 그건 당신이 IA를 그리지 않았기 때문입니다!"

**Consistency (일관성)**:
- 친근하고 격려하는 톤
- 실무자 관점에서의 설명
- 구체적 예시와 체크리스트 제공

### 시리즈별 핵심 메시지

#### 1. Information Architecture 시리즈
- **핵심**: 사용자가 길을 잃지 않도록 하는 것이 IA의 목표
- **실용성**: 3클릭 룰, 카드 소팅 방법 제시
- **차별점**: 기술적 설명보다는 사용자 경험 중심 접근

#### 2. PRD 시리즈  
- **핵심**: 1페이지 PRD의 위력
- **실용성**: 구체적 작성 템플릿과 체크리스트
- **차별점**: 완벽한 문서보다는 소통의 도구로서 PRD 강조

#### 3. 기획 리뷰 시리즈
- **핵심**: 리뷰는 비판이 아닌 협업
- **실용성**: 자존심 상하지 않는 마인드셋과 실전 대화법
- **차별점**: 감정적 측면까지 고려한 전인적 접근

#### 4. 디자인 리뷰 시리즈
- **핵심**: 제한이 아닌 가능성 찾기
- **실용성**: 개발팀과의 소통 방법과 대안 제시 기법
- **차별점**: 기술적 제약을 창의적으로 극복하는 방법 제시
