# API 기초 개념 - hanib_tech 콘텐츠 시리즈

> *"아직 API가 일부분만 나온 상태라.. 외부 API 연동이.. ..아아 넹!"*

## 🎯 콘텐츠 개요

이 시리즈는 개발 초보자들이 가장 혼란스러워하는 API 개념을 친근하고 이해하기 쉽게 설명한 한입씨 테크의 교육 콘텐츠입니다. 총 7개의 카드로 구성되어 있으며, API의 기본 개념부터 실제 활용 사례까지를 단계적으로 설명합니다.

## 📚 콘텐츠 구성 (7개 카드)

### 1️⃣ 인트로: 현실적인 상황 제시
**대화 내용**:
- "아직 API가 일부분만 나온 상태라.."
- "외부 API 연동이.."
- "..아아 넹!"

**스토리텔링 방식**:
- 실제 개발 현장에서 들을 수 있는 대화
- 말풍선을 통한 자연스러운 상황 연출
- 독자의 공감대 형성과 호기심 유발

**핵심 메시지**: "그래서 API가 도대체 뭐에요?"

---

### 2️⃣ API 정의와 첫 번째 혼란
**메인 메시지**: "API는 Application Programming Interface의 약자로 두 소프트웨어가 서로 통신할 수 있게하는 메커니즘입니다."

**시각적 스토리텔링**:
```
[개발자 캐릭터] "그래서에요..?"
[고양이 캐릭터] "..어려워요.." (눈물 흘리며)
```

**교육적 포인트**:
- 공식적인 정의 제시
- 하지만 여전히 어렵다는 현실적 반응
- 독자와 같은 눈높이에서의 접근

**톤앤매너**:
- 정의는 정확하게, 반응은 솔직하게
- 어려운 것은 어렵다고 인정하는 정직함

---

### 3️⃣ API의 핵심 가치 제시
**메인 메시지**: "API란 데이터와 소프트웨어간의 상호작용!"

**핵심 내용**:
- **API 서버로 예를 들어볼까요 👀**
- 인스타그램 앱 개발자는 어떻게 **서버와 상호작용**해야 피드를 조회하고, 유저의 팔로우 관계를 업데이트 할 수 있는지 알아야 해요
- 이때, **API 서버를 통해 어떻게 데이터를 저장하고 조회해야 하는지** 알 수 있어요

**시각적 요소**:
- 활짝 웃는 캐릭터로 긍정적 분위기 조성
- 구체적인 예시 (인스타그램) 활용

**차별점**:
- 추상적 개념을 구체적 상황으로 설명
- 실제 앱 개발 시나리오와 연결

---

### 4️⃣ API 문서화의 중요성
**메인 메시지**: "수많은 API를 문서로 관리!"

**핵심 내용**:
- 서버 개발자는 좋아요 API, 팔로우 API 등 **수많은 API들을** 문서화해서 앱 개발자에게 제공해요
- *일반적으로 Swagger 라는 툴을 사용하여 코드에서 추출하여 제공해요👀

**실제 예시**:
```
피드 조회 API
좋아요 API  
팔로우 API
↓
Swagger 도구 아이콘
```

**실무적 통찰**:
- Swagger 도구 소개
- 실제 개발 프로세스와의 연결
- 문서화의 실질적 중요성 강조

---

### 5️⃣ 일상에서 볼 수 있는 API 사례 1 - 소셜 로그인
**메인 메시지**: "우리의 일상에서 볼 수 있는 수많은 API - 1"

**구체적 예시**:
- **소셜 로그인 API**
- 페이스북, 구글, 카카오 로그인을 위해서는 각 플랫폼이 제공하는 API 문서를 참고하여 각각의 방식대로 구현해주어야 해요

**시각적 요소**:
```
네이버 로그인 ↗️
다음은 네이버 로그인 ↗️ 사용자의 로그인 상태 확인

페이스북 포그인을 위한 서는 각 플랫폼이 Facebook 포그인 시 사용자에
게 앱의 권한을 묻는 내용을 구현해야 합니다.

FB.getLoginStatus(function(response) {
    statusChangeCallback(response);
});

JSON 응답 예시:
{
    'status': 'connected',
    'authResponse': {
        'accessToken': '[access-token]',
        'expiresIn':'[unix-timestamp]',
        'signedRequest':'[signed-request]',
        'userID': '[user-id]'
    }
}
```

**교육적 가치**:
- 모든 사람이 경험해본 소셜 로그인 예시
- 실제 코드와 JSON 응답 제시
- 개발자 입장에서의 구현 복잡성 이해

---

### 6️⃣ 일상에서 볼 수 있는 API 사례 2 - 결제 시스템
**메인 메시지**: "우리의 일상에서 볼 수 있는 수많은 API - 2"

**구체적 예시**:
- **결제 API**
- 결제 기능을 구현하기 위해서는 각 PG사들이 제공하는 API 문서를 확인하여 구현해야 해요

**실제 서비스 예시**:
```
토스페이먼츠 (toss payments)
네이버페이 (N Pay)
카카오페이 (pay)
아임포트 (I'mport)
```

**추가 정보**:
- "그 외에도 플랫폼 별 Map API, SMS 발송 API 등 다양한 API가 있어요👀"

**실무적 관점**:
- 실제 개발에서 자주 사용하는 서비스들
- 각각 다른 API 문서를 학습해야 하는 현실
- 개발자의 업무 복잡성에 대한 이해

---

### 7️⃣ 마무리: 친근한 정리
**메인 메시지**: "알아들기 어려웠던 API, 조금 이해되시나요?"

**시각적 요소**:
- 고양이 캐릭터와 개발자 캐릭터가 함께
- 친근하고 따뜻한 분위기

**마무리 톤**:
- 압박하지 않는 부드러운 질문
- 학습자의 이해도에 대한 배려
- 완벽한 이해보다는 '조금'이라도 이해했으면 하는 겸손한 접근

---

## 🎨 콘텐츠 스타일 분석

### 교육적 접근법의 특징

#### 1. 현실 기반 시작
- 개발 현장의 실제 대화로 시작
- "일부분만 나온 상태", "외부 API 연동" 등 실무 용어 사용
- 독자가 직면할 수 있는 상황 제시

#### 2. 단계적 이해 유도
```
혼란스러운 정의 → 핵심 가치 → 실무 도구 → 일상 경험 → 친근한 마무리
```

#### 3. 감정적 공감대 형성
- "어려워요.." (눈물)
- "그래서에요..?" (혼란)
- "조금 이해되시나요?" (겸손한 확인)

### 시각적 스토리텔링

#### 캐릭터 활용
- **개발자 캐릭터**: 궁금해하는 역할
- **고양이 캐릭터**: 학습자 대변, 감정 표현
- **한입 캐릭터**: 설명자, 친근한 가이드

#### 색상 전략
- **노란색**: 밝고 친근한 분위기
- **회색**: 차분한 설명 공간
- **검은색**: 중요한 정보 강조

### 실무 연결성

#### 구체적 서비스 예시
- 인스타그램 (소셜 미디어)
- 페이스북, 구글, 카카오 (소셜 로그인)
- 토스페이먼츠, 네이버페이 (결제)

#### 실제 도구 소개
- Swagger (API 문서화)
- Facebook SDK (코드 예시)
- JSON 응답 구조
