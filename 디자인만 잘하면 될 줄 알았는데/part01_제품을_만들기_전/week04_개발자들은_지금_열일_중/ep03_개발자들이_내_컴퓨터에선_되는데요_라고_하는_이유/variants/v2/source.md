# 개발자들이 "내 컴퓨터에선 되는데요"라고 하는 이유 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

"내 컴퓨터에선 되는데요"는 환경 차이 때문에 발생한다. 개발 환경, 테스트 환경, 운영 환경이 다르면 같은 코드도 다르게 동작할 수 있다.

---

## 환경이 다르면 생기는 문제들

| 차이 요소 | 예시 | 발생하는 문제 |
|----------|------|--------------|
| OS | Windows vs Mac vs Linux | 경로 구분자, 줄바꿈 문자 |
| 브라우저 | Chrome vs Safari vs Firefox | CSS 렌더링, JS API 지원 |
| 버전 | Node 16 vs Node 18 | 함수 동작 차이 |
| 데이터 | 테스트 DB vs 실제 DB | 데이터 양, 특수 케이스 |
| 네트워크 | 로컬 vs 원격 | 속도, 타임아웃 |

## 개발팀에 요청 시 체크리스트

### Before: 좋지 않은 요청
```
"이거 안 돼요"
"버튼 눌러도 반응이 없어요"
"로그인이 안 됩니다"
```

### After: 좋은 요청
```
[환경]
- iPhone 14 Pro, iOS 17.1
- Safari 브라우저
- 앱 버전 2.3.1

[재현 단계]
1. 로그인 화면 진입
2. 이메일/비밀번호 입력
3. 로그인 버튼 탭
4. 5초 후 "네트워크 오류" 팝업

[기대 결과]
- 메인 화면으로 이동

[실제 결과]
- 에러 팝업 후 화면 그대로

[스크린샷]
- (첨부)
```

## QA/테스터 협업 체크리스트

### 테스트 전 확인
- [ ] 테스트 환경 명세 공유받았는가?
- [ ] 테스트 계정 준비되었는가?
- [ ] 테스트 데이터 준비되었는가?
- [ ] 예상 동작 문서화되었는가?

### 테스트 중 기록
- [ ] 발견된 이슈 즉시 기록
- [ ] 환경 정보 함께 기록
- [ ] 스크린샷/영상 첨부
- [ ] 재현 가능 여부 확인

### 테스트 후 공유
- [ ] 이슈 우선순위 정리
- [ ] 환경별 결과 정리
- [ ] 미해결 이슈 목록화

## 실무 팁

:::callout[tip]
**버그 리포트 템플릿을 만들어 두세요**

Notion, Jira, Linear 등에 버그 리포트 템플릿을 미리 만들어두면:
- 필수 정보 누락 방지
- 개발팀 확인 시간 단축
- 히스토리 추적 용이
:::

:::callout[warning]
**"아까는 됐는데"는 정보가 아닙니다**

환경이 바뀌면 결과도 바뀝니다.
- 네트워크 상태 변화
- 캐시/쿠키 상태 변화
- 서버 상태 변화
- 시간에 따른 데이터 변화
:::

## 빠른 체크

| 상황 | 확인할 것 |
|------|----------|
| "안 돼요" | 정확히 뭐가 어떻게 안 되는지 |
| "느려요" | 얼마나 느린지 (초 단위) |
| "이상해요" | 기대 결과 vs 실제 결과 |
| "에러나요" | 에러 메시지 전문 |
