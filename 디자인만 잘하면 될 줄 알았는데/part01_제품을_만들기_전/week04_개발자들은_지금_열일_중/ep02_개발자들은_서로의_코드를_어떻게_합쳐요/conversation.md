**N**: 개발자들이 같은 파일 동시에 고치면 어떻게 돼?

**한입**: Git이라는 도구가 관리해줘. Google Docs처럼 여러 명이 작업할 수 있어.

**N**: 그럼 자동으로 합쳐져?

**한입**: 다른 부분 고치면 자동으로 합쳐져. 근데 같은 줄 고치면 충돌이 나.

**N**: 충돌?

**한입**: 응. "A는 빨간색, B는 파란색으로 바꿨는데 뭘로 할래?" 이런 거. 개발자가 직접 선택해야 해.

**N**: 그래서 개발자들이 "브랜치"라고 하는 거구나?

**한입**: 맞아. 나뭇가지처럼 각자 자기 가지에서 작업하고, 완성되면 메인에 합쳐.

**N**: PR은 뭐야? 슬랙에서 자주 보던데.

**한입**: Pull Request. "내 코드 합쳐주세요" 하는 요청이야. 합치기 전에 다른 사람들이 검토해.

**N**: 아, 그래서 "PR 올렸어요" 다음에 바로 적용 안 되는 거구나.

**한입**: 응. 리뷰받고, 테스트 통과하고, 승인받아야 합쳐져.

**N**: GitHub은 뭐야?

**한입**: Git으로 관리하는 코드를 저장하는 곳. 카카오톡이 앱이면, 카카오 서버 같은 거야.

**N**: 개발자한테 "작업 언제 끝나요?" 물어보면 왜 확답을 안 줘?

**한입**: 코드 짜는 시간 + 리뷰 시간 + 충돌 해결 시간 + 테스트 시간... 변수가 많거든.
