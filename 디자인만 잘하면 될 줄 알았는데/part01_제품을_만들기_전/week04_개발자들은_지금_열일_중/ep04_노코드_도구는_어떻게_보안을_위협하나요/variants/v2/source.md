# 노코드 도구는 어떻게 보안을 위협하나요 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

노코드 도구의 무분별한 사용은 Shadow IT를 만들고, 이는 데이터 유출과 보안 취약점으로 이어질 수 있다. 사용 전 승인과 관리가 필요하다.

---

## Shadow IT란?

IT 부서의 승인 없이 직원들이 개별적으로 사용하는 IT 서비스/도구

### 흔한 Shadow IT 사례
- 개인 구글 드라이브에 회사 문서 저장
- 승인 안 된 협업 도구 사용
- 노코드 도구로 만든 비공식 시스템
- 개인 Notion에 고객 데이터 정리

## 노코드 도구 보안 위험 체크리스트

### 데이터 저장 위치
- [ ] 데이터가 어디에 저장되는지 확인했는가?
- [ ] 서버 위치(국가)를 확인했는가?
- [ ] 데이터 암호화 방식을 확인했는가?
- [ ] 백업 정책을 확인했는가?

### 접근 권한
- [ ] 누가 데이터에 접근할 수 있는가?
- [ ] 권한 관리 기능이 있는가?
- [ ] 퇴사자 계정 처리 방법이 있는가?
- [ ] 활동 로그를 확인할 수 있는가?

### 외부 연동
- [ ] 어떤 외부 서비스와 연동되는가?
- [ ] API 키가 노출될 위험은 없는가?
- [ ] 연동된 서비스의 보안 수준은 어떠한가?

---

## 노코드 도구 도입 전 체크리스트

### 승인 절차
- [ ] IT/보안팀에 사용 승인 요청
- [ ] 어떤 데이터를 다룰지 명시
- [ ] 사용 목적과 기간 명시
- [ ] 대안 검토 여부 확인

### 보안 검토
- [ ] SOC 2 / ISO 27001 인증 여부
- [ ] GDPR / 개인정보보호법 준수 여부
- [ ] 데이터 처리 동의서(DPA) 체결 가능 여부
- [ ] 보안 사고 이력 확인

### 계약 검토
- [ ] 서비스 약관 검토
- [ ] 데이터 소유권 조항 확인
- [ ] 서비스 종료 시 데이터 처리 방법
- [ ] SLA(서비스 수준 협약) 확인

## 데이터 민감도별 사용 가이드

### 공개 가능 데이터 (Low Risk)
- 블로그 콘텐츠
- 마케팅 자료
- 공개 제품 정보
→ **대부분의 노코드 도구 사용 가능**

### 내부용 데이터 (Medium Risk)
- 내부 프로세스 문서
- 팀 회의록
- 프로젝트 일정
→ **승인된 도구만 사용, 접근 권한 관리 필수**

### 민감한 데이터 (High Risk)
- 고객 개인정보
- 매출/재무 데이터
- 전략 문서
→ **노코드 도구 사용 금지, 공식 시스템만 사용**

### 기밀 데이터 (Critical)
- 급여 정보
- 의료 정보
- 금융 거래 정보
→ **외부 서비스 사용 금지**

## 안전한 노코드 사용 가이드

### DO (해야 할 것)
- [ ] IT팀 승인 후 사용
- [ ] 업무용 계정으로 가입
- [ ] 2단계 인증 활성화
- [ ] 최소 권한 원칙 적용
- [ ] 정기적으로 접근 권한 검토
- [ ] 사용 종료 시 데이터 삭제

### DON'T (하지 말아야 할 것)
- [ ] 개인 계정으로 회사 데이터 저장
- [ ] 고객 개인정보를 노코드 도구에 저장
- [ ] 비밀번호/API 키를 노코드 도구에 저장
- [ ] 퇴사자 계정 방치
- [ ] 무료 플랜으로 민감한 데이터 처리

---

## 사고 발생 시 대응 체크리스트

### 즉시 조치
- [ ] 접근 권한 즉시 차단
- [ ] 관련 비밀번호 변경
- [ ] IT/보안팀에 즉시 보고
- [ ] 영향 범위 파악

### 사후 조치
- [ ] 사고 경위 문서화
- [ ] 재발 방지 대책 수립
- [ ] 필요 시 관계 기관 신고
- [ ] 영향받은 사용자 통보

---

## 실무 팁

:::callout[tip]
**"편하니까"는 이유가 아닙니다**

노코드 도구가 편하다고 무작정 사용하면:
- 데이터가 여러 곳에 분산
- 퇴사자 계정으로 정보 유출
- 서비스 종료 시 데이터 손실
- 보안 사고 시 책임 소재 불분명
:::

:::callout[warning]
**무료 플랜의 함정**

무료 플랜은 보통:
- 데이터 암호화 미지원
- 접근 로그 미제공
- 백업 기능 제한
- SLA 보장 없음
:::
