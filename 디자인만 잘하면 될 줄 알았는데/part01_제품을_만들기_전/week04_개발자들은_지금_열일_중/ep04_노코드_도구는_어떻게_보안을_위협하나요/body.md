# 노코드 도구는 어떻게 보안을 위협하나요

## 코드 없이 만드는 시대

"코드 한 줄 없이 앱을 만들 수 있어요!" Notion, Zapier, Airtable, Webflow... 이런 노코드(No-Code) 도구들이 인기예요. 그런데 왜 개발팀에서 걱정할까요?

## 노코드 도구란?

프로그래밍 없이 드래그 앤 드롭으로 서비스를 만들 수 있는 도구예요.

**대표적인 노코드 도구:**
- **Notion**: 문서/데이터베이스 관리
- **Zapier**: 앱 간 자동화 연결
- **Airtable**: 스프레드시트 + 데이터베이스
- **Webflow**: 웹사이트 제작
- **Bubble**: 웹앱 제작

## 왜 보안 문제가 될까?

### 1. "섀도우 IT" 발생

:::callout[warning]
IT 부서 모르게 만들어진 시스템을 "섀도우 IT"라고 해요.
:::

```
마케팅팀: "고객 데이터 Airtable로 관리하면 편하겠다!"
         (IT팀 모르게 셋업)
    ↓
6개월 후: 그 Airtable에 고객 개인정보 10만 건이...
```

### 2. 권한 관리 구멍

**일반적인 개발:**
- 누가 어떤 데이터에 접근할 수 있는지 설계
- 권한 체계를 코드로 구현
- 보안팀 검토

**노코드 도구:**
- 링크만 있으면 누구나 접근?
- "공유" 버튼 한 번으로 전체 공개?
- 퇴사자도 계속 접근 가능?

### 3. 데이터 유출 위험

| 상황 | 위험 |
|------|------|
| Notion 페이지 공개 설정 | 검색엔진에 노출 |
| Zapier로 외부 연동 | 데이터가 제3자 서버로 |
| Airtable 공유 링크 | URL 알면 누구나 접근 |

### 4. 규정 준수 문제

회사에는 지켜야 할 보안 규정이 있어요.
- 개인정보보호법
- 내부 보안 정책
- 고객사 보안 요구사항

노코드 도구가 이 규정을 다 지키는지 확인하기 어려워요.

## 실제로 일어난 일들

### 사례 1: 공개된 Notion 페이지

```
직원이 편의상 Notion 페이지를 "웹에 공개"로 설정
    ↓
구글에서 회사 내부 문서가 검색됨
    ↓
고객사 정보, 매출 데이터 노출
```

### 사례 2: Zapier 자동화 실수

```
마케팅팀이 Zapier로 고객 이메일 자동 수집 설정
    ↓
연동된 외부 서비스에 고객 이메일 전송
    ↓
개인정보 무단 제3자 제공
```

### 사례 3: 퇴사자 접근 권한

```
업무용 Airtable을 개인 계정으로 생성
    ↓
담당자 퇴사
    ↓
회사는 그 데이터에 접근 불가
퇴사자는 계속 접근 가능
```

## 안전하게 사용하려면

### 회사 차원에서

1. **승인된 도구 목록 만들기**
   - 사용 가능한 노코드 도구 정의
   - 민감한 데이터엔 사용 금지

2. **사용 가이드라인 제공**
   - 공유 설정 기본값 제한
   - 개인정보 입력 금지 범위 명시

3. **정기적인 점검**
   - 어떤 노코드 도구가 사용되는지 파악
   - 권한 설정 점검

### 개인 차원에서

:::callout[important]
노코드 도구 사용 전 체크리스트:
:::

- [ ] 이 데이터를 여기에 넣어도 될까?
- [ ] 공유 범위가 적절한가?
- [ ] IT/보안팀에 알렸나?
- [ ] 퇴사 시 인수인계 가능한가?

## 디자이너가 알아야 할 점

### 프로토타입 vs 실제 서비스

**프로토타입/테스트용으로 OK:**
- 가짜 데이터로 만든 목업
- 팀 내부 공유용 문서
- 일시적인 협업 도구

**실제 서비스용으로 주의:**
- 실제 고객 데이터
- 결제/개인정보 관련
- 장기적으로 사용할 시스템

### 팀 협업 시

"노코드로 빨리 만들자!"는 좋은 아이디어일 수 있어요.
하지만 먼저 이것만 확인해주세요:

1. **개인정보가 들어가나요?**
2. **IT팀이 알고 있나요?**
3. **나중에 어떻게 관리하나요?**

## 노코드 vs 코드 개발

| 항목 | 노코드 | 코드 개발 |
|------|--------|----------|
| 속도 | 빠름 | 느림 |
| 보안 통제 | 어려움 | 세밀한 통제 가능 |
| 유지보수 | 도구에 종속 | 자체 관리 |
| 규모 확장 | 한계 있음 | 유연함 |
| 적합한 용도 | 프로토타입, 소규모 | 본격 서비스 |

## 정리

:::callout[important]
노코드 도구는 편리하지만, 보안 관리의 사각지대가 될 수 있어요.
민감한 데이터는 IT팀과 상의하고, 공유 설정을 항상 확인하세요.
:::

**핵심:**
- 노코드 도구는 IT팀 모르게 사용되면 보안 위험
- 공유 설정, 권한 관리를 주의해야 함
- 개인정보는 노코드 도구에 넣기 전 확인 필요
- 프로토타입 용도와 실서비스 용도를 구분해야 함
