**N**: 서버 개발자한테 물어봤더니 DevOps한테 물어보래. 뭐가 다른 거야?

**한입**: 둘 다 서버 관련이긴 한데, 하는 일이 달라.

**N**: 어떻게?

**한입**: 레스토랑으로 비유하면, 서버 개발자는 요리사야. 음식을 만드는 사람.

**N**: 그럼 DevOps는?

**한입**: 주방 시설 관리자. 가스레인지 잘 되는지, 냉장고 온도 맞는지 확인하는 사람.

**N**: 아... 요리사가 가스레인지 고치진 않지.

**한입**: 그치. 서버 개발자는 기능을 만들고, DevOps는 그 기능이 잘 돌아가게 환경을 관리해.

**N**: 그럼 배포는 누가 해?

**한입**: 보통 DevOps. 개발자가 코드 완성하면 DevOps가 서버에 올려줘.

**N**: 서버 느리면 누구 탓이야?

**한입**: 둘 다일 수 있어. 코드가 비효율적이면 서버 개발자 탓, 서버 성능이 부족하면 DevOps가 해결해야 하고.

**N**: 스타트업은 둘 다 하는 사람도 있다며?

**한입**: 응, 작은 팀은 개발자가 DevOps도 해. 근데 커지면 분리해. 둘 다 전문성이 필요하거든.

**N**: 그럼 나는 누구한테 물어봐야 해?

**한입**: "이 기능 어떻게 돼요?"는 서버 개발자, "언제 배포돼요?"는 DevOps. 간단하지?
