# 유저 여정 설계

## "5%만 핵심 기능을 써본다면?"

마케팅비를 앱 다운로드당 1만원씩 쓰는데, 유저들이 핵심 기능도 써보지 않고 이탈한다면 얼마나 아까울까요?

## 유저 여정이란?

### 정의

```
유저 여정 (User Journey)
= 유저가 앱에 진입한 후 핵심 가치에 도달하기까지의 흐름

처음 접속 → 가입 → 탐색 → 핵심 기능 → 재방문
```

:::callout[important]
바로 디자인부터 시작하고 싶겠지만, 유저 여정을 먼저 설계해야 해요.
안 그러면 나중에 큰 문제가 생겨요.
:::

## 실제 사례: 데이팅 앱의 실패와 성공

### 실패한 유저 여정

```
한달에 수억 마케팅비를 쓰던 데이팅 앱이 있었어요.
핵심 기능: 1:1 화상 채팅
유저 테스트 반응: 너무 좋음!

그런데... 성장하지 못하고 유저들이 계속 이탈했어요.
```

### 문제 발견

```
"핵심 가치를 경험하는 유저가 몇 %예요?"
"5% 미만이에요"

핵심 기능이 유저 여정의 너무 뒤에 있었어요!
```

### 기존 유저 여정 (9단계!)

[illust: ID="P1-W2-E01-01" DESC="실패한 유저 여정 - 9단계의 긴 깔때기(funnel) 그림. 회원가입부터 화상채팅까지 각 단계마다 유저가 이탈하는 모습"]

```
회원가입 → 프로필 입력 → 가입 완료 → 상세 프로필 입력
→ 디스커버 → 하트 보내기 → 매치 → 채팅 → 1:1 화상 채팅

각 단계마다 이탈률이 30%씩만 되어도
9단계를 거치면 5%만 남아요. 실제로 그랬고요.
```

### 개선된 유저 여정

```
[서브 플롯 추가]
회원가입 → 프로필 입력 → 가입 완료 → N:N 음성 채팅방 합류

4단계만에 핵심 경험(실시간 음성 대화)을 할 수 있게 됐어요!
```

==서브 플롯을 추가한 후, 한 달 만에 1천 명 이상이 음성 채팅을 하는 기능으로 빠르게 성장했어요.==

## 유저 여정의 역할

### 1. 지도 역할

```
유저 경험을 매끄럽게 만들어요
- 어디서 어디로 가야 하는지 명확
- 길을 잃지 않음
```

### 2. 복잡함 방지

```
제품이 발전해도 흐름이 꼬이지 않아요
- 새 기능 추가할 때 어디에 넣을지 명확
- 중복 기능 발생 방지
```

### 3. 팀 정렬

```
팀 전체가 같은 그림을 봐요
- "이 기능은 어느 단계에 있지?"
- "유저가 여기서 뭘 해야 하지?"
```

## IA(Information Architecture)로 구조화하기

### IA란?

```
IA = 앱의 정보를 어떻게 구조화하고 배치할지에 대한 전략
쉽게 말하면 '무엇을 어디에 놓을지에 대한 지도'
```

[illust: ID="P1-W2-E01-02" DESC="IA 구조도 예시 - 데이팅 앱의 5개 메인 탭(홈, 디스커버, 채팅, 좋아요, 마이페이지)과 하위 기능들이 트리 구조로 연결된 모습"]

### IA를 그리면 보이는 것들

**1. 정보의 위계**

```
1단계: 메인 탭 (홈, 디스커버, 채팅, 마이페이지)
2단계: 각 탭의 주요 기능
3단계: 세부 기능
```

**2. 중복 발견**

```
"유저가 어디서든 프로필을 볼 수 있으면 좋겠지?"
→ 여러 곳에 같은 버튼을 넣음
→ 나중에 업데이트할 때 일부만 수정되는 문제 발생
→ 데이터 분석하면 90% 이상이 한 경로만 사용

중복은 유지보수 비용만 늘려요!
```

**3. 부자연스러운 경험 발견**

```
흔한 문제들:
- 막다른 길: 매치 실패했는데 다음 안내 없음
- 순환 오류: 프로필 수정 후 원래 화면으로 못 돌아감
- 이탈 포인트: 회원가입 오류 시 처음부터 다시
```

## 실전! 30분 만에 IA 만들기

### [0-10분] 브레인덤프

```
떠오르는 기능을 전부 적기
판단하지 말고 일단 다 쏟아내기

예: 운동 기록, 통계, 캘린더, 루틴, 친구,
    랭킹, 프로필, 설정, 타이머, 가이드 영상...
```

### [10-20분] 그룹핑과 우선순위

```
핵심 질문: "이거 없으면 운동 앱이 아니야?"

필수: 운동 기록, 타이머, 기본 통계
v2.0: 친구 기능, 챌린지
v3.0: 트레이너 매칭
```

### [20-25분] 3탭 룰 체크

```
홈에서 각 기능까지 몇 번 탭해야 하는지 세기

자주 쓰는 기능 → 앞에
가끔 쓰는 기능 → 뒤에

운동 시작: 홈에서 바로
설정: 마이페이지 깊숙이
```

### [25-30분] 데드엔드 체크

```
각 화면에서 다음 액션이 명확한지 확인
막다른 길은 없는지
뒤로가기는 예측 가능한지
```

## 체크리스트

### IA 체크리스트

```
□ 핵심 기능이 3탭 이내에 있는가?
□ 중복된 기능이 여러 곳에 흩어져 있지 않은가?
□ 메뉴 이름이 직관적인가? (설정? 더보기? 기타? ❌)
```

### 유저 플로우 체크리스트

```
□ 각 화면에서 하나의 액션만 요구하는가?
□ 핵심 가치를 3분 안에 경험할 수 있는가?
□ 각 단계마다 스킵 옵션이 있는가?
□ 막다른 길 없이 다음 액션이 명확한가?
```

### 테스트 체크리스트

```
□ 5명에게 테스트했을 때 3명 이상 성공하는가?
□ 부모님이 튜토리얼 없이 사용할 수 있는가?
□ 광고 메시지와 앱의 첫 화면이 일치하는가?
```

:::callout[warning]
3개 이상 "아니오"가 나왔다면 출시를 미루고 IA부터 다시 그리세요.
지금 2주 더 투자하는 게 나중에 6개월을 갈아엎는 것보다 100배 나아요.
:::

## 정리

:::callout[quote]
"복잡함은 쌓이는 것이고, 단순함은 설계하는 것이다"
:::

**핵심:**
- 유저 여정 = 핵심 가치에 도달하기까지의 흐름
- 핵심 기능이 너무 뒤에 있으면 유저는 이탈해요
- IA로 구조화하면 문제점이 보여요
- 출시 전에 체크리스트로 검증하세요
- 처음부터 제대로 설계하면 기능이 100개가 되어도 단순함을 유지할 수 있어요
