## 유저 여정 설계하기

+------------------------------------------------------------+
| 한입: N님, 만약 유저들 중 5%만이 우리 앱의 핵심 기능을     |
| 사용해본다면 어떨 것 같아요?                               |
|                                                            |
| N: 그럼 너무 아까운데요..?                                 |
|                                                            |
| 한입: 그쵸. 마케팅비를 앱 다운로드로 나눴을 때 앱          |
| 다운로드당 1만원인데, 핵심 기능도 써보지 않고 이탈한다면   |
| 얼마나 아깝겠어요.                                         |
|                                                            |
| N: 그럼 어떻게 해야 해요?                                  |
+------------------------------------------------------------+

앱을 만들기로 했으면 가장 먼저 해야하는 것은 유저 여정(User journey)를
설계하는 것입니다. 유저가 앱 또는 웹사이트에 진입한 후 어떤 흐름으로
핵심 가치에 도달할 수 있는지 설계하는거죠.
바로 디자인부터 시작해서 알잘딱하면 될 것 같은데 왜 해야 할까요?

제가 겪었던 일화를 바탕으로 설명해보겠습니다.

한달에 몇억씩 마케팅비를 태우던 데이팅 앱이 있었습니다. 그 제품은 나름의
컨셉과 핵심 기능이 명확했음에도 성장하지 못하고 유저들은 계속
이탈했어요. 그런데 유저 테스트를 해보면 반응이 너무 좋았습니다.
스튜디오의 기획 직군은 그 이유를 '아직 선순환이 될 정도의 최소 유저를
확보하지 못했기 때문이다'로 정의했고 조만간 마케팅 비를 더 늘리기로
했습니다.

A: "저희 앱의 핵심 가치가 뭔가요?"
B: "유저들 간의 1대1 화상 채팅이에요"
A: "그 핵심 가치를 경험하는 유저는 전체 유저 중에 몇명인가요?"
B: "5% 미만이에요"
A: "우리의 핵심 가치가 유저 여정 너무 뒤에 있는거 아니에요? 핵심 가치를
경험해봤다면 이탈하지 않았을 수도 있는거잖아요"
B: "그건 그렇지만 핵심 가치를 경험하기 위한 절차들이에요. 유저들 간의
1대1 화상 통화는 굉장히 매력적이고 강력하지만 위험하기도 해요. 서로
매치가 되고난 후에 사용하는게 맞아요."
A: "아무리 그래도 잘 이해가 되지 않아요. 지금 CAC(Customer Acquistion
Cost, 고객 획득 비용)이 2만원 정도인데 그 사람들 중 거의 대부분이 핵심
가치를 경험하지 못해서 이탈하는게 맞나요? 그리고 서로 매치 이후에
사용하더라도 1대1 화상 채팅은 위험할 수 있어요. 우리의 신고 지표가
그것을 증명해주고 있어요. 그래서 심리적 허들도 높고요."

A: "우리에게는 더 안전한 오디오 채팅이 필요하고, 이걸 가입하자마자 바로
경험할 수 있는 Sub-plot 이 필요해요"

이후 이 제품은 좀 더 안전한 오디오 채팅 기반의 서브 플랏을 추가했고,
출시 한달만에 1천명 이상의 유저들이 오디오 채팅을 하는 기능으로 빠르게
성장하면서 피봇팅에 성공했습니다.

연매출 수천억원대 기업의 20명 규모 팀이 왜 수년간 실패를 반복했을까요?
바로 유저 여정을 제대로 고려하지 않았기 때문입니다. 핵심 기능만 떼어내서
실험한 유저 테스트는 현실을 반영하지 못했고, 그러다보니 제품이 성장하지
못하는 진짜 원인을 찾지 못했던 거예요.

## 유저 여정의 지도

유저 여정을 설계하는 것은 여러 장점이 있습니다. 지도 역할을 해서 유저
경험을 더 매끄럽게 하고, 제품이 발전함에 따라 복잡한 흐름이 생기는 것을
방지하기도 하며, 팀이 길을 잃지 않는데에도 도움이 됩니다.

방금 예시의 유저 여정을 그려보면 문제가 명확해집니다.

**기존의 실패한 유저 여정 (메인 플롯):**
회원가입 → 프로필 입력 → 가입 완료 → 상세 프로필 입력 → 디스커버 → 하트
보내기 → 매치 → 채팅 → 1:1 화상 채팅

9단계나 거쳐야 핵심 가치에 도달했어요. 각 단계마다 이탈률이 30%씩만
되어도 9단계를 거치면 5%만 남는 셈이죠.
실제로 그랬고요.

**개선된 유저 여정 (서브 플롯 추가):**
[메인 플롯] 기존과 동일
[서브 플롯 - 빠른 체험] 회원가입 → 프로필 입력 → 가입 완료 → N:N 음성
채팅 (N:N 음성 채팅방 탐색 → 채팅방 합류 및 소통) → 하트 보내기 → 매치

서브 플롯을 추가하니 4단계만에 핵심 경험(실시간 음성 대화)을 할 수 있게
됐고, 이 경험을 통해 "아, 이런 앱이구나!"를 바로 알 수 있게 됐죠.
부담도 덜하고 재미도 있어서 더 오래 머물렀고, 리텐션도 좋아졌습니다.

## IA(Information Architecture) - 유저 여정을 구조화하기

유저 여정의 설계가 완료되었다면 그 유저 여정을 위해 앱을 어떻게
구조화하는게 좋을 것인지 고민해야 합니다. 이때 그리는 앱의 구체적인
구조도를 IA라고 하는데요.

IA는 웹사이트나 앱의 정보를 어떻게 구조화하고 조직하며 레이블링할지를
다루는 전체 전략을 뜻하는데요, 쉽게 말하면 '무엇을 어디에, 어떤
맥락으로 배치할 것인지에 대한 지도'라고 보시면 됩니다.

이렇게 구조화하면 몇 가지가 명확해집니다:

**1. 정보의 위계가 보인다**
- 1단계: 5개 메인 탭 (홈, 디스커버, 채팅, 좋아요, 마이페이지)
- 2단계: 각 탭의 주요 기능 (프로필 카드, 매치 리스트 등)
- 3단계: 세부 기능 (필터 옵션, 화상 채팅 버튼 등)

**2. 중복을 발견하고 대처할 수 있다**

**3. 부자연스러운 유저 경험을 발견할 수 있다**

특히 이런 문제들이 자주 발견됩니다:
- **막다른 길(Dead end)**: 매치 실패했는데 다음에 뭘 해야 할지 안내가 없음
- **순환 구조 오류**: 프로필 수정하러 들어갔는데 원래 화면으로 돌아갈 수 없음
- **이탈 포인트**: 회원가입 중 오류가 났는데 처음부터 다시 해야 함

## 실전! 30분 만에 IA 만들기

**[0-10분] 브레인덤프** 
떠오르는 기능을 전부 적어보세요.

**[10-20분] 그룹핑과 우선순위** 
비슷한 것끼리 묶고, MVP에 넣을 것과 뺄 것을 정합니다.

**[20-25분] 3탭 룰 체크** 
홈에서 각 기능까지 몇 번 탭해야 하는지 세어보세요.

**[25-30분] 데드엔드 체크** 
각 화면에서 다음 액션이 명확한지 확인하세요.

## 체크리스트 - 출시 전 마지막 점검

**IA 체크리스트:**
□ 핵심 기능이 3탭 이내에 있는가?
□ 중복된 기능이 여러 곳에 흩어져 있지 않은가?
□ 메뉴 이름이 직관적인가?

**유저 플로우 체크리스트:**
□ 각 화면에서 하나의 액션만 요구하는가?
□ 핵심 가치를 3분 안에 경험할 수 있는가?
□ 각 단계마다 스킵 옵션이 있는가?
□ 막다른 길 없이 다음 액션이 명확한가?

**테스트 체크리스트:**
□ 5명에게 테스트했을 때 3명 이상 성공하는가?
□ 부모님이 튜토리얼 없이 사용할 수 있는가?
□ 광고 메시지와 앱의 첫 화면이 일치하는가?

**"복잡함은 쌓이는 것이고, 단순함은 설계하는 것이다"**
