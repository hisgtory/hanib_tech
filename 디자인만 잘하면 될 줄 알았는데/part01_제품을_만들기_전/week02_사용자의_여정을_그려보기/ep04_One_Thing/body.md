# One Thing

## "이것저것 다 넣고 싶은데..."

기획할 때 가장 흔한 실수가 있어요. "이것도 넣자, 저것도 넣자" 하다가 서비스가 뚱뚱해지는 거예요.

## One Thing의 힘

### 성공한 서비스들의 공통점

```
카카오톡: 무료 메시지
토스: 간편 송금
배달의민족: 배달 주문
당근마켓: 동네 중고거래

→ 처음엔 딱 하나만 잘했어요
```

### 지금의 기능들은?

```
카카오톡 현재 기능:
- 메시지, 통화, 영상통화
- 카카오페이, 선물하기
- 카카오T, 카카오맵
- 이모티콘, 게임
- ...

→ 이건 10년에 걸쳐 추가된 거예요
→ 처음부터 다 있었던 게 아니에요
```

:::callout[important]
처음부터 많은 기능을 넣으면 아무것도 제대로 못해요.
:::

## 왜 하나에 집중해야 할까?

### 1. 리소스의 한계

```
개발 리소스가 100이라면:

A안: 기능 10개에 각각 10씩
→ 모든 기능이 어중간

B안: 기능 1개에 100 올인
→ 하나가 완벽

→ 사용자는 어중간한 10개보다
   완벽한 1개에 끌려요
```

### 2. 사용자 혼란

```
"이 앱 뭐하는 앱이야?"

기능이 많으면:
- 설명하기 어려움
- 기억에 안 남음
- 차별점 없음

기능이 하나면:
- "아, 그거 하는 앱!"
- 바이럴 쉬움
- 명확한 포지셔닝
```

### 3. 검증의 어려움

```
기능이 10개일 때:
사용자가 안 쓰면...
- 어떤 기능이 문제지?
- 다 안 좋은 건가?
- 어디를 고쳐야 하지?

기능이 1개일 때:
사용자가 안 쓰면...
- 이 기능이 필요 없거나
- 실행이 잘못됐거나
→ 원인 파악이 쉬움
```

## One Thing 찾는 법

### 질문 1: 핵심 가치는?

```
우리 서비스가 제공하는 가장 큰 가치는?

예시:
- "시간 절약" → 어떻게?
- "돈 절약" → 어떻게?
- "재미" → 어떤 재미?

→ 가장 구체적인 답을 찾으세요
```

### 질문 2: 없으면 안 되는 건?

```
이 기능이 없으면 서비스가 성립 안 되는가?

배달의민족:
- 주문 기능 없음 → 서비스 불가 ✗
- 리뷰 기능 없음 → 서비스 가능 ○

→ 없으면 안 되는 것 = Core
→ 있으면 좋은 것 = Nice to have
```

### 질문 3: 경쟁자와 다른 점?

```
같은 카테고리에서 우리만의 차별점은?

중고거래 앱이라면:
- 당근: 동네 기반 (빠른 거래)
- 번개장터: 취미 기반 (관심사 매칭)

→ 우리의 한 가지는?
```

## One Thing의 함정

### "나중에 빼면 되지"

```
❌ 잘못된 생각:
"일단 다 넣고, 안 쓰면 빼자"

왜 안 될까:
- 빼는 건 넣는 것보다 어려움
- 쓰는 사람이 한 명이라도 있으면 못 뺌
- 유지보수 비용은 계속 발생
```

### "경쟁사도 있으니까"

```
❌ 잘못된 생각:
"경쟁사에 있는 기능은 우리도 있어야지"

왜 안 될까:
- 경쟁사 따라하면 카피캣
- 모든 기능을 따라할 수 없음
- 우리만의 강점이 사라짐
```

:::callout[tip]
"빼는 용기"가 좋은 서비스를 만들어요.
:::

## 실제 적용하기

### MVP 기능 정하기

```
Step 1: 모든 아이디어 나열
- 기능 A
- 기능 B
- 기능 C
- ...

Step 2: Must have / Nice to have 분류
- Must have: 없으면 서비스 안 됨
- Nice to have: 있으면 좋음

Step 3: Must have 중에서도 핵심 1개 선택
- 이것만 완벽하면 된다

Step 4: 나머지는 v2 백로그로
```

### 팀 설득하기

```
"이 기능도 넣으면 좋지 않아요?"

→ "맞아요, 좋을 것 같아요.
   근데 지금 리소스로는 이것 하나도 
   제대로 못할 것 같아요.
   이거 먼저 성공하고, 
   다음에 추가하면 어떨까요?"
```

## 정리

:::callout[quote]
"완벽함이란 더 이상 더할 것이 없을 때가 아니라, 더 이상 뺄 것이 없을 때 달성된다."
- 생텍쥐페리
:::

**핵심:**
- 처음엔 딱 하나만 잘하세요
- 나머지는 성공하고 나서 추가해도 늦지 않아요
- "빼는 용기"가 좋은 서비스를 만들어요
- 어중간한 10개보다 완벽한 1개가 낫습니다
