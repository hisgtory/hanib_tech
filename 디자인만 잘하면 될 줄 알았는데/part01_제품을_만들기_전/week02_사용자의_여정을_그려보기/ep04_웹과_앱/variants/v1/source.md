## 앱 vs 웹, 기술적 차이

**앱 (Native App)**
- iOS/Android 각각 별도 개발 필요
- 앱스토어를 통해 배포
- 기기의 기능(카메라, GPS, 푸시 알림) 직접 접근 가능
- 오프라인에서도 일부 기능 사용 가능

**웹 (Web App)**
- 하나의 코드로 모든 기기에서 작동
- URL로 바로 접근 가능
- 브라우저를 통해 기기 기능 제한적 접근
- 인터넷 연결 필수

## 업데이트와 배포의 차이

이 차이가 실무에서는 정말 크게 다가옵니다.

**앱 업데이트:**
1. 코드 수정
2. 앱스토어 심사 제출 (iOS 1-3일, Android 몇 시간)
3. 심사 통과
4. 유저가 업데이트 설치

**웹 업데이트:**
1. 코드 수정
2. 서버 배포
3. 유저가 새로고침하면 바로 적용

버그가 발견됐을 때 웹은 30분 만에 고칠 수 있지만,
앱은 심사 때문에 며칠이 걸릴 수 있어요.

## 개발 비용과 시간

| 구분 | 네이티브 앱 | 웹 |
|------|------------|-----|
| 개발 인력 | iOS + Android + 서버 | 프론트엔드 + 서버 |
| 개발 기간 | 3-6개월 | 1-3개월 |
| 유지보수 | 두 플랫폼 각각 | 하나만 |
| 비용 | 높음 | 중간 |

**하이브리드 옵션도 있어요:**

- **React Native / Flutter**: 하나의 코드로 iOS, Android 앱 개발
- **PWA (Progressive Web App)**: 웹이지만 앱처럼 설치 가능

## 언제 앱을 선택할까?

**앱이 필수인 경우:**
- 푸시 알림이 핵심 기능일 때 (배달, 메시징)
- 카메라/GPS를 자주 사용할 때
- 오프라인 기능이 필요할 때
- 높은 성능이 필요할 때 (게임, 영상 편집)

**웹이 더 나은 경우:**
- 빠른 출시가 중요할 때
- 검색 유입이 중요할 때 (콘텐츠, 커머스)
- 가벼운 기능만 필요할 때
- 예산이 제한적일 때

## 실전 의사결정 프레임워크

```
1. 푸시 알림이 핵심인가?
   → Yes: 앱 고려
   
2. 카메라/GPS를 자주 쓰는가?
   → Yes: 앱 고려
   
3. 빠른 출시가 중요한가?
   → Yes: 웹 우선
   
4. 검색 유입이 중요한가?
   → Yes: 웹 우선
   
5. 예산이 충분한가?
   → No: 웹 우선 또는 하이브리드
```

**"처음엔 웹으로 시작하고, 
PMF(Product-Market Fit)를 찾으면 앱으로 확장하라"**

이게 많은 스타트업이 선택하는 전략이에요.
토스도, 당근도 처음부터 완벽한 앱은 아니었습니다.
