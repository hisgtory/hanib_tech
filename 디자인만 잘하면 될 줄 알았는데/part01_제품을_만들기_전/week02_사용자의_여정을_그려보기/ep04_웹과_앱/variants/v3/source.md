## 고객 획득 비용 (CAC)

**웹의 장점:**
- SEO를 통한 무료 유입 가능
- 링크 공유가 쉬움 (바이럴)
- 랜딩페이지로 빠른 전환 유도
- 광고 클릭 → 즉시 사용 가능

**앱의 장점:**
- 앱스토어 검색 노출
- 앱스토어 피처링 (무료 홍보)
- 설치 후 지속적 관계 형성

**비용 비교:**
| 채널 | 웹 | 앱 |
|------|-----|-----|
| SEO | 낮음 | 해당없음 |
| 검색광고 | 중간 | 높음 |
| SNS 광고 | 중간 | 높음 |
| 인플루언서 | 중간 | 중간 |

앱은 "설치"라는 허들이 있어서 전환율이 낮고, 
결과적으로 CAC가 높아지는 경향이 있어요.

## 리텐션과 인게이지먼트

여기서는 앱이 압도적으로 유리합니다.

**푸시 알림의 위력:**
- 이메일 오픈율: 15-25%
- 푸시 알림 오픈율: 40-60%

배달, 커머스, SNS처럼 반복 사용이 중요한 서비스는
푸시 알림이 매출에 직접적인 영향을 미쳐요.

**앱 아이콘 = 광고판:**
유저 폰에 앱 아이콘이 있다는 것 자체가 리마인더예요.
"아, 운동해야지" 하면서 앱 아이콘을 보는 것처럼요.

## 수익화 모델별 선택

**구독 모델:**
- 앱: 인앱결제 수수료 30% (애플/구글)
- 웹: 결제 수수료 3-5% (PG사)
- → 웹이 마진에 유리

**광고 모델:**
- 앱: 인앱 광고 단가 높음
- 웹: 디스플레이 광고 단가 낮음
- → 앱이 유리

**커머스:**
- 앱: 재구매율 높음, 푸시로 프로모션
- 웹: 검색 유입, 비교 구매
- → 둘 다 필요

## 스타트업 단계별 전략

**초기 (PMF 찾는 중):**
- 웹 우선 추천
- 빠른 실험과 피봇 가능
- 개발 비용 절약

**성장기 (PMF 찾음):**
- 앱 개발 시작
- 웹과 앱 병행
- 리텐션 강화

**확장기:**
- 앱에 투자 집중
- 차별화된 앱 경험
- 오프라인 연계

## 투자자가 "앱 있어요?"라고 묻는 이유

**긍정적 해석:**
- 앱이 있으면 리텐션 데이터가 명확함
- 앱스토어 순위 = 시장 검증
- 모바일 우선 시대에 맞는 전략

**하지만:**
- 웹 기반 성공 사례도 많음 (노션, 피그마, 슬랙)
- 앱 개발 비용이 runway를 갉아먹을 수 있음
- 중요한 건 "앱이 있느냐"가 아니라 "성장하고 있느냐"

## 의사결정 체크리스트

**앱을 먼저 만들어야 하는 경우:**
□ 푸시 알림이 매출에 직접적 영향
□ 하루에 여러 번 사용하는 서비스
□ 오프라인 기능 필수
□ 카메라/위치 기반 핵심 기능

**웹을 먼저 만들어야 하는 경우:**
□ SEO가 중요한 콘텐츠/커머스
□ B2B 또는 데스크톱 사용이 많은 서비스
□ 예산이 제한적
□ 빠른 실험이 필요한 초기 단계

**"기술이 아니라 비즈니스가 플랫폼을 결정한다"**

결국 중요한 건 유저가 어디서, 어떻게 우리 서비스를 사용하느냐예요.
그 답이 웹인지 앱인지 알려줄 거예요.
