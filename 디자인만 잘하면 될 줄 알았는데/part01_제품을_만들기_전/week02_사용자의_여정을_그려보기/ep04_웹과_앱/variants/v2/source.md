## 사용 맥락의 차이

**앱 사용자:**
- 이동 중에 한 손으로 사용
- 짧은 시간, 여러 번 사용
- 푸시 알림으로 다시 돌아옴
- 화면이 작음

**웹 사용자:**
- 책상에 앉아 마우스로 사용
- 긴 시간, 집중해서 사용
- 검색이나 링크로 유입
- 화면이 큼

## 모바일 앱의 UX 원칙

**1. 엄지 존(Thumb Zone) 고려**

스마트폰을 한 손으로 잡았을 때, 엄지가 닿기 쉬운 영역이 있어요.
중요한 버튼은 이 영역에 배치해야 합니다.

- 화면 하단: 쉬움 (내비게이션 바)
- 화면 중앙: 보통
- 화면 상단: 어려움 (덜 중요한 것들)

**2. 한 화면에 하나의 액션**

앱에서는 "One Thing at a Time" 원칙이 중요해요.
복잡한 폼을 한 화면에 다 보여주지 말고,
단계별로 나눠서 보여주세요.

**3. 제스처 활용**

앱에서는 스와이프, 롱프레스 같은 제스처를 활용할 수 있어요.
- 스와이프로 삭제
- 당겨서 새로고침
- 롱프레스로 메뉴 표시

## 웹의 UX 원칙

**1. 정보 밀도 높이기**

화면이 크니까 더 많은 정보를 보여줄 수 있어요.
테이블, 대시보드 같은 복잡한 레이아웃이 가능합니다.

**2. 키보드 단축키**

파워 유저들은 마우스보다 키보드를 선호해요.
Ctrl+S, Ctrl+K 같은 단축키를 제공하면 좋습니다.

**3. 호버 상태 활용**

마우스가 있으니까 호버(마우스 올려놓기)를 활용할 수 있어요.
- 호버하면 미리보기 표시
- 호버하면 추가 옵션 표시

## 같은 서비스, 다른 UX의 예시

**노션(Notion):**
- 웹: 사이드바 항상 표시, 풍부한 단축키
- 앱: 사이드바 숨김, 간단한 에디터

**인스타그램:**
- 웹: 탐색 중심, DM 제한적
- 앱: 촬영/스토리 중심, 풍부한 인터랙션

**배달의민족:**
- 웹: 메뉴 비교, 리뷰 탐색에 최적화
- 앱: 빠른 재주문, 푸시 알림

## 반응형 vs 적응형

**반응형 (Responsive):**
- 하나의 코드가 화면 크기에 맞춰 변함
- 개발 비용 낮음, 유지보수 쉬움
- 최적화된 경험 제공 어려움

**적응형 (Adaptive):**
- 디바이스별로 다른 버전 제공
- 개발 비용 높음
- 최적화된 경험 가능

**언제 뭘 선택할까?**
- 리소스 부족, 빠른 출시 필요 → 반응형
- 모바일 경험이 매우 중요 → 적응형 (별도 앱)

## 체크리스트

**앱 UX 체크리스트:**
□ 중요한 버튼이 엄지 존에 있는가?
□ 한 화면에 하나의 목표만 있는가?
□ 로딩 시간이 짧은가?
□ 오프라인에서도 기본 동작하는가?

**웹 UX 체크리스트:**
□ 큰 화면을 효율적으로 활용하는가?
□ 키보드로 주요 기능 사용 가능한가?
□ 호버 상태가 정의되어 있는가?
□ 페이지 로딩 속도가 빠른가?

**"좋은 UX는 사용자가 어디서 사용하든 자연스러워야 한다"**
