# 웹과 앱

## "우리 서비스, 웹으로 할까요 앱으로 할까요?"

새 서비스를 시작할 때 가장 먼저 마주치는 질문이에요. 정답은 없지만, 선택을 위한 기준은 있어요.

## 웹과 앱의 차이

### 웹 (Web)

```
특징:
- 브라우저에서 접속
- 설치 불필요
- URL만 있으면 접근 가능
- 업데이트 즉시 반영

예시:
- 네이버, 구글
- 쇼핑몰, 블로그
- 회사 홈페이지
```

### 앱 (App)

```
특징:
- 스토어에서 다운로드
- 설치 필요
- 기기 기능 활용 가능 (카메라, GPS, 푸시 알림)
- 오프라인에서도 일부 기능 사용 가능

예시:
- 카카오톡, 인스타그램
- 배달의민족, 토스
- 게임
```

## 선택 기준

### 웹이 적합한 경우

```
✓ 검색을 통해 유입되어야 할 때
  - SEO(검색 최적화)가 중요하다면
  - 콘텐츠 중심 서비스

✓ 설치 장벽을 낮추고 싶을 때
  - 링크 하나로 바로 접근
  - 첫 사용자 전환이 중요

✓ 빠른 업데이트가 필요할 때
  - 앱스토어 심사 없이 즉시 반영
  - 자주 변경되는 서비스

✓ 개발 비용을 줄이고 싶을 때
  - 하나만 만들면 모든 기기에서 작동
```

### 앱이 적합한 경우

```
✓ 자주 사용하는 서비스일 때
  - 매일 접속하는 서비스
  - 홈 화면에 아이콘 필요

✓ 기기 기능이 필요할 때
  - 카메라, GPS, 센서
  - 푸시 알림

✓ 오프라인 사용이 필요할 때
  - 인터넷 없이도 작동
  - 데이터 로컬 저장

✓ 결제가 핵심일 때
  - 인앱 결제
  - 정기 구독
```

:::callout[tip]
"둘 다"도 답이 될 수 있어요. 많은 서비스가 웹과 앱을 함께 운영해요.
:::

## 실제 사례로 보기

### 배달의민족

```
앱 중심 서비스인 이유:
- 위치 기반 (GPS 필수)
- 푸시 알림 (배달 상태)
- 자주 사용 (주 3-4회)
- 결제 중심

웹도 있지만:
- 메뉴 검색용
- SEO용 (검색 유입)
```

### 브런치 (Brunch)

```
웹 중심 서비스인 이유:
- 콘텐츠 중심 (글 읽기)
- 검색 유입 중요 (SEO)
- 설치 장벽 낮춤
- 가끔 방문

앱도 있지만:
- 열성 독자용
- 푸시 알림 (새 글)
```

### 토스

```
앱 전용인 이유:
- 보안이 중요 (금융)
- 매일 사용
- 푸시 알림 필수 (송금, 이체)
- 생체 인증 (지문, Face ID)

웹이 없는 이유:
- 보안 리스크
- 기기 인증 어려움
```

## MVP 단계에서의 선택

### 빠른 검증이 목적이라면

```
웹 먼저 권장:

1. 개발 속도 빠름
2. 배포 즉시 가능
3. 수정 쉬움
4. 비용 적음

→ 가설 검증 후 앱 개발 결정
```

### 앱이 필수라면

```
한 플랫폼만 먼저:

iOS vs Android 중 하나 선택
- 타겟 유저가 많이 쓰는 것
- 한국: 보통 Android 우세
- 젊은층: iOS 비중 높음

→ 성공하면 나머지 플랫폼 추가
```

:::callout[important]
처음부터 웹, iOS, Android 다 만들려고 하지 마세요. 리소스가 분산돼요.
:::

## 하이브리드 옵션

### 웹앱 (PWA)

```
Progressive Web App:
- 웹인데 앱처럼 동작
- 홈 화면에 설치 가능
- 푸시 알림 가능
- 오프라인 지원

장점:
- 한 번 개발로 웹+앱 효과
- 앱스토어 심사 불필요

단점:
- 네이티브보다 기능 제한
- iOS에서 일부 기능 미지원
```

### 크로스 플랫폼

```
React Native, Flutter:
- 한 코드로 iOS + Android
- 네이티브에 가까운 성능

장점:
- 개발 비용 절감
- 유지보수 편함

단점:
- 복잡한 기능에서 한계
- 네이티브 개발자 필요할 수 있음
```

## 디자이너가 알아야 할 것

### 플랫폼별 UX 차이

```
웹:
- 마우스/키보드 중심
- 큰 화면 활용
- 뒤로가기 = 브라우저 버튼
- URL 공유 쉬움

앱:
- 터치 중심
- 작은 화면 최적화
- 뒤로가기 = 제스처 or 버튼
- 딥링크 필요
```

### 디자인 가이드라인

```
iOS: Human Interface Guidelines (HIG)
Android: Material Design

→ 각 플랫폼 사용자가 익숙한 패턴을 따르세요
```

## 정리

:::callout[important]
웹과 앱 중 "정답"은 없어요. 
서비스 특성, 타겟 유저, 핵심 기능을 고려해서 선택하세요.
MVP 단계에서는 하나에 집중하는 게 좋아요.
:::

**핵심:**
- 검색/콘텐츠 중심이면 웹
- 자주 쓰고 기기 기능 필요하면 앱
- 둘 다 필요하면 순서를 정해서
- 처음부터 다 만들려고 하지 마세요
