# 핵심 가치를 앞당기는 방법

## "가입하면 좋은 거 보여드릴게요"

회원가입 먼저 하라는 앱, 솔직히 귀찮지 않나요? 성공하는 앱들은 다르게 합니다.

## 핵심 가치 먼저 보여주기

### 성공한 앱들의 공통점

[illust: ID="P1-W2-E02-01" DESC="핵심 가치 우선 전략 - 틱톡, 스포티파이, 핀터레스트 앱 아이콘과 각각의 핵심 경험(영상 재생, 음악 미리듣기, 이미지 탐색)을 화살표로 연결한 그림"]

```
틱톡: 앱 켜자마자 영상 재생, 가입은 나중에
스포티파이: 가입 없이 30초 미리듣기
핀터레스트: 탐색은 자유, 저장할 때만 가입 요구
미디엄: 월 3개 기사는 무료로 읽기
```

:::callout[important]
핵심 가치에 마음이 흔들린다면, 유저는 회원가입 같은 번거로운 절차도 기꺼이 진행해요.
:::

## One Thing at a Time

### 모바일 시대의 황금률

```
한 화면에서 하나의 액션만 요구하세요.
유저가 직관적으로 인식하고 빠르게 행동할 수 있어요.
```

### 운동 앱 예시

**나쁜 예: 한 화면에 다 넣기**

```
운동 종목 선택
무게 입력
세트 수 입력
횟수 입력
휴식시간 설정
메모 입력
[저장]

→ 완료율 낮음
```

**좋은 예: 단계별로 나누기**

```
[화면 1] 운동 종목은? → 스쿼트
[화면 2] 무게는? → 60kg
[화면 3] 몇 세트? → 5세트
[화면 4] 저장할까요? → 저장

→ 각 단계가 단순해서 오히려 빠르게 느껴져요
```

==인스타그램, 배달의민족, 당근, 에어비앤비, 토스 등에서 매일 경험하는 검증된 UX 패턴이에요.==

### 반면에...

```
한 페이지에 가득한 입력창 = 은행, 공공기관 UX

왜 우리가 그런 서비스를 싫어하는지 생각해보세요.
```

## 광고에서부터 핵심 가치

### 광고도 핵심 가치를 보여줘야 해요

```
❌ 밈을 활용한 재미있는 광고
   → 눈길은 가는데 뭐하는 앱인지 모름
   → 클릭 안 함

✅ "송금 수수료 없음, 10초 만에 송금 완료"
   → 핵심 가치가 명확
   → 필요한 사람은 바로 클릭
```

### 토스의 사례

```
토스는 핵심 가치("간편 송금")를 광고로 직접 보여줬어요.
높은 클릭율로 시장에서 원한다는 것을 광고만으로 검증했다고 해요.
```

### 학습 앱들도 마찬가지

```
스픽, 말해보카의 광고:
실제 학습 화면을 보여줌
→ "이런 식으로 공부하는구나" 바로 이해
→ 관심 있는 사람만 다운로드
```

## IA 설계에 반영하기

### 복잡한 프로세스는 여러 단계로

```
IA 설계할 때 미리 고려하세요:

1. 복잡한 프로세스 → 여러 단계로 쪼개기
2. 각 단계 → 최대한 단순하게
3. 핵심 가치 → 최대한 앞으로

그래야 나중에 디자인 단계에서
구조를 다시 바꾸는 일이 없어요.
```

### 웹도 마찬가지

```
웹 트래픽의 60% 이상이 모바일이에요.
그래서 웹도 Mobile First로 설계하는 케이스가 늘고 있어요.
```

## 체크리스트

```
□ 핵심 가치를 회원가입 없이 경험할 수 있는가?
□ 각 화면에서 하나의 액션만 요구하는가?
□ 광고에서 핵심 가치가 명확하게 전달되는가?
□ 복잡한 프로세스가 단계별로 나뉘어 있는가?
□ 핵심 가치까지 3단계 이내에 도달 가능한가?
```
