## 전환율 데이터가 말해주는 것

**필드 개수별 폼 완료율:**
| 필드 개수 | 완료율 |
|-----------|--------|
| 3개 | 25% |
| 5개 | 20% |
| 7개 | 15% |
| 10개 이상 | 10% 미만 |

필드 하나가 추가될 때마다 전환율이 약 4-5% 떨어져요.

## 실제 A/B 테스트 사례

**사례 1: 이커머스 체크아웃**
- A: 원페이지 체크아웃 (모든 정보 한 화면)
- B: 단계별 체크아웃 (배송 → 결제 → 확인)
- 결과: B가 전환율 23% 높음

**사례 2: SaaS 온보딩**
- A: 10개 질문 한 번에
- B: 3개씩 나눠서 3단계
- 결과: B가 완료율 67% 높음

**사례 3: 설문조사**
- A: 한 페이지에 20개 질문
- B: 페이지당 5개 질문, 4페이지
- 결과: B가 완료율 2배

## 단계별 분리의 심리학

**1. 시작이 쉬워 보임**
"이메일만 입력하면 되네?" → 일단 시작
시작하면 완료할 확률이 높아져요 (Zeigarnik 효과)

**2. 작은 성취감**
각 단계를 완료할 때마다 도파민 분비
"다음" 버튼 누를 때마다 기분이 좋아짐

**3. 매몰 비용 효과**
"여기까지 왔는데..."
이미 투자한 시간 때문에 포기하기 어려움

## 어디까지 나눌 수 있을까?

**극단적 분리: Typeform 스타일**
- 질문 하나당 한 화면
- 장점: 집중도 극대화, 모바일 최적화
- 단점: 전체 과정이 길게 느껴질 수 있음

**적정 분리: 2-3개씩 그룹핑**
- 관련 있는 정보끼리 묶기
- 이름+이메일 / 비밀번호 / 추가정보
- 대부분의 서비스에 적합

**주의: 과도한 분리는 역효과**
- 너무 많은 단계 → "언제 끝나?"
- 적정선: 3-5단계

## 전환율 개선 실전 가이드

**Step 1: 현재 폼 분석**
- 총 필드 개수 세기
- 각 필드별 이탈률 측정

**Step 2: 필수 vs 선택 분리**
- 정말 지금 필요한 정보만 필수로
- 나머지는 나중에 또는 선택으로

**Step 3: 단계 나누기**
```
1단계: 핵심 정보 (이메일/비밀번호)
2단계: 기본 정보 (이름/전화번호)
3단계: 부가 정보 (생년월일/성별) - 스킵 가능
```

**Step 4: A/B 테스트**
- 기존 vs 새 버전 비교
- 최소 1주일, 충분한 샘플 수집

## 측정해야 할 지표

| 지표 | 설명 |
|------|------|
| 폼 시작률 | 첫 필드에 입력 시작한 비율 |
| 단계별 이탈률 | 각 단계에서 떠난 비율 |
| 폼 완료율 | 최종 제출까지 완료한 비율 |
| 완료 시간 | 시작부터 완료까지 걸린 시간 |

**"측정하지 않으면 개선할 수 없다"**

데이터를 보면서 계속 개선해나가세요.
