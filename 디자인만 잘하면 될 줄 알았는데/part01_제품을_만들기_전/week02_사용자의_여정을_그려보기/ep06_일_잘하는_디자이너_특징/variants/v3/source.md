## 계속 성장하는 디자이너의 마인드셋

### 1. 끊임없이 배우는 자세

**멈추는 디자이너:**
- "나는 5년 차니까 이 정도면 됐지"
- "요즘 애들이 쓰는 툴은 굳이..."
- "트렌드? 유행 따라가면 안 돼"

**성장하는 디자이너:**
- "AI 툴 써봤는데 생각보다 쓸만하던데?"
- "주니어한테 피그마 신기능 배웠어요"
- "트렌드 중에서 우리 제품에 적용할 건 뭐가 있을까?"

:::callout[quote]
"전문가란 해당 분야에서 
저지를 수 있는 모든 실수를 
이미 저질러본 사람이다."
- 닐스 보어
:::

**성장하는 디자이너의 학습 루틴:**

| 일간 | 주간 | 월간 |
|-----|-----|-----|
| 좋은 앱 UI 스크린샷 | 디자인 아티클 3개 | 사이드 프로젝트 |
| 새 기능 출시 체크 | Dribbble/Behance 탐색 | 컨퍼런스/밋업 참석 |
| 오늘 배운 것 메모 | 피드백 받은 것 정리 | 포트폴리오 업데이트 |

### 2. 비즈니스를 이해하려는 노력

**디자인만 아는 디자이너:**
> "이 버튼 색상은 브랜드 컬러에 맞춰서..."

**비즈니스도 아는 디자이너:**
> "이 CTA 버튼 최적화하면 전환율 15% 올릴 수 있어요.
> 월 매출로 치면 약 3천만 원 차이입니다."

:::callout[important]
PM 회의에 같이 들어가 보세요.
디자인 결정에 "왜"가 보이기 시작합니다.
:::

**디자이너가 알면 좋은 비즈니스 지표:**

```
사용자 관련:
├── DAU/MAU (활성 사용자 수)
├── Retention (재방문율)
└── NPS (추천 의향 점수)

비즈니스 관련:
├── Conversion Rate (전환율)
├── ARPU (사용자당 매출)
└── CAC (고객 획득 비용)

디자인이 영향 미치는 것:
├── Task Success Rate
├── Time on Task  
└── Error Rate
```

### 3. 실패를 성장의 재료로

**실패를 피하는 디자이너:**
- 안전한 디자인만 한다
- 도전적인 프로젝트를 피한다
- 실패하면 남 탓을 한다

**실패에서 배우는 디자이너:**
- "이번에 망했는데, 원인을 찾아봤어요"
- "A/B 테스트 졌지만 인사이트를 얻었어요"
- "다음에는 이렇게 해볼게요"

**실패 회고 템플릿:**

```markdown
## 프로젝트: [이름]

### 무엇이 안 됐나?
- 

### 왜 안 됐을까?
- (표면적 원인)
- (근본 원인)

### 다음에는?
- (구체적 액션)

### 이번에 배운 것
- 
```

## 성장을 위한 실전 액션

### 🎯 이번 주 해볼 것

**Day 1-2: 현재 레벨 체크**
- 최근 작업물 3개의 강점/약점 분석
- 롤모델 디자이너 3명 선정

**Day 3-4: 학습 계획 수립**
- 부족한 영역 1개 선정
- 관련 강의/아티클 3개 북마크

**Day 5-7: 실행**
- 사이드 프로젝트 주제 정하기
- 피드백 받을 사람 섭외

### 📚 추천 리소스

**UI/UX 트렌드:**
- Mobbin (앱 UI 레퍼런스)
- Refero (웹 UI 레퍼런스)
- Nielsen Norman Group (UX 리서치)

**비즈니스 이해:**
- Product Hunt (신규 서비스 트렌드)
- 팀 PM의 회의 참관
- MAU, Conversion 등 지표 대시보드 접근 요청

**소프트 스킬:**
- 디자인 크리틱 세션 참여
- 주니어 멘토링 해보기
- 사내 스터디 그룹 만들기
