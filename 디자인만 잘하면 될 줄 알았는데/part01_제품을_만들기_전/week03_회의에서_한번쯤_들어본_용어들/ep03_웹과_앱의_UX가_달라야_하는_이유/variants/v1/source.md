# 웹과 앱의 UX가 달라야 하는 이유

## 스토리텔링 버전

:::conversation
**N**: 한입님, 저희 웹사이트 디자인을 그대로 앱으로 옮기면 안 될까요? 똑같은 서비스인데 두 번 디자인하는 게 비효율적인 것 같아서요.

**한입**: 그렇게 생각하실 수 있어요. 근데 N님, 지금 커피 마시고 계시죠?

**N**: 네? 아, 네 마시고 있어요.

**한입**: 그 커피, 책상에 앉아서 마시는 것과 걸어가면서 마시는 거랑 같을까요?

**N**: 음... 같은 커피지만 느낌이 다르긴 하죠. 걸으면서 마시면 뚜껑 달린 텀블러가 편하고...

**한입**: 바로 그거예요! 웹과 앱도 마찬가지예요.
:::

### 사용 환경이 완전히 달라요

**웹 사용자는:**
- 책상에 앉아서
- 마우스와 키보드로
- 큰 모니터를 보면서
- 집중해서 오래 사용해요

**앱 사용자는:**
- 지하철에서 한 손으로 손잡이 잡고
- 엄지손가락 하나로
- 작은 화면을 보면서
- 짧게 여러 번 사용해요

:::callout[important]
지하철에서 한 손으로 폰 들고 사용한다고 상상해보세요. 웹처럼 복잡한 UI가 가능할까요?
:::

### 호버가 없다는 건 큰 차이예요

:::conversation
**N**: 호버요? 마우스 올리면 뭔가 뜨는 거요?

**한입**: 네! 웹에서는 마우스를 올리면 툴팁이 뜨고, 메뉴가 펼쳐지고, 미리보기가 되죠. 근데 앱에서는요?

**N**: 아... 손가락을 올려놓을 수가 없네요. 누르거나 안 누르거나죠.

**한입**: 맞아요. 그래서 웹에서 호버로 처리하던 모든 것들을 앱에서는 다른 방식으로 해결해야 해요.
:::

**웹에서 가능한 것:**
- 클릭 (정확한 포인터)
- 호버 (마우스 올려서 미리보기)
- 오른쪽 클릭 (컨텍스트 메뉴)

**앱에서 가능한 것:**
- 탭 (손가락이라 부정확)
- 스와이프 (밀어서 넘기기)
- 롱프레스 (꾹 누르기)
- 핀치 줌 (두 손가락으로 확대)

### 네비게이션도 완전히 달라요

**웹:**
- 상단에 수평 메뉴
- 드롭다운으로 하위 메뉴
- 로고 클릭하면 홈으로

**앱:**
- 하단에 탭바 (엄지가 닿기 쉬운 곳!)
- 스와이프로 뒤로 가기
- 상단은 현재 화면 제목과 액션 버튼만

### 흔한 실수들

:::conversation
**N**: 그럼 웹 디자인 그대로 앱으로 옮기면 어떤 문제가 생겨요?

**한입**: 몇 가지 대표적인 문제가 있어요.

1. **호버 효과 그대로 사용** → 앱에서는 아예 동작 안 해요
2. **작은 클릭 영역** → 손가락으로 정확히 누르기 어려워요
3. **복잡한 폼** → 모바일 키보드로 입력하기 불편해요
4. **마우스 우클릭 메뉴** → 앱에서는 롱프레스로 대체해야 해요

**N**: 그럼 반응형 웹으로 만들면 해결되지 않나요?

**한입**: 반응형은 화면 크기만 조절할 뿐, 인터랙션 방식은 바꾸지 않아요. 호버도 그대로고, 터치 최적화도 안 되어 있죠.
:::

### 핵심 정리

| 구분 | 웹 | 앱 |
|------|-----|-----|
| 사용 환경 | 책상, 집중 | 이동 중, 짧게 |
| 입력 방식 | 마우스, 키보드 | 손가락 |
| 호버 | 가능 | 불가능 |
| 네비게이션 | 상단 메뉴 | 하단 탭바 |
| 터치 영역 | 작아도 됨 | 최소 44pt |

:::callout[tip]
"같은 서비스니까 같은 디자인"이 아니라, "같은 서비스지만 다른 경험"으로 접근하세요!
:::

웹과 앱은 같은 서비스여도 사용 환경이 달라요.
책상 vs 지하철, 마우스 vs 엄지손가락.
그래서 UX도 달라야 해요!
:::
