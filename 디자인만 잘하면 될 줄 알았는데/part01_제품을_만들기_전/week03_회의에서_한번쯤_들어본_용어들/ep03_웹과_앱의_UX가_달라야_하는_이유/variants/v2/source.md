# 웹과 앱의 UX가 달라야 하는 이유 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

웹과 앱은 입력 방식, 화면 크기, 사용 맥락이 다르므로 같은 기능이라도 UX 설계가 달라야 한다.

---

## 웹 vs 앱 핵심 차이점

| 구분 | 웹 (Desktop) | 앱 (Mobile) |
|------|-------------|-------------|
| 입력 장치 | 마우스 + 키보드 | 터치 (손가락) |
| Hover 상태 | 있음 | 없음 |
| 화면 크기 | 넓음 (1920px+) | 좁음 (375px~) |
| 사용 맥락 | 집중, 멀티태스킹 | 이동 중, 짧은 시간 |
| 뒤로가기 | 브라우저 버튼 | 스와이프/버튼 |

---

## 웹 디자인 체크리스트

### 인터랙션
- [ ] Hover 상태 디자인 되어 있는가?
- [ ] 클릭 가능한 요소에 cursor: pointer 적용했는가?
- [ ] 키보드 탭 이동 순서가 논리적인가?
- [ ] 단축키(Shortcut) 필요한 기능이 있는가?

### 레이아웃
- [ ] 넓은 화면을 효율적으로 활용하는가?
- [ ] 사이드바/메뉴 상시 노출이 적절한가?
- [ ] 멀티 컬럼 레이아웃이 필요한가?
- [ ] 최대 너비(max-width) 설정했는가?

### 네비게이션
- [ ] 브라우저 뒤로가기와 호환되는가?
- [ ] URL이 상태를 반영하는가? (딥링크)
- [ ] 탭/창 간 이동을 고려했는가?

---

## 앱 디자인 체크리스트

### 인터랙션
- [ ] 터치 영역이 최소 44x44pt 이상인가?
- [ ] Hover 대신 다른 피드백 방식이 있는가?
- [ ] 스와이프 제스처가 직관적인가?
- [ ] 한 손 조작이 가능한가?

### 레이아웃
- [ ] 좁은 화면에서 정보 우선순위가 명확한가?
- [ ] 스크롤 방향이 일관적인가?
- [ ] Safe Area(노치, 홈바)를 고려했는가?
- [ ] 가로 모드 대응이 필요한가?

### 네비게이션
- [ ] 하단 탭바 사용이 적절한가?
- [ ] 뒤로가기 동작이 명확한가?
- [ ] 화면 전환 애니메이션이 자연스러운가?

---

## 반응형 웹 체크리스트

### 브레이크포인트
- [ ] 모바일 (375px~): 단일 컬럼
- [ ] 태블릿 (768px~): 2컬럼 가능
- [ ] 데스크톱 (1024px~): 멀티 컬럼

### 모바일 웹 주의사항
- [ ] 터치 영역 충분한가?
- [ ] Hover 의존 UI 제거했는가?
- [ ] 가로 스크롤 발생하지 않는가?
- [ ] 폰트 크기 최소 16px 이상인가?

---

## 플랫폼별 컨벤션

### iOS 특징
- 하단 스와이프로 홈 이동
- 우측 스와이프로 뒤로가기
- 상단 좌측에 뒤로가기 버튼
- 하단 탭바 일반적

### Android 특징
- 시스템 뒤로가기 버튼 존재
- 상단 햄버거 메뉴 + 드로어 패턴
- Material Design 가이드라인
- FAB(Floating Action Button) 패턴

---

## 실무 적용 가이드

### 웹 → 앱 전환 시
1. Hover 상태 모두 제거/대체
2. 터치 영역 확대
3. 정보 계층 재정리 (좁은 화면)
4. 제스처 인터랙션 추가 검토

### 앱 → 웹 전환 시
1. Hover 상태 추가
2. 키보드 단축키 검토
3. 넓은 화면 레이아웃 활용
4. 마우스 정밀 클릭 고려
