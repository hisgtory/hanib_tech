# Week 3. 회의에서 한번쯤 들어본 용어들

[illust: ID="P1-W3-COVER" DESC="회의실에서 개발자들이 대화하는 동안, N이 물음표가 가득한 표정으로 듣고 있는 장면. 말풍선에 'API', '프론트', '백엔드' 등 용어들이 날아다님"]

회의실에 앉아 있으면 개발자들끼리 주고받는 대화가 마치 외국어처럼 들릴 때가 있어요.

"프론트에서 API 호출하면 백엔드에서 응답 주는 거죠?"
"이번 스프린트에서 리팩터링 좀 해야 할 것 같은데요."
"라이브러리 버전 업데이트 때문에 빌드가 깨졌어요."

N은 고개를 끄덕이면서도 속으로는 물음표가 가득했습니다.

"프론트? 백? 그게 뭔데..."
"API가 뭐길래 자꾸 얘기하지?"
"라이브러리랑 프레임워크는 뭐가 다른 거야?"

한입이 N의 표정을 보고 웃으며 말했습니다.

"괜찮아요. 처음엔 다 그래요. 사실 이런 용어들, 개발자들도 처음엔 헷갈려 했어요. 
하나씩 알아가다 보면 회의 내용이 들리기 시작할 거예요."

이번 주에는 개발자들이 자주 쓰는 용어들을 하나씩 살펴볼 거예요.
외계어처럼 느껴지던 단어들이 점점 익숙해질 겁니다.

그러면 회의 시간이 조금은 덜 외롭겠죠?
