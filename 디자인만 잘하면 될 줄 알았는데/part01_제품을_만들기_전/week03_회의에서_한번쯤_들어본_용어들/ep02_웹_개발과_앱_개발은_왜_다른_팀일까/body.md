# 웹 개발과 앱 개발은 왜 다른 팀일까

## 같은 프로그래밍인데 왜?

"웹 개발 3년 했으면 앱 개발도 금방 배우겠네요?" 이런 말 들어본 적 있나요? 사실 웹과 앱은 같은 프로그래밍이지만 **완전히 다른 세상**이에요.

## 플랫폼의 차이: 웹은 하나, 앱은 둘

웹 개발자가 가장 먼저 당황하는 부분이에요. 웹은 브라우저라는 '하나의 환경'에서 돌아가지만, 앱은 iOS와 Android라는 '두 개의 완전히 다른 환경'에서 돌아가거든요.

**웹 개발의 단순함:**
- HTML + CSS + JavaScript = 모든 브라우저에서 동작
- 브라우저 호환성 이슈는 있지만 기본 구조는 같음
- 배포는 서버에 올리기만 하면 끝

**앱 개발의 복잡함:**
- iOS: Swift/Objective-C + Xcode + Apple 가이드라인
- Android: Kotlin/Java + Android Studio + Google 가이드라인
- 각각 다른 UI 컴포넌트, 다른 디자인 가이드라인
- 배포는 심사(앱스토어 등록)를 거쳐야 함

## 디버깅의 차이: 웹은 F12, 앱은 미지수

웹 개발자들이 가장 사랑하는 도구가 뭘까요? 바로 브라우저 개발자 도구(F12)에요.

**웹 디버깅:**
- F12 누르고 콘솔 오류 확인
- CSS 수정하며 실시간 확인
- 리로드하면 바로 결과 확인 가능

**앱 디버깅:**
- Xcode 또는 Android Studio에서 에뮬레이터 실행
- 로그는 IDE 콘솔에서만 확인 가능
- UI 수정 후 리빌드하는데 30초~1분 소요

## 개발 속도의 차이

**웹 개발:**
- 코드 수정 → 저장 → 즉시 결과 확인
- Hot Reload로 실시간 변경 사항 반영

**앱 개발:**
- 코드 수정 → 컴파일 → 리빌드 → 에뮬레이터 실행 → 확인
- 각 단계마다 30초~2분 대기

## 배포의 차이: 웹은 업로드, 앱은 심사

**웹 배포:**
- 빌드 → 서버 업로드 → 끝
- 핫픽스나 버그수정도 즉시 반영

**앱 배포:**
- 빌드 → 앱스토어 심사 제출 → 대기(1주일) → 심사 통과 → 오픈
- iOS App Store와 Google Play Store 각각 별도 심사
- 버그수정도 심사를 거쳐야 하므로 즉시 반영 불가

:::callout[warning]
치명적인 버그가 발견되면 웹은 5분 안에 수정 가능하지만, 앱은 최소 1주일을 기다려야 해요.
:::

## 디자이너가 알아야 할 포인트

==웹과 앱은 같은 서비스라도 플랫폼별 제약이 다르다는 것을 이해해야 해요.==

**실무에서 자주 듣는 말:**
- "웹에서는 되는데 앱에서는 안 돼요" → 플랫폼별 기술 제약
- "iOS랑 Android 따로 만들어야 해요" → 네이티브 개발의 특성
- "앱 업데이트는 시간이 걸려요" → 심사 대기 시간

**디자인 핸드오프 시 고려할 점:**
- 웹: 반응형 디자인, 다양한 해상도
- iOS: Human Interface Guidelines 준수
- Android: Material Design 준수

## 그래도 크로스 플랫폼 도구가 있어요

React Native, Flutter 같은 도구로 한 번에 iOS/Android 앱을 만들 수 있어요. 하지만 완벽하진 않아서, 복잡한 기능은 결국 네이티브로 개발하게 되는 경우가 많아요.

## 정리

| 구분 | 웹 | 앱 |
|------|-----|-----|
| 플랫폼 | 브라우저 1개 | iOS/Android 2개 |
| 배포 | 서버 업로드 | 앱스토어 심사 |
| 수정 반영 | 즉시 | 1주일 이상 |
| 디버깅 | F12 개발자 도구 | IDE 콘솔 |
| 개발 속도 | 빠름 | 상대적으로 느림 |

:::callout[tip]
"같은 서비스인데 왜 앱은 더 오래 걸려요?"라는 질문의 답이 여기에 있어요!
:::
