# 아이디어를 제대로 구체화하는 방법

+------------------------------------------------------------+
| 한입: 지금 우리가 쓰는 앱들 중에 최초로 나온 제품이 몇     |
| 개나 될까요?                                               |
|                                                            |
| N: 음... 대부분이 그렇지 않을까요?                        |
|                                                            |
| 한입: 반도 안 돼요. 크롬, 구글, 인스타그램, 토스, 배민... |
| 다 후발주자였어요.                                         |
|                                                            |
| N: 어디서 이런 차이가 발생하는거에요?                      |
+------------------------------------------------------------+

## 제품을 혁신하는 3단계: PAS

선두주자의 입지를 나눠갖기 위해서는 선두주자가 해소해주지 못하는 그
이상의 무언가가 필요합니다.

그런데 그것은 생각보다 커다란 것이 아니라 '그러고보니 왜 꼭
이래야하지?', '이거 나만 불편한가?'와 같은 사소한 질문에서 성공하는
경우가 많습니다.

하지만 이런 문제 의식만으로는 부족합니다. 그 문제를 어떻게 풀 것인가
정의해야 해요.

제품을 성공적으로 혁신하기 위해서는 3단계가 필요합니다.

저는 개인적으로 이 개념을 PAS Framework 이라고 부릅니다.

1.  Problem (문제정의): 선두주자가 **아직 풀지 못했거나, 오히려 만들어낸
    부작용**을 사용자 관찰과 데이터로 **구체적으로 언어화**합니다.
    사용자의 관점에서 후기 형태로 정의되어야 합니다.

2.  Approach (해법 제시): 문제를 푸는 **핵심 원리/메커니즘**을 정합니다.
    **제품 형태가 아니라 작동 원리**를 명시합니다. "왜 이것이 그 문제를
    구조적으로 줄이는가?"에 답해야 합니다.

3.  Solution (솔루션 구현): 해법이 제품화된 모습입니다. 여러 형태가
    가능하며, **동일한 Approach 아래 다양한 솔루션**이 경쟁합니다.

Approach 와 Solution 이 어떻게 다를까요?\
데이팅 앱으로 예를 들어보겠습니다.

1.  문제 정의\
    1:1 데이팅은 너무 지루해. 매번 호구조사만 반복되다가 제대로
    이어지지도 않아. 그래서 메시지를 시작할 때의 설렘이 점점 떨어져.

2.  해법 제시\
    1:1 이 아니게 해보자. 1:1 이 아니면 사람들 간의 상호작용으로 인해 더
    다양한 변수들이 생길 수 있잖아.

3.  솔루션\
    a. 그럼 1:1 데이팅을 화상 통화로 진행하고 다른 사람들이 그걸
    시청하면서 코치해주는 기능을 넣는거 어때?\
    b. 관심사 기반으로 4명씩 매치되게 해서 단톡방을 만들면 어때?\
    c. 메타버스 상에서 맵을 돌아다니다가 테이블에 합석해서 자유롭게
    대화를 나누게 하는거 어때?

조금 이해가 되시나요?

해법은 정말 다양한 제품으로 탄생할 수 있습니다.\
문제 정의, 해법 제시가 잘못되어 실패하기도 하지만, 솔루션 구현이
잘못되어 시장에서 사라지는 제품도 정말 많습니다.

이러한 세가지 과정을 거쳐 실제 제품이 성공까지 이어지기 위해서 팀은
자유롭게 아이디어를 주고받으며 좋은 아이디어를 찾아야 합니다.

이렇게 발산하며 아이디어를 쌓아가는 과정을 **브레인스토밍**이라고
합니다.

## 문제를 해결할 최적의 방법을 찾는 방법, 브레인스토밍

브레인스토밍은 어떻게 하는게 좋을까요? 그냥 모여서 '우리 지금부터
아이디어를 나눠보자!'로 충분할까요?\
생각보다 브레인스토밍 회의는 실패로 끝나는 경우가 많습니다.

우선 브레인스토밍은 PAS 각 단계로 나눠서 진행해야 합니다.

문제 정의 단계에서는 아이디어보다는 각자가 겪는 답답함, 문제들을
공유하는 시간이 될겁니다. 이 단계의 브레인스토밍은 주관적으로 겪고 있는
진짜 문제들을 공유하는 것이기 때문에 오히려 난이도가 제일 낮아요.

브레인스토밍이 실패로 끝났다면 **십중팔구 AS 단계에서 문제가 생겼을
가능성이 높습니다.** 이때부터는 정말로 '아이디어'를 주고받는 과정이기
때문에 누군가의 아이디어에 '그건 너무 말이 안 되지 않아?', '그건 이래서
별로인 거 같은데?' 와 같은 즉각적인 피드백이 나오는 경우가 많고 팀은
점점 아이디어 제시에 소극적으로 변하기 때문입니다.

브레인스토밍 단계에서는 전혀 말이 되지 않아 보이는 아이디어도 적극적으로
주고 받아야 합니다.

지금은 당연하게 여겨지는 혁신들도, 처음엔 "말이 안 된다"는 평가를 받았던
것들이 많습니다.

**대표적으로 에어비앤비, 구글, 쿠팡, 토스 같은 서비스들이 그렇습니다.**

'누군지도 모르는 사람 집에서 자고 싶은 사람이 어디 있어?', '누군지도
모르는 사람에게 집을 내주고 싶은 사람이 어디있어?' 라며 투자를 거절 받기
일쑤였습니다.

하지만 직접 가서 사진을 찍어주고, 강력한 상호 후기 시스템을 구축하면서
에어비앤비는 글로벌로 성공한 기업이 됐습니다.

에릭 슈미츠가 구글의 CEO가 되기 전까지 웹사이트가 돈을 버는 것은
불가능한 것이었고 IT 창업은 미친 짓이었습니다.

하지만 에릭 슈미트는 구글 CEO로서 CPC 입찰 기반의 구글 애즈(AdWords)와
AdSense를 정착·확산시켜 검색 및 외부 웹사이트가 광고 수익을 낼 수 있는
모델을 확립했습니다. 그의 밑에서 온라인 광고와 세일즈를 총괄하던 셰릴
샌드버그는 이후 페이스북 COO로 합류해 정밀 타겟팅·모바일·동영상 광고 등
광고 제품군을 확대하며 페이스북의 광고 플랫폼을 폭발적으로 성장시켰죠.

쿠팡의 성공이 증명되기 전까지의 한국은 택배 수령까지 5일이 걸리는데에서
크게 불편함을 느끼지 못했습니다.

택배를 받는데에 5일씩이나 걸려야 돼? 그냥 다음날 바로올 수는 없어? 라는
아이디어가 단칼에 거절당했다면 소셜 커머스였던 쿠팡이 오늘 날의 모습을
갖추지는 못했겠죠.

토스 증권이 기존의 증권앱과 똑같았다면 승산이 있었을까요?

타성에 젖은 사고방식을 가지고 있었다면, '어차피 투자할 사람들 이미 다
하고 있는데 우린 아직까지 투자 안하고 있는 사람을 타겟하자. 어차피 차트
볼줄도 모르는데 복잡한 차트 필요해? 추세만 알아도 될거같아.' 와 같은
생각은 하지 못했겠죠.

그래서 브레인스토밍 회의에는 다음 규칙들이 필요합니다.

## 브레인스토밍의 4가지 기본 규칙

- 비판 금지: 아이디어에 대한 평가나 비판은 나중에

- 자유분방한 사고: 엉뚱하고 비현실적인 아이디어도 환영

- 양을 추구: 질보다 양, 많은 아이디어가 나올수록 좋음

- 결합과 개선: 다른 사람의 아이디어를 발전시키거나 결합

농담으로 던진 말 한마디가 사실은 엄청 효율적인 아이디어였던 사례는 정말
많습니다.

브레인스토밍의 목적은 문제를 해결할 가장 최적의 방법을 찾기 위해
구성원들이 모두 모여 아이디어를 발산하는 것입니다.

이 단계에서 아이디어를 평가하게 되면, **발산이 아닌 수렴의 단계로 넘어가
버리는 셈이죠.**

이 단계에서는 겁내지 말고, 자유롭게 아이디어를 발산해보세요.

좋은 아이디어는 언제나 '말도 안 되는 생각'에서 시작됩니다.
