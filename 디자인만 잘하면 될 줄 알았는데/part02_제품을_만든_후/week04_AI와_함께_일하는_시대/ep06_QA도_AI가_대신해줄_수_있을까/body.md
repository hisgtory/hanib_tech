# QA도 AI가 대신해줄 수 있을까

## "테스트할 시간이 없어요"

기능은 계속 나오는데, 테스트할 시간은 부족해요. AI가 QA도 대신해줄 수 있을까요? 어디까지 가능할까요?

## AI가 도와줄 수 있는 QA 영역

### 1. 테스트 케이스 생성

```
"이 로그인 기능에 대한 테스트 케이스 만들어줘"

→ AI가 다양한 시나리오 제안:
- 정상 로그인
- 잘못된 비밀번호
- 빈 필드
- SQL 인젝션 시도
- ...
```

:::callout[tip]
AI는 다양한 엣지 케이스를 생각해내는 데 도움이 돼요.
사람이 놓치기 쉬운 것들도 제안해줘요.
:::

### 2. 테스트 코드 작성

```
"이 함수에 대한 유닛 테스트 코드 작성해줘"

→ AI가 테스트 코드 생성
→ 개발자가 검토 후 적용
```

### 3. 버그 리포트 분석

```
"이 에러 로그 분석해줘. 원인이 뭘까?"

→ AI가 로그 분석해서 가능한 원인 제시
```

### 4. 리그레션 탐지

```
과거 버그 패턴 학습
→ 비슷한 패턴 감지
→ "이 부분 주의하세요" 알림
```

## 실제 도구들

### AI 테스팅 도구

```
Testim: AI 기반 테스트 자동화
Applitools: 시각적 테스트 (UI 변화 감지)
Mabl: 자가 치유 테스트 (셀렉터 변경 자동 대응)
Katalon: AI 기반 테스트 관리
```

### 일반 AI 활용

```
ChatGPT/Claude:
- 테스트 케이스 생성
- 테스트 코드 작성
- 버그 분석 도움
```

## AI QA의 장점

### 1. 빠른 테스트 케이스 생성

```
사람: 로그인 테스트 케이스 10개 생각 (1시간)
AI: 50개 생성 (5분)
→ 사람이 검토하고 선별 (30분)
```

### 2. 반복 테스트 자동화

```
매 배포마다 같은 테스트 반복
→ AI 도구가 자동 실행
→ 사람은 결과만 확인
```

### 3. 패턴 인식

```
"이런 코드 변경 시 여기서 버그 많이 났음"
→ AI가 패턴 학습해서 경고
```

## AI QA의 한계

### 1. 맥락 이해

```
"이게 버그인지 의도인지" 판단 어려움
비즈니스 로직 이해 부족
```

### 2. UX 판단

```
"이게 사용하기 불편한지" 판단 못 함
기능적으론 맞는데 UX가 나쁜 경우 못 잡음
```

### 3. 창의적 테스트

```
예상치 못한 사용 방식
실제 사용자처럼 행동하기
→ 사람의 직관 필요
```

:::callout[important]
AI는 반복적/정량적 테스트에 강해요.
하지만 "이게 좋은 경험인가?"는 사람이 판단해야 해요.
:::

## 현실적인 활용법

### 1. 테스트 케이스 브레인스토밍

```
기획서 주고:
"이 기능 테스트할 때 확인해야 할 것들 뽑아줘"

→ AI가 리스트 제안
→ 사람이 우선순위 정하고 보완
```

### 2. 자동화 테스트 작성 도움

```
"이 시나리오대로 테스트 코드 작성해줘"

→ 기본 코드 생성
→ 사람이 수정/보완
```

### 3. 버그 분석 보조

```
"이 에러가 왜 나는지 가능한 원인들 알려줘"

→ 가능성 리스트 제공
→ 사람이 하나씩 확인
```

### 4. 릴리즈 전 체크리스트

```
"이 기능 배포 전 확인해야 할 체크리스트 만들어줘"

→ 기본 체크리스트 생성
→ 우리 상황에 맞게 수정
```

## 미래 전망

### 지금

```
AI: 보조 도구
사람: 핵심 판단
```

### 가까운 미래

```
AI: 반복 테스트 대부분 자동화
사람: 창의적 테스트, UX 판단에 집중
```

### 변하지 않는 것

```
"이게 사용자에게 좋은 경험인가?"
→ 여전히 사람이 판단
```

## 정리

:::callout[important]
AI는 QA의 반복적인 부분을 도와줘요.
테스트 케이스 생성, 자동화 코드 작성에 활용하세요.
하지만 "이게 좋은 경험인가?"는 사람이 판단해야 해요.
:::

**핵심:**
- AI로 테스트 케이스 생성을 빠르게 할 수 있어요
- 반복적인 테스트 자동화에 도움이 돼요
- 맥락과 UX 판단은 사람이 해야 해요
- AI는 보조 도구, 핵심 판단은 사람
- 절약한 시간으로 더 중요한 테스트에 집중하세요
