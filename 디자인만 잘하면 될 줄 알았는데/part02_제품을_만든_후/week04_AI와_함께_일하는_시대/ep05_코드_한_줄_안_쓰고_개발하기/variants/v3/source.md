## 카드 2: 노코드 vs 로우코드

🔧 **뭐가 다를까?**

| 노코드 | 로우코드 |
|--------|----------|
| 👆 드래그앤드롭 | ⌨️ 일부 코딩 |
| 🧑‍🎨 비개발자용 | 🧑‍💻 개발자/비개발자 |
| 🚧 유연성 제한 | 🔓 확장 가능 |
| Notion, Airtable | Bubble, Retool |

---

## 카드 3: 언제 노코드?

✅ **노코드가 적합한 경우**

- 🚀 MVP 빠르게 만들 때
- 👨‍💻 개발 리소스 부족할 때
- 🧪 시장 검증이 목적일 때
- 🔧 내부 도구 만들 때
- ⚡ 단순한 기능일 때

---

## 카드 4: 용도별 추천 도구

🧰 **이럴 땐 이 도구**

| 용도 | 도구 |
|------|------|
| 웹사이트 | Webflow, Framer |
| 웹앱 | Bubble, Softr |
| 자동화 | Zapier, Make |
| DB | Airtable, Notion |
| 폼 | Typeform, Tally |

## 카드 5: 노코드의 한계

⚠️ **알아야 할 것**

- 📈 트래픽 많으면 한계
- 🔒 보안 직접 통제 어려움
- 💰 성장하면 비용 증가
- 🔗 벤더 종속 위험
- 🛠️ 복잡한 로직 구현 어려움
