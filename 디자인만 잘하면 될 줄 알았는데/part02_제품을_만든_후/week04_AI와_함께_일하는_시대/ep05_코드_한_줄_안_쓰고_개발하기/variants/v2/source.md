# 코드 한 줄 안 쓰고 개발하기 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

노코드/로우코드 도구를 활용하면 개발자 없이도 MVP를 빠르게 만들 수 있다. 단, 도구의 한계를 알고 적합한 상황에서 사용해야 한다.

---

## 노코드 vs 로우코드 비교

| 구분 | 노코드 | 로우코드 |
|------|--------|----------|
| 코딩 필요 | 전혀 없음 | 일부 필요 |
| 대상 사용자 | 비개발자 | 개발자/비개발자 |
| 유연성 | 제한적 | 중간 |
| 복잡한 기능 | 어려움 | 가능 |
| 예시 | Notion, Airtable | Bubble, Retool |

---

## 노코드 도구 분류

### 용도별 도구 정리

| 용도 | 도구 예시 | 특징 |
|------|----------|------|
| 웹사이트 | Webflow, Wix, Framer | 랜딩 페이지, 포트폴리오 |
| 웹앱 | Bubble, Softr, Glide | 데이터베이스 기반 앱 |
| 자동화 | Zapier, Make, n8n | 앱 간 연결, 워크플로우 |
| 데이터베이스 | Airtable, Notion | 정보 관리, 협업 |
| 폼/설문 | Typeform, Tally | 데이터 수집 |
| 이커머스 | Shopify, Gumroad | 상품 판매 |

## 노코드 도입 체크리스트

### 적합한 상황
- [ ] MVP/프로토타입 빠르게 만들어야 함
- [ ] 개발 리소스가 부족함
- [ ] 단순한 기능 구현
- [ ] 시장 검증이 목적
- [ ] 내부 도구/어드민

### 부적합한 상황
- [ ] 복잡한 비즈니스 로직
- [ ] 높은 트래픽 대응 필요
- [ ] 고도화된 보안 요구
- [ ] 플랫폼 독점 기능 필요
- [ ] 장기적 확장성 중요

---

## 도구 선택 체크리스트

### 선택 기준
- [ ] 만들려는 것과 도구 특성 매칭
- [ ] 예상 사용자 수 감당 가능
- [ ] 필요한 기능 지원 여부
- [ ] 요금제 검토 (무료/유료)
- [ ] 데이터 내보내기(export) 가능 여부

### 확인 사항
- [ ] 한국어 지원
- [ ] 결제 연동 (국내 PG)
- [ ] 커스텀 도메인
- [ ] API 연동 가능
- [ ] 커뮤니티/지원 활성도

## 노코드 프로젝트 진행 체크리스트

### 1. 기획 단계
- [ ] 핵심 기능 정의 (3개 이하)

- [ ] 적합한 도구 선정

### 2. 개발 단계
- [ ] 데이터베이스 설계
- [ ] 화면 구성
- [ ] 기능 연결
- [ ] 자동화 설정

### 3. 테스트 단계
- [ ] 핵심 기능 동작 확인
- [ ] 다양한 기기/브라우저 테스트
- [ ] 에러 케이스 확인
- [ ] 성능 확인

### 4. 배포 단계
- [ ] 커스텀 도메인 연결
- [ ] SSL 설정

- [ ] 백업 설정

## 노코드 한계와 대응

### 알아야 할 한계

| 한계 | 대응 방법 |
|------|----------|
| 성능 제한 | 트래픽 많으면 코드 전환 검토 |
| 기능 제한 | API, 외부 서비스 연동 |
| 벤더 종속 | 데이터 정기 백업 |
| 비용 증가 | 성장 시 코드 전환 검토 |
| 보안 우려 | 민감 데이터 별도 관리 |

### 언제 코드로 전환?
- [ ] 월 비용이 개발자 인건비 초과
- [ ] 성능 한계에 도달
- [ ] 필요한 기능 구현 불가
- [ ] 보안/규정 준수 요구
- [ ] 대규모 확장 필요

## 실무 팁

:::callout[tip]
**"노코드는 시작, 코드는 확장"**

좋은 전략:
1. 노코드로 MVP 빠르게 출시
2. 시장 반응 확인
3. 검증되면 코드로 재구축
4. 확장성 확보

→ 속도와 확장성 모두 잡기
:::

:::callout[warning]
**노코드의 숨겨진 비용**

주의할 점:
- 월 구독료 누적
- 사용량 기반 과금
- 기능 추가 시 요금제 업그레이드
- 다른 도구로 이전 시 재작업

→ 장기 비용 계산하고 시작하세요
:::
