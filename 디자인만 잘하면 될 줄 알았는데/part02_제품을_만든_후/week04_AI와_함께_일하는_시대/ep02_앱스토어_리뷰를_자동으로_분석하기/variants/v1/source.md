# 앱스토어 리뷰를 자동으로 분석하기

## 스토리텔링 버전

:::conversation
**N**: 한입님, 앱스토어 리뷰가 수천 개인데 다 읽을 수가 없어요. 사용자들이 뭘 원하는지 알고 싶은데...

**한입**: AI한테 분석 시키면 몇 분이면 끝나요!

**N**: 정말요? 어떻게요?
:::

### 기존 방식 vs AI 방식

**기존 방식:**
```
1. 리뷰 하나하나 읽기
2. 엑셀에 분류
3. 수작업 집계
→ 며칠 걸림
```

**AI 방식:**
```
1. 리뷰 데이터 추출
2. AI에게 분석 요청
3. 자동 분류 + 인사이트
→ 몇 분이면 끝
```

### 실전 활용법

:::conversation
**한입**: 실제로 해볼게요. 리뷰를 복사해서 AI에게 이렇게 요청해요.

```
"아래는 우리 앱의 최근 리뷰 100개야.
다음을 분석해줘:

1. 주요 불만 사항 TOP 5
2. 사용자들이 좋아하는 점 TOP 5
3. 자주 요청되는 기능
4. 전체 감성 분석 (긍정/부정/중립 비율)
5. 개선 우선순위 제안"
```

**N**: 이렇게 하면 다 정리해주나요?

**한입**: 네! 한눈에 볼 수 있게 요약해줘요.
:::

### 분석할 수 있는 것들

**1. 감성 분석**
```
긍정: 60%
부정: 30%  
중립: 10%

→ 시간에 따른 변화도 추적 가능
  "지난달보다 부정 비율이 10% 늘었네?"
```

**2. 주제별 분류**
```
- 버그 관련: 25건
- 속도 문제: 18건
- 새 기능 요청: 15건
- 디자인 피드백: 12건
- 결제 문제: 8건
```

:::callout[tip]
어떤 이슈가 가장 많이 언급되는지 한눈에 파악할 수 있어요!
:::

**3. 기능별 분석**
```
"로그인 기능에 대한 리뷰만 모아서 분석해줘"
"결제 관련 불만 사항만 정리해줘"
```

**4. 경쟁사 비교**
```
"우리 앱 리뷰와 경쟁사 앱 리뷰를 비교해줘.
우리가 잘하는 것과 못하는 것은?"
```

### 실제 활용 사례

:::conversation
**한입**: 이렇게 활용할 수 있어요.

**제품 로드맵에 반영:**
- 리뷰 분석 결과: "검색 기능이 불편하다" 언급 50회
- → 검색 개선을 다음 스프린트에 반영

**버전별 반응 비교:**
- "2.0 버전 출시 후 리뷰와 1.9 버전 리뷰 비교해줘"
- → 새 기능에 대한 반응 파악

**긴급 이슈 감지:**
- "최근 1주일 리뷰 중 '앱이 안 된다' 관련 찾아줘"
- → 긴급 버그 대응
:::

### 주의할 점

:::callout[warning]
AI 분석만 믿지 마세요!
:::

**1. 샘플 편향**
- 리뷰 쓰는 사람 ≠ 모든 사용자
- 극단적으로 좋거나 싫을 때 리뷰를 써요
- 중간층의 의견은 반영 안 될 수 있어요

**2. 맥락 손실**
- AI가 요약하면서 뉘앙스가 사라질 수 있어요
- 중요한 리뷰는 직접 읽어보세요

**3. 정량 + 정성**
- 숫자만 보지 말고
- 실제 리뷰 내용도 같이 보세요
