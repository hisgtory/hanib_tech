# 앱스토어 리뷰를 자동으로 분석하기

## "리뷰가 너무 많아서 다 못 읽어요"

앱스토어에 리뷰가 쌓여가요. 사용자들이 뭘 좋아하고 뭘 싫어하는지 알고 싶은데, 수천 개의 리뷰를 하나하나 읽기는 힘들어요.

## AI로 리뷰 분석하기

### 기존 방식

```
1. 리뷰 하나하나 읽기
2. 엑셀에 분류
3. 수작업 집계

→ 시간이 너무 오래 걸림
```

### AI 활용 방식

```
1. 리뷰 데이터 추출
2. AI에게 분석 요청
3. 자동 분류 + 인사이트

→ 몇 분이면 끝
```

## 실전 활용법

### 1. 리뷰 데이터 수집

```
앱스토어/플레이스토어에서 리뷰 내보내기
또는 리뷰 수집 도구 사용
```

### 2. AI에게 분석 요청

```
프롬프트 예시:

"아래는 우리 앱의 최근 리뷰 100개야.
다음을 분석해줘:

1. 주요 불만 사항 TOP 5
2. 사용자들이 좋아하는 점 TOP 5
3. 자주 요청되는 기능
4. 전반적인 감성 분석 (긍정/부정/중립 비율)
5. 개선 우선순위 제안"
```

### 3. 결과 정리

```
AI가 정리해준 인사이트를
팀에 공유하거나 로드맵에 반영
```

## 분석할 수 있는 것들

### 감성 분석

```
긍정: 60%
부정: 30%
중립: 10%

→ 시간에 따른 변화 추적 가능
```

### 주제별 분류

```
- 버그 관련: 25건
- 속도 문제: 18건
- 새 기능 요청: 15건
- 디자인 피드백: 12건
- 결제 문제: 8건
```

:::callout[tip]
어떤 이슈가 가장 많이 언급되는지 한눈에 파악할 수 있어요.
:::

### 기능별 평가

```
"로그인 기능에 대한 리뷰만 모아서 분석해줘"
"결제 관련 불만 사항만 정리해줘"
```

### 경쟁사 비교

```
"우리 앱 리뷰와 경쟁사 앱 리뷰를 비교해줘.
우리가 잘하는 것과 못하는 것은?"
```

## 활용 사례

### 제품 로드맵 반영

```
리뷰 분석 결과:
"검색 기능이 불편하다" 언급 50회

→ 검색 개선을 다음 스프린트에 반영
```

### 버전별 반응 비교

```
"2.0 버전 출시 후 리뷰와 
1.9 버전 리뷰를 비교해줘.
새 기능에 대한 반응은?"
```

### 긴급 이슈 감지

```
"최근 1주일 리뷰 중 
'앱이 안 된다', '크래시' 관련 리뷰 찾아줘"

→ 긴급 버그 대응
```

## 주의할 점

### 1. 샘플 편향

```
리뷰 쓰는 사람 ≠ 모든 사용자
극단적으로 좋거나 싫을 때 리뷰를 써요.
중간층의 의견은 반영 안 될 수 있어요.
```

### 2. 맥락 손실

```
AI가 요약하면서 뉘앙스가 사라질 수 있어요.
중요한 리뷰는 직접 읽어보세요.
```

### 3. 정량 + 정성

```
숫자만 보지 말고,
실제 리뷰 내용도 같이 보세요.
```

:::callout[important]
AI 분석은 방향을 잡는 데 좋지만,
깊은 이해는 직접 읽어야 해요.
:::


## 정리

수천 개의 리뷰도 AI로 몇 분 만에 분석할 수 있어요.
트렌드 파악에는 AI, 깊은 이해는 직접 읽기.
:::
