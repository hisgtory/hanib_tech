# AI가 PRD 초안을 써준다면

## "백지가 제일 무서워요"

새 기능을 기획해야 하는데, 빈 문서를 보면 막막해요. 뭘 먼저 써야 할지, 어떤 구조로 써야 할지. 처음 시작이 가장 어렵죠.

## AI가 도와줄 수 있는 것

### 초안 작성

```
빈 문서 → AI 초안 → 수정/보완 → 완성

0에서 시작하는 것보다
60점짜리를 80점으로 고치는 게 쉬워요.
```

:::callout[tip]
AI는 훌륭한 초안 작성 도우미예요.
빈 페이지의 공포를 없애줘요.
:::

### 구조 잡기

```
"이런 기능의 PRD를 쓰려고 해.
어떤 섹션들이 필요할까?"

→ 목차와 구조 제안받기
```

### 빠뜨린 부분 찾기

```
"이 PRD에 빠진 게 있을까?"

→ 놓친 고려사항 발견
```

## 실전 활용법

### 1단계: 맥락 제공

```
"우리는 음식 배달 앱이야.
타깃 사용자는 20~30대 직장인이고,
경쟁 앱은 배민, 쿠팡이츠가 있어.

이번에 '단체 주문' 기능을 추가하려고 해.
회사에서 여러 사람이 한 번에 주문하는 거야."
```

### 2단계: PRD 초안 요청

```
"이 기능에 대한 PRD 초안을 써줘.
포함할 내용:
- 문제 정의
- 목표 (KPI 포함)
- 사용자 스토리
- 기능 요구사항
- 예외 케이스
- 출시 일정 제안"
```

### 3단계: 반복 수정

```
"예외 케이스 부분 더 자세히 써줘"
"KPI를 더 구체적으로"
"개발팀이 보기 좋게 기술 요구사항 추가해줘"
```

### 4단계: 팀 리뷰

```
AI 초안을 팀에 공유하고
피드백을 받아서 수정
```

## AI가 잘하는 것

### 표준적인 구조

```
PRD에 들어가야 할 일반적인 섹션들을 알아요.
빠뜨리기 쉬운 부분도 챙겨줘요.
```

### 다양한 관점

```
"사용자 관점에서 이 기능의 문제점은?"
"개발자 관점에서 이 기능의 리스크는?"
"비즈니스 관점에서 이 기능의 가치는?"
```

### 빠른 초안

```
30분 걸릴 초안을 5분에 받을 수 있어요.
절약한 시간으로 더 중요한 논의에 집중.
```

## AI가 못하는 것

### 우리 회사만의 맥락

```
❌ 우리 팀의 역학 관계
❌ 과거 프로젝트의 실패 경험
❌ 현재 개발팀의 리소스 상황
```

### 진짜 사용자 인사이트

```
❌ 실제 인터뷰 내용
❌ 데이터에서 나온 패턴
❌ 현장에서 관찰한 문제
```

### 최종 판단

```
❌ 이 기능을 해야 할까?
❌ 우선순위는 어떻게?
❌ 이 접근이 맞을까?
```

:::callout[important]
AI는 도구이고, 결정은 사람이 해요.
초안을 믿지 말고, 비판적으로 검토하세요.
:::

## 효과적인 프롬프트 예시

### 문제 정의 도움

```
"다음 사용자 피드백들을 보고
핵심 문제가 뭔지 정리해줘:
[피드백 목록]"
```

### 사용자 스토리 생성

```
"이 기능에 대한 사용자 스토리 5개 써줘.
형식: 'As a [사용자], I want [기능] so that [가치]'"
```

### 엣지 케이스 탐색

```
"이 기능에서 발생할 수 있는
예외 상황과 엣지 케이스 10개 뽑아줘"
```

### 경쟁 분석

```
"[경쟁사]의 비슷한 기능을 분석해줘.
우리가 차별화할 수 있는 포인트는?"
```

## 워크플로우 제안

```
1. 아이디어 정리 (내 머릿속 생각 정리)
2. AI에게 초안 요청
3. 초안 검토 + 우리 맥락 추가
4. AI에게 개선 요청 (반복)
5. 팀 리뷰
6. 최종 수정
7. 이해관계자 공유
```

## 정리

:::callout[important]
AI는 빈 페이지의 공포를 없애줘요.
초안은 AI가, 판단과 맥락은 사람이.
:::

**핵심:**
- AI로 PRD 초안을 빠르게 만들 수 있어요
- 맥락을 충분히 제공하면 더 좋은 결과
- 우리 회사만의 맥락은 직접 추가해야 해요
- 최종 판단은 사람의 몫
- 절약한 시간으로 더 깊은 논의에 집중하세요
