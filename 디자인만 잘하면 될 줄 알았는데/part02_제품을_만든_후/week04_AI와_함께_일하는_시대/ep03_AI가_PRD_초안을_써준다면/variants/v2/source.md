# AI가 PRD 초안을 써준다면 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

AI는 PRD 초안 작성, 구조 제안, 누락 항목 점검에 효과적이다. 단, 최종 판단과 맥락 반영은 PM의 몫이다. AI는 보조 도구로 활용해야 한다.

---

## AI PRD 활용 단계

| 단계 | AI 역할 | PM 역할 |
|------|---------|---------|
| 아이디어 정리 | 브레인스토밍, 구조화 | 방향 결정 |
| 초안 작성 | 템플릿 기반 초안 | 맥락 추가 |
| 검토 | 누락 항목 체크 | 우선순위 판단 |
| 수정 | 문장 다듬기 | 이해관계자 조율 |
| 완성 | 포맷 정리 | 최종 승인 |

## AI PRD 초안 요청 체크리스트

### 기본 정보 제공
- [ ] 제품/서비스 소개
- [ ] 타겟 사용자
- [ ] 해결하려는 문제
- [ ] 경쟁사/기존 솔루션
- [ ] 제약 조건

### 원하는 산출물 명시
- [ ] 문서 형식 (1 pager, 상세 PRD)
- [ ] 포함할 섹션
- [ ] 분량 가이드
- [ ] 톤앤매너

## PRD 프롬프트 템플릿

### 기본 초안 요청
```
당신은 경험 많은 PM입니다.

[제품 개요]
- 제품명: [제품명]
- 타겟: [타겟 사용자]
- 문제: [해결하려는 문제]

위 정보를 바탕으로 PRD 초안을 작성해주세요.

포함할 섹션:
1. 배경 및 목적
2. 타겟 사용자
3. 문제 정의
4. 핵심 기능
5. 성공 지표
6. 일정
7. 리스크
```

### 기능 상세화 요청
```
[기능명]에 대해 상세 스펙을 작성해주세요.

포함할 내용:
- 기능 설명
- 사용자 시나리오
- 입력/출력
- 예외 케이스
- UI 요구사항
- 기술 요구사항
```

### 사용자 스토리 생성
```
[기능명]에 대한 사용자 스토리를 작성해주세요.

형식: "As a [유저 타입], I want to [행동], so that [목적]"

다양한 유저 타입과 시나리오를 포함해주세요.
```

## AI PRD 검토 체크리스트

### 내용 검토
- [ ] 목적이 명확한가?
- [ ] 타겟 사용자가 구체적인가?
- [ ] 문제 정의가 적절한가?
- [ ] 기능 설명이 충분한가?
- [ ] 성공 지표가 측정 가능한가?

### 맥락 추가 필요
- [ ] 팀 상황 반영
- [ ] 기술적 제약 추가
- [ ] 비즈니스 맥락 보완
- [ ] 이해관계자 고려
- [ ] 우선순위 조정

### 톤 조정
- [ ] 팀 문화에 맞는 표현
- [ ] 전문 용어 적절성
- [ ] 명확성 vs 상세함 밸런스

## PRD 섹션별 AI 활용

| 섹션 | AI 활용 방법 |
|------|-------------|
| 배경/목적 | 시장 조사 요약, 문제 정의 |
| 타겟 사용자 | 페르소나 생성, 시나리오 |
| 기능 요구사항 | 스토리 생성, 스펙 상세화 |
| 성공 지표 | KPI 제안, 벤치마크 |
| 일정 | 작업 분해, 의존성 정리 |
| 리스크 | 리스크 항목 도출, 대응책 |

## 실무 팁

:::callout[tip]
**"AI 초안 → PM 수정"이 최적**

왜?
- 빈 페이지 시작보다 수정이 빠름
- 놓친 항목 발견 가능
- 다양한 관점 확보

→ AI는 시작점, PM은 완성자
:::

:::callout[warning]
**AI가 모르는 것들**

- 팀의 역량과 상황
- 정치적 맥락
- 숨겨진 제약 조건
- 이해관계자 선호
- 과거 히스토리

→ 맥락은 PM만 알아요
:::
