# 디자인 작업이 30분만에 끝난다고요 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

AI 디자인 도구는 아이디어 시각화, 에셋 생성, 배리에이션 작업에 효과적이다. 단, UX 설계와 맥락 판단은 여전히 사람의 영역이다.

---

## AI 디자인 도구 분류

| 용도 | 도구 예시 | 특징 |
|------|----------|------|
| 이미지 생성 | Midjourney, DALL-E, Stable Diffusion | 텍스트→이미지 |
| UI 생성 | Galileo AI, Uizard | 프롬프트→UI |
| 에셋 생성 | IconifyAI, RemoveBG | 아이콘, 배경 제거 |
| 이미지 편집 | Adobe Firefly, Canva AI | 편집, 확장 |
| 디자인 보조 | Figma AI, Framer AI | 자동 레이아웃 |

## 이미지 생성 프롬프트 체크리스트

### 기본 구조
- [ ] 주제: 무엇을 그릴지
- [ ] 스타일: 어떤 느낌으로
- [ ] 구도: 어떤 배치로
- [ ] 분위기: 색감, 조명
- [ ] 제외 항목: 뭘 넣지 말지

### 프롬프트 예시
```
미니멀한 핀테크 앱 온보딩 화면,
플랫 일러스트 스타일,
파란색과 흰색 컬러 팔레트,
여백이 많은 깔끔한 레이아웃,
사람 일러스트 포함
```

### 품질 향상 키워드
- [ ] 고품질: "high quality", "detailed"
- [ ] 스타일: "flat design", "3D render", "minimalist"
- [ ] 조명: "soft lighting", "studio light"
- [ ] 해상도: "4K", "high resolution"

## 실무 팁

:::callout[tip]
**"첫 스케치는 AI, 완성은 사람"**

효과적인 워크플로우:
1. AI로 다양한 방향 탐색 (10개+)
2. 괜찮은 방향 3개 선정
3. Figma에서 직접 디자인
4. 피드백 반영해 완성

→ 탐색 시간 단축 + 품질 유지
:::

:::callout[warning]
**AI 디자인의 함정**

주의할 점:
- 예쁘지만 쓸모없는 디자인
- 브랜드와 맞지 않는 스타일
- 실제 구현 불가능한 요소
- 접근성 무시

→ 항상 목적과 맥락을 기억하세요
:::
