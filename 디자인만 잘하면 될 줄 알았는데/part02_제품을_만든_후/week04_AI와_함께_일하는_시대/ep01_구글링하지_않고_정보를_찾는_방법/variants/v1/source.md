### 구글링 vs AI 대화

:::conversation
**한입**: 비교해볼게요.

**구글링:**
1. "react usestate 사용법" 검색
2. 여러 링크 중 하나 클릭
3. 읽다가 내 상황과 다름

**AI:**
1. "React에서 버튼 클릭하면 숫자가 1씩 올라가게 하고 싶은데 useState 어떻게 써?"

**N**: 아, 자연어로 물어볼 수 있으니까 구체적으로 물어볼 수 있네요!
:::

### AI에게 잘 물어보는 법

**1. 맥락을 알려주세요**

```
❌ "프론트엔드 좋은 라이브러리 추천해줘"

✅ "React로 대시보드 만들고 있는데, 
   차트 그리는 라이브러리 추천해줘.
   데이터가 실시간으로 업데이트돼야 해."
```

**2. 역할을 부여하세요**

```
"너는 5년차 UX 디자이너야.
 초보 기획자인 내가 만든 와이어프레임을 봐줘.
 어떤 점이 부족할까?"
```

**3. 원하는 형식을 말하세요**

```
"표로 정리해줘"
"3줄로 요약해줘"
"초등학생도 이해할 수 있게 설명해줘"
```

**4. 후속 질문을 하세요**

:::conversation

**N**: 마치 전문가한테 계속 물어보는 것처럼요?

### AI가 도와줄 수 있는 것들

| 용도 | 예시 |
|------|------|
| 개념 이해 | "기술 부채가 뭐야? 비개발자가 이해하기 쉽게" |
| 번역/요약 | "이 영어 기술 문서 한국어로 번역해줘" |
| 문서 작성 | "이 기능에 대한 PRD 초안 작성해줘" |
| 아이디어 | "우리 앱 온보딩 개선 아이디어 10개 내줘" |
| 코드 이해 | "이 에러 메시지 무슨 뜻이야?" |

### 주의할 점

:::callout[warning]
AI는 자신 있게 거짓말할 수 있어요! (할루시네이션)
중요한 건 꼭 다른 소스로 확인하세요.
:::

**1. 팩트 체크 필요**
- AI가 그럴듯하게 말해도 틀릴 수 있어요
- 특히 최신 정보나 숫자는 확인 필요

**2. 회사 기밀 주의**
```
❌ 고객 데이터, 소스코드, 내부 문서 붙여넣기
✅ 민감 정보 제거 후 질문
```

**3. 생각의 대체는 안 돼요**
```
AI: 정보 수집, 초안 작성, 아이디어 확장
사람: 판단, 결정, 책임
```
