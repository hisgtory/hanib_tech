# 구글링하지 않고 정보를 찾는 방법

## "검색해도 답이 안 나와요"

무언가를 알고 싶을 때 구글에 검색하죠. 근데 요즘은 검색해도 광고만 나오거나, 원하는 답을 찾기 어려울 때가 많아요. 더 좋은 방법이 있을까요?

## AI에게 직접 물어보기

### ChatGPT, Claude 등 AI 챗봇

```
구글링: 키워드 입력 → 링크 클릭 → 글 읽기 → 다시 검색...
AI: 질문 입력 → 바로 답변
```

:::callout[tip]
AI는 "무엇을 알고 싶은지"를 자연어로 물어볼 수 있어요.
:::

### 예시 비교

```
구글링: "react usestate 사용법"
→ 수많은 글 중 하나 클릭
→ 읽다가 내 상황과 다름
→ 다시 검색

AI: "React에서 버튼 클릭하면 숫자가 1씩 올라가게 
    하고 싶은데 useState 어떻게 써?"
→ 내 상황에 맞는 코드 예시 바로 제공
```

## AI에게 잘 물어보는 법

### 1. 맥락을 알려주세요

```
❌ "프론트엔드 좋은 라이브러리 추천해줘"
✅ "React로 대시보드 만들고 있는데, 
   차트 그리는 라이브러리 추천해줘.
   데이터가 실시간으로 업데이트돼야 해."
```

### 2. 역할을 부여하세요

```
"너는 5년차 UX 디자이너야.
 초보 기획자인 내가 만든 와이어프레임을 봐줘.
 어떤 점이 부족할까?"
```

### 3. 원하는 형식을 말하세요

```
"표로 정리해줘"
"3줄로 요약해줘"
"초등학생도 이해할 수 있게 설명해줘"
```

### 4. 후속 질문을 하세요

```
1차: "API가 뭐야?"
2차: "그럼 REST API랑 GraphQL은 뭐가 달라?"
3차: "우리 프로젝트에는 뭐가 더 맞을까? 
     사용자 100만 정도 예상하고 있어."
```

==대화를 이어가면 점점 더 정확한 답을 얻어요.==

## AI를 활용할 수 있는 작업들

### 개념 이해

```
"기술 부채가 뭐야? 비개발자가 이해하기 쉽게 설명해줘"
"마이크로서비스 아키텍처 장단점 표로 정리해줘"
```

### 번역/요약

```
"이 영어 기술 문서 한국어로 번역해줘"
"이 논문 핵심만 5줄로 요약해줘"
```

### 문서 작성

```
"이 기능에 대한 PRD 초안 작성해줘"
"사용자 인터뷰 질문지 만들어줘"
```

### 아이디어 브레인스토밍

```
"우리 앱 온보딩 개선 아이디어 10개 내줘"
"이 문제를 해결할 수 있는 다른 방법 없을까?"
```

### 코드 관련

```
"이 에러 메시지 무슨 뜻이야?"
"이 코드 뭐 하는 건지 설명해줘"
```

## AI 사용 시 주의할 점

### 1. 팩트 체크 필요

```
AI가 그럴듯하게 말해도 틀릴 수 있어요.
특히 최신 정보나 숫자는 확인 필요.
```

:::callout[warning]
AI는 자신 있게 거짓말할 수 있어요 (할루시네이션).
중요한 건 꼭 다른 소스로 확인하세요.
:::

### 2. 회사 기밀 주의

```
❌ 고객 데이터, 소스코드, 내부 문서 붙여넣기
✅ 민감 정보 제거 후 질문
```

### 3. 저작권 문제

```
AI가 생성한 텍스트/이미지의 저작권은 
아직 법적으로 불명확해요.
상업적 사용 시 주의.
```

### 4. 생각의 대체는 안 돼요

```
AI: 정보 수집, 초안 작성, 아이디어 확장
사람: 판단, 결정, 책임
```

## 정리

:::callout[important]
AI는 새로운 검색 방법이에요.
맥락을 주고 대화하듯 물어보세요.
단, 중요한 정보는 꼭 확인하세요.
:::

**핵심:**
- AI에게 자연어로 질문하면 바로 답변을 받아요
- 맥락, 역할, 형식을 명확히 하면 더 좋은 답을 얻어요
- 후속 질문으로 대화를 이어가세요
- AI의 답변은 팩트 체크가 필요해요
- 기밀 정보는 넣지 마세요
