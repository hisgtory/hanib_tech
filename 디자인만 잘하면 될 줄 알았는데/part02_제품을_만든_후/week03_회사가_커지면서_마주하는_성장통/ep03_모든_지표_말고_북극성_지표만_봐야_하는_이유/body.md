# 모든 지표 말고 북극성 지표만 봐야 하는 이유

## "대시보드에 숫자가 100개예요"

회사가 커지면서 데이터 분석 도구가 생겼어요. DAU, MAU, 전환율, 체류시간, 클릭수... 숫자가 너무 많아서 뭘 봐야 할지 모르겠어요.

## 지표가 많으면 생기는 문제

### 모든 지표를 다 올리려고 해요

```
이번 달 목표:
- DAU 10% 올리기
- 결제 전환율 5% 올리기
- 체류시간 20% 늘리기
- 이탈률 10% 낮추기
- ...

결과: 아무것도 안 오름
```

:::callout[warning]
모든 걸 잡으려면 아무것도 못 잡아요.
:::

### 팀마다 다른 지표를 봐요

```
마케팅팀: "가입자 수가 중요해!"
개발팀: "서버 응답속도가 중요해!"
디자인팀: "완료율이 중요해!"
영업팀: "매출이 중요해!"

→ 다들 자기 지표만 최적화
→ 전체적으론 좋아지지 않음
```

## 북극성 지표란?

### 모든 팀이 같이 보는 하나의 지표

```
북극성 = 방향을 알려주는 별
북극성 지표 = 회사의 방향을 알려주는 지표
```

==모든 팀이 이 하나의 숫자를 올리기 위해 일해요.==

### 북극성 지표의 조건

```
1. 고객 가치를 반영해야 함
   ❌ 페이지뷰 (사용자에게 가치 없음)
   ✅ 콘텐츠 완독률 (가치를 느꼈다는 신호)

2. 비즈니스 성장과 연결되어야 함
   좋은 지표가 오르면 → 매출도 오름

3. 모든 팀이 영향을 줄 수 있어야 함
   개발, 디자인, 마케팅 모두 기여 가능
```

### 서비스별 북극성 지표 예시

```
넷플릭스: 월간 시청 시간
페이스북: 일일 활성 사용자 (DAU)
슬랙: 주당 메시지 전송 수
에어비앤비: 예약된 숙박일 수
스포티파이: 청취 시간
```

:::callout[tip]
"사용자가 우리 서비스에서 가치를 느끼는 순간"을 숫자로 표현한 거예요.
:::

## 북극성 지표를 정하는 방법

### 1. 핵심 가치 정의하기

```
우리 서비스의 핵심 가치는 뭘까?

배달 앱: 빠르고 편하게 음식 받기
뉴스 앱: 좋은 정보 얻기
피트니스 앱: 운동 습관 만들기
```

### 2. 가치를 측정할 수 있는 지표 찾기

```
배달 앱:
- 주문 완료 수? (가치 반영)
- 배달 시간 만족도? (가치 반영)
- 앱 다운로드 수? (가치와 거리 있음)

→ "월간 주문 완료 수"가 북극성 지표 후보
```

### 3. 모든 팀이 동의하기

```
북극성 지표: 월간 주문 완료 수

마케팅: 더 많은 사람에게 알려서 주문↑
개발: 주문 과정 빠르게 해서 이탈↓
디자인: 주문 UX 개선해서 전환율↑
운영: 배달 품질 높여서 재주문↑
```

## 다른 지표는 안 봐도 돼요?

### 서브 지표(Input Metrics)로 관리

```
북극성: 월간 주문 완료 수

서브 지표:
- 신규 가입자 수 (마케팅팀 담당)
- 장바구니 → 결제 전환율 (디자인팀 담당)
- 앱 로딩 속도 (개발팀 담당)
- 재주문율 (운영팀 담당)
```

:::callout[important]
서브 지표들이 좋아지면 → 북극성 지표가 오른다
북극성 지표가 오르면 → 비즈니스가 성장한다
:::

### 지표의 계층 구조

```
        [북극성 지표]
             ↑
    서브 지표1  서브 지표2  서브 지표3
         ↑          ↑          ↑
    세부지표들   세부지표들   세부지표들
```

## 실전에서의 적용

### 모든 회의에서 북극성 먼저

```
회의 시작:
"이번 주 북극성 지표는 어땠나요?"

기능 제안할 때:
"이 기능이 북극성 지표에 어떤 영향을 주나요?"
```

### 의사결정 기준으로 활용

```
A안: 북극성 지표 5% 상승 예상
B안: 북극성 지표 영향 불확실

→ A안 선택
```

### 북극성이 안 오르면 다시 생각

```
열심히 했는데 북극성 지표가 안 오른다?
→ 하고 있는 일을 다시 점검
→ 다른 접근법 필요
```

## 흔한 실수

### 1. 너무 자주 바꾸기

```
❌ 이번 달은 이거, 다음 달은 저거
✅ 최소 6개월~1년은 같은 지표
```

### 2. 허영 지표를 북극성으로

```
❌ 앱 다운로드 수 (가치와 멀어요)
✅ 활성 사용자 수 (가치에 가까워요)
```

### 3. 측정이 어려운 지표

```
❌ "고객 만족" (어떻게 측정해요?)
✅ "NPS 점수" 또는 "재구매율" (측정 가능)
```

## 정리

:::callout[important]
모든 지표를 다 보려면 아무것도 못 봐요.
하나의 북극성 지표를 정하고, 모든 팀이 그걸 올리기 위해 일하세요.
:::

**핵심:**
- 지표가 많으면 집중력이 분산돼요
- 북극성 지표 = 모든 팀의 목표
- 고객 가치와 비즈니스 성장을 반영해야 해요
- 서브 지표로 각 팀의 역할을 명확히 해요
- 최소 6개월은 같은 북극성을 유지하세요
