## 핵심 요약

리텐션은 유저가 계속 제품을 사용하는 비율이다. 신규 유저 획득보다 기존 유저 유지가 비용 효율적이며, 리텐션이 성장의 핵심이다.

---

## 리텐션 핵심 개념

| 개념 | 설명 |
|------|------|
| D1/D7/D30 | 가입 후 1/7/30일째 재방문율 |
| Retention Curve | 시간에 따른 잔존율 그래프 |
| Churn | 이탈 (= 1 - Retention) |
| Cohort | 같은 시점에 가입한 유저 그룹 |
| Sticky Factor | DAU/MAU 비율 |

## 리텐션 vs 획득 밸런스

| 관점 | 신규 획득 | 리텐션 개선 |
|------|----------|------------|
| 비용 | 5-25배 더 비쌈 | 상대적 저비용 |
| 효과 | 즉시 성장 가능 | 복리 효과 |
| 리스크 | CAC 상승 위험 | 천장 있음 |
| 우선순위 | PMF 이전 | PMF 이후 |

## 실무 팁

:::callout[tip]
**리텐션 2% 개선 = 신규 획득 10% 효과**

왜 리텐션이 중요한가:
- 기존 유저 → 추가 비용 없이 매출
- 기존 유저 → 입소문 유도
- 기존 유저 → 데이터 축적

→ 리텐션은 복리로 성장합니다
:::

:::callout[warning]
**"매일 푸시 보내면 되겠지?"는 위험**

과도한 리텐션 시도의 역효과:
- 알림 끄기 / 앱 삭제
- 브랜드 이미지 하락
- 법적 이슈 (스팸)

→ 가치 있는 알림만, 적절한 빈도로
:::
