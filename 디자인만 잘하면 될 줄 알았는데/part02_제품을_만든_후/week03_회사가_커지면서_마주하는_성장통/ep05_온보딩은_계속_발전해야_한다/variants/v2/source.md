# 온보딩은 계속 발전해야 한다 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

온보딩은 신규 유저가 제품의 핵심 가치를 경험하도록 안내하는 과정이다. 첫 경험이 리텐션을 결정하므로, 지속적인 측정과 개선이 필요하다.

---

## 온보딩 핵심 개념

| 개념 | 설명 |
|------|------|
| Aha Moment | 유저가 제품의 가치를 깨닫는 순간 |
| Activation | 핵심 행동을 완료한 상태 |
| Time to Value | 가입부터 가치 경험까지 걸리는 시간 |
| Friction | 가치 경험을 방해하는 요소 |

---

## 온보딩 플로우 기본 체크리스트

### 가입 단계
- [ ] 필수 입력 최소화 (이메일만 또는 소셜 로그인)
- [ ] 비밀번호 규칙 명확히 안내
- [ ] 에러 메시지 친절하게
- [ ] 진행률 표시 (옵션)

### 환영 단계
- [ ] 환영 메시지
- [ ] 제품의 핵심 가치 전달
- [ ] 첫 번째 행동 유도 CTA
- [ ] Skip 옵션 제공

### 핵심 행동 유도
- [ ] Aha Moment로 빠르게 안내
- [ ] 단계별 가이드 제공
- [ ] 진행 상황 피드백
- [ ] 완료 시 축하/보상

---

## 온보딩 유형별 체크리스트

### 툴팁/힌트 방식
- [ ] 핵심 기능만 설명
- [ ] 3-5개 이내 단계
- [ ] 언제든 닫을 수 있게
- [ ] 다시 볼 수 있게

### 프로그레스 바 방식
- [ ] 설정 완료 % 표시
- [ ] 각 단계 클릭 가능
- [ ] 완료 시 보상/혜택
- [ ] 건너뛰기 가능

### 빈 상태 (Empty State) 방식
- [ ] 할 일을 명확히 제시
- [ ] 예시 데이터 제공 (옵션)
- [ ] 첫 행동 CTA 강조
- [ ] 도움말 링크

### 코치마크 방식
- [ ] UI 위에 오버레이 안내
- [ ] 클릭으로 다음 단계
- [ ] 현재 위치 하이라이트
- [ ] 진행률 표시

## 실무 팁

:::callout[tip]
**"완벽한 온보딩"보다 "빠른 가치 경험"**

긴 온보딩 = 좋은 온보딩? X

- 유저는 배우러 온 게 아니라 문제 해결하러 옴
- 설명보다 직접 해보게 하기
- 필요한 것만 그때그때 알려주기
- Learning by doing!
:::

:::callout[warning]
**온보딩 ≠ 제품 설명서**

흔한 실수:
- 모든 기능을 다 보여주려 함
- 읽어야 하는 텍스트가 많음
- 클릭할 게 많음
- 건너뛸 수가 없음

→ 핵심 하나만 전달하세요
:::
