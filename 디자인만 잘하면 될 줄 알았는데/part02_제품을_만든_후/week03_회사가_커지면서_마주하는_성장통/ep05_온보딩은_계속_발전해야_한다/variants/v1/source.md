# 온보딩은 계속 발전해야 한다

## 스토리텔링 버전

:::conversation
**N**: 한입님, 저희 가입자는 많은데 다 떠나요. 첫 화면 보고 바로 나가버리는 것 같아요...

**한입**: 첫인상이 별로였나 보네요. 온보딩을 살펴봐야 할 것 같아요.

**N**: 온보딩이요?

**한입**: 네! 첫 만남부터 "아, 이거 괜찮네!" 하는 순간까지 안내하는 과정이에요.
:::

### 온보딩 = 첫 데이트

:::conversation
**한입**: 첫 데이트 생각해보세요.

**N**: 네?

**한입**: 처음 만났는데 상대방이 30분 동안 자기 이야기만 하면?

**N**: 지루하죠. 다음 만남은 없을 것 같아요.

**한입**: 앱도 마찬가지예요! 처음 들어왔는데 튜토리얼 5개 보고, 회원정보 입력하고, 관심사 선택하고... 이러면 지쳐서 나가버려요.
:::

### 좋은 온보딩의 원칙

**1. 빨리 가치를 보여주세요**

```
❌ 튜토리얼 → 회원가입 → 관심사 → 드디어 시작
✅ 핵심 기능 맛보기 → 좋으면 회원가입
```

:::callout[important]
사용자의 인내심은 5초예요!
가치를 빨리 보여줘야 해요.
:::

**2. 한 번에 하나씩**

```
❌ 첫 화면에서 모든 기능 설명
✅ 지금 필요한 것 하나만 안내
```

**3. 행동으로 배우게**

```
❌ "이 버튼을 누르면 됩니다" (글로 설명)
✅ "눌러보세요" (직접 해보게)
```

**4. 건너뛸 수 있게**

```
❌ 튜토리얼 강제 시청
✅ "건너뛰기" 옵션 항상 제공
```

### 온보딩의 4단계

:::conversation
**한입**: 온보딩에는 단계가 있어요.

**1단계: 첫 인상 (0~10초)**
- 사용자: "이게 뭐 하는 앱이야?"
- 해야 할 것: 한 문장으로 가치 전달

**2단계: 첫 행동 (10초~2분)**
- 사용자: "어떻게 쓰는 거야?"
- 해야 할 것: 가장 중요한 첫 행동 유도

**3단계: 가치 경험 (2분~첫 세션)**
- 사용자: "이게 나한테 도움이 돼?"
- 해야 할 것: "아, 이거 괜찮네!" 순간 만들기

**4단계: 습관화 (첫 주~첫 달)**
- 사용자: "계속 쓸 이유가 있어?"
- 해야 할 것: 리마인더, 새 기능 점진적 소개
:::

### Aha Moment를 찾아라

:::conversation
**N**: "아, 이거 괜찮네!" 순간이 중요하다고 하셨는데, 어떻게 찾아요?

**한입**: 유명한 사례들이 있어요.

**페이스북:** 7일 내 친구 10명 추가
**슬랙:** 팀에서 메시지 2,000개 전송
**줌:** 첫 화상회의 완료

**N**: 그럼 저희도 찾아야겠네요?

**한입**: 네! 리텐션 높은 사용자들의 공통 행동을 분석해보세요. 그게 힌트예요.
:::

### 계속 발전해야 해요

:::callout[warning]
온보딩은 "완성"이 없어요!
3년 전 온보딩을 그대로 쓰면 안 돼요.
:::

**개선 방법:**

1. **퍼널 분석**
   - 가입 100명 → 첫 화면 80명 → 첫 행동 40명...
   - 어디서 많이 빠지나?

2. **사용자 테스트**
   - 실제로 처음 쓰는 사람 관찰
   - "어디서 막히셨어요?"

3. **A/B 테스트**
   - A: 기존 온보딩
   - B: 새 온보딩
   - 어느 쪽이 전환율 높나?

### 흔한 실수

```
❌ 너무 많은 정보: 첫 화면에서 다 설명하려 함
❌ 강제적인 흐름: 건너뛸 수 없는 튜토리얼
❌ 가치 전달 지연: 다 입력하고 나서야 시작
❌ 업데이트 안 함: 수년간 같은 온보딩
```
