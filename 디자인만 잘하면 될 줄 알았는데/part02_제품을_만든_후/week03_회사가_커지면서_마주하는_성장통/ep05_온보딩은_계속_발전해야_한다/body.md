# 온보딩은 계속 발전해야 한다

## "첫 화면 보고 다 나가요"

열심히 만든 서비스인데, 가입하자마자 이탈하는 사람이 많아요. 첫 경험이 중요하다는 건 알겠는데, 온보딩을 어떻게 개선해야 할까요?

## 온보딩이란?

### 첫 만남부터 가치를 느끼는 순간까지

```
온보딩 = 처음 들어온 사용자가
         서비스의 가치를 느끼도록 안내하는 과정
```

:::callout[tip]
목표는 "아, 이거 괜찮네!"라는 순간(Aha moment)에 도달하게 하는 거예요.
:::

### 온보딩이 실패하면

```
가입 → 첫 화면 → "뭐 하는 앱이지?" → 이탈

→ 아무리 좋은 기능이 있어도 못 써보고 떠남
```

## 좋은 온보딩의 원칙

### 1. 빨리 가치를 보여주세요

```
❌ 튜토리얼 5개 → 회원정보 입력 → 관심사 선택 → 드디어 시작
✅ 핵심 기능 먼저 체험 → 필요할 때 정보 입력
```

==사용자는 인내심이 없어요. 가치를 빨리 보여줘야 해요.==

### 2. 한 번에 하나씩

```
❌ 첫 화면에 모든 기능 설명
✅ 지금 필요한 것 하나만 안내
```

:::callout[warning]
설명이 많으면 아무것도 안 읽어요.
:::

### 3. 행동으로 배우게

```
❌ "이 버튼을 누르면 됩니다" 글로 설명
✅ "눌러보세요" 하고 직접 눌러보게
```

### 4. 언제든 건너뛸 수 있게

```
❌ 튜토리얼 강제 시청
✅ "건너뛰기" 옵션 제공
```

이미 아는 사용자도 있어요.

## 온보딩 단계별 설계

### 1단계: 첫 인상 (0~10초)

```
사용자 질문: "이게 뭐 하는 앱이야?"

해야 할 것:
- 명확한 가치 제안 (한 문장)
- 시각적으로 깔끔한 첫 화면
- 즉시 시작 버튼
```

### 2단계: 첫 행동 (10초~2분)

```
사용자 질문: "어떻게 쓰는 거야?"

해야 할 것:
- 가장 중요한 첫 행동 유도
- 성공 시 보상/피드백
- 다음 단계 안내
```

### 3단계: 가치 경험 (2분~첫 세션)

```
사용자 질문: "이게 나한테 도움이 돼?"

해야 할 것:
- 핵심 기능으로 문제 해결 경험
- "아, 이거 괜찮네!" 순간
```

### 4단계: 습관화 (첫 주~첫 달)

```
사용자 질문: "계속 쓸 이유가 있어?"

해야 할 것:
- 정기적인 리마인더
- 새 기능 점진적 소개
- 성과/진전 보여주기
```

## 온보딩 유형

### 1. 기능 투어

```
화면 순서대로 기능 설명
장점: 체계적
단점: 지루함, 기억 안 남
```

### 2. 체험형 온보딩

```
실제로 해보면서 배우기
장점: 기억에 남음
단점: 설계 복잡
```

### 3. 점진적 공개

```
처음엔 기본만, 익숙해지면 점점 더 공개
장점: 부담 없음
단점: 고급 기능 발견 못 할 수 있음
```

### 4. 개인화된 온보딩

```
사용자 목적에 따라 다른 경로
장점: 관련성 높음
단점: 개발 복잡
```

## 온보딩 개선하기

### 1. 데이터로 확인

```
퍼널 분석:
가입 100명 → 첫 화면 80명 → 첫 행동 40명 → ...
어디서 이탈이 많은지 확인
```

### 2. 사용자 테스트

```
실제 사용자에게 첫 사용 경험 관찰
"어디서 막히시나요?"
"이게 뭔지 아시겠어요?"
```

### 3. A/B 테스트

```
A: 기존 온보딩
B: 새 온보딩
어느 쪽이 전환율이 높은지 비교
```

### 4. 반복 개선

```
온보딩은 한 번 만들고 끝이 아니에요.
데이터 보면서 계속 개선해야 해요.
```

:::callout[important]
온보딩은 "완성"이 없어요. 계속 발전해야 해요.
:::

## 흔한 실수

### 1. 너무 많은 정보

```
❌ 첫 화면에서 모든 기능 설명
✅ 딱 하나만, 나중에 점진적으로
```

### 2. 강제적인 흐름

```
❌ 건너뛸 수 없는 튜토리얼
✅ 언제든 건너뛰기 가능
```

### 3. 가치 전달 지연

```
❌ 회원정보 다 입력하고 나서야 시작
✅ 먼저 맛보기, 나중에 정보 입력
```

### 4. 업데이트 안 함

```
❌ 3년 전 온보딩 그대로
✅ 데이터 보면서 계속 개선
```

## 정리

:::callout[important]
온보딩의 목표는 사용자가 "아, 이거 괜찮네!"를 느끼게 하는 거예요.
빨리 가치를 보여주고, 데이터 보면서 계속 개선하세요.
:::

- 온보딩 = 첫 만남부터 Aha moment까지
- 빨리 가치를 보여주세요
- 한 번에 하나씩, 행동으로 배우게
- 퍼널 분석으로 이탈 지점 확인
- 온보딩은 계속 발전해야 해요
