## 카드 2: 왜 쪼개야 할까?

🤔 **커지면 생기는 문제**

- 👥 10명이 하나의 코드를 만지면 충돌
- 📞 모든 결정에 모든 사람이 필요
- 🐌 작은 수정도 영향 범위가 너무 큼
- 🤷 누가 뭘 담당하는지 모호함

---

## 카드 3: 팀 쪼개기

👥 **팀 분리 신호**

| 신호 | 의미 |
|------|------|
| 회의가 너무 길다 | 이해관계자 과다 |
| 결정이 느리다 | 책임 불명확 |
| 충돌이 잦다 | 영역 겹침 |
| 10명 초과 | 피자 두 판의 법칙 |

---

## 카드 4: 코드 쪼개기

💻 **코드 분리 (마이크로서비스)**

| 모놀리식 | 마이크로서비스 |
|----------|----------------|
| 🏢 하나의 큰 건물 | 🏘️ 여러 작은 집 |
| 한 곳 고장 → 전체 영향 | 독립적 운영 |
| 배포 = 전체 재시작 | 부분 배포 가능 |
| 시작은 빠름 | 확장은 쉬움 |

---

## 카드 5: 쪼개는 타이밍

⏰ **언제 쪼개야 할까?**

- ✅ 팀이 10명을 넘어갈 때
- ✅ 배포 주기가 길어질 때
- ✅ 장애 영향 범위가 너무 클 때
- ✅ 팀 간 의존성이 병목이 될 때

❌ 너무 일찍 쪼개지 마세요!
