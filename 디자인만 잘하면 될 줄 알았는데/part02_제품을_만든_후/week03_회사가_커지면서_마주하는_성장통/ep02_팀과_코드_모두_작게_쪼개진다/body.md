# 팀과 코드 모두 작게 쪼개진다

## "왜 갑자기 팀이 나뉘었죠?"

처음엔 개발자 5명이 다 같이 일했는데, 어느 순간 "주문팀", "결제팀", "배송팀"으로 나뉘었어요. 코드도 여러 개로 쪼개졌대요. 왜 이렇게 된 걸까요?

## 작게 쪼개는 이유

### 팀이 커지면 생기는 문제

```
5명 팀: 카톡방 하나로 소통 끝
50명 팀: 누가 뭘 하는지 모름, 회의만 하다 하루 끝남
```

:::callout[tip]
팀이 커질수록 소통 비용이 기하급수적으로 늘어나요.
:::

### 코드가 커지면 생기는 문제

```
작은 코드: 수정하면 바로 배포
큰 코드: 한 곳 수정 → 다른 곳에 영향 → 테스트만 3일
```

==조직이든 코드든, 커지면 느려지고 복잡해져요.==

## 팀을 쪼개는 방법

### 기능별 팀 (Feature Team)

```
주문팀: 장바구니, 주문서, 주문 내역
결제팀: 결제 수단, 결제 처리, 환불
배송팀: 배송 추적, 배송지 관리
```

**장점:**
- 한 기능을 끝까지 책임짐
- 빠른 의사결정

**단점:**
- 팀 간 기능이 겹치면 혼란

### 투 피자 룰 (Two Pizza Rule)

```
제프 베조스의 룰:
"팀이 피자 두 판으로 배부를 수 없다면 너무 큰 거다"

→ 보통 6~10명이 적정
```

## 코드를 쪼개는 방법

### 모놀리스 vs 마이크로서비스

```
모놀리스 (Monolith):
- 하나의 큰 코드 덩어리
- 모든 기능이 하나로 연결
- 초기 스타트업에 적합

마이크로서비스 (Microservices):
- 여러 개의 작은 서비스
- 각 서비스가 독립적
- 큰 조직에 적합
```

:::callout[warning]
너무 빨리 마이크로서비스로 가면 오히려 복잡해져요.
규모가 커졌을 때 쪼개는 게 좋아요.
:::

### 비유로 이해하기

```
모놀리스 = 원룸
- 모든 게 한 공간에
- 처음엔 효율적
- 사람 늘면 불편

마이크로서비스 = 아파트
- 각자 독립된 공간
- 관리비(인프라 비용) 증가
- 사람 많을 때 효율적
```

## 쪼개면 생기는 변화

### 1. 담당자가 명확해져요

```
전: "이 버그 누가 고쳐요?" → 서로 미룸
후: "결제 관련이네? 결제팀!" → 담당 명확
```

### 2. 배포가 독립적이에요

```
전: 주문 기능 수정 → 전체 배포 → 결제도 영향 받음
후: 주문 기능 수정 → 주문 서비스만 배포 → 다른 건 영향 없음
```

### 3. 기술 선택이 자유로워요

```
주문팀: Python으로 개발
결제팀: Java로 개발
각 팀이 자기 상황에 맞게 선택 가능
```

## 쪼개면 생기는 문제

### 1. 팀 간 소통이 필요해요

```
주문팀: "결제 API 이렇게 바꿔주세요"
결제팀: "그거 바꾸면 다른 곳도 다 고쳐야 해요"
→ 조율 필요
```

### 2. 데이터가 흩어져요

```
사용자 정보가 여러 서비스에 나뉨
→ 한눈에 보기 어려움
→ 데이터 일관성 관리 필요
```

### 3. 장애가 복잡해져요

```
"결제가 안 돼요"
→ 결제 서비스 문제? 주문 서비스 문제? 네트워크 문제?
→ 추적이 어려움
```

## 디자이너에게 미치는 영향

### 담당 개발팀이 달라져요

```
화면 A: 주문팀 담당
화면 B: 결제팀 담당

같은 프로젝트인데 다른 팀과 협업
```

### 일관성 유지가 어려워요

```
주문팀이 만든 버튼
결제팀이 만든 버튼
→ 미묘하게 다를 수 있음

디자인 시스템의 중요성↑
```

### API 연동 화면이 늘어요

```
전: 하나의 서버에서 모든 데이터
후: 주문 데이터, 결제 데이터, 배송 데이터 따로

→ 로딩 상태 각각 고려 필요
→ 에러 상태 각각 고려 필요
```

## 정리

:::callout[important]
조직과 코드가 커지면 작게 쪼개야 해요.
쪼개면 빠르고 독립적이지만, 협업과 일관성에 신경 써야 해요.
:::

**핵심:**
- 팀이 커지면 소통 비용이 급증해요
- 코드가 커지면 수정/배포가 느려져요
- 기능별로 팀과 코드를 쪼개요
- 쪼개면 독립성↑, 하지만 협업 필요
- 디자이너도 여러 팀과 일하게 돼요
