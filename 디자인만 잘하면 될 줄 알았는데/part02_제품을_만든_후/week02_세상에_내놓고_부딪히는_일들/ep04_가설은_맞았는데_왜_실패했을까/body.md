# 가설은 맞았는데 왜 실패했을까

## "우리 가설은 맞았는데요?"

유저 리서치도 했고, 데이터 분석도 했어요. 가설도 검증됐죠. 근데 왜 서비스는 성장하지 않을까요?

## 가설 검증의 함정

### 가설이 맞았다는 건 무슨 뜻일까?

```
"사람들은 건강한 음식 배달을 원한다" - 가설

설문 결과: 80%가 "원한다"고 답함 - 검증?
```

:::callout[warning]
사람들이 "원한다"고 말하는 것과 실제로 "돈을 내는 것"은 완전히 다른 문제예요.
:::

### 말과 행동의 차이

```
사람들이 말한 것:
- "건강한 음식을 먹고 싶어요"
- "운동을 더 하고 싶어요"
- "책을 더 읽고 싶어요"

사람들이 실제로 한 것:
- 치킨을 시켰다
- 넷플릭스를 봤다
- 유튜브를 봤다
```

==사람들은 자신이 되고 싶은 모습을 말해요. 실제 행동과는 다르죠.==

## 가설이 맞아도 실패하는 이유

### 1. 문제는 맞았는데 솔루션이 틀렸어요

```
문제: 직장인들이 점심을 건강하게 먹기 어렵다 ✓
솔루션: 건강식 배달 앱을 만들자

왜 안 됐을까?
- 이미 배달 앱이 너무 많다
- 건강식이 맛이 없다
- 가격이 비싸다
```

### 2. 시장 크기가 너무 작았어요

```
"20대 여성 중 비건이면서 요가를 하는 사람"

가설: 이 타깃은 프리미엄 비건 요가복을 원한다
현실: 그 사람들이 전국에 1,000명밖에 없다
```

:::callout[tip]
틈새시장은 좋지만, 너무 좁으면 비즈니스가 안 돼요.
:::

### 3. 타이밍이 안 맞았어요

```
너무 빠른 경우:
- 기술이 아직 성숙하지 않음
- 사용자가 준비되지 않음

너무 늦은 경우:
- 이미 경쟁자가 시장을 점령함
- 대기업이 들어옴
```

### 4. 실행에서 무너졌어요

```
좋은 가설 + 나쁜 실행 = 실패
나쁜 가설 + 좋은 실행 = 그래도 실패
좋은 가설 + 좋은 실행 = 성공 가능성
```

**실행에서 무너지는 경우:**
- 제품 완성도가 낮음
- 마케팅을 못함
- 자금이 부족함
- 팀이 와해됨

### 5. 경쟁을 무시했어요

```
"우리가 처음이야!" → 실제론 비슷한 게 10개 있음
"우리가 더 좋아!" → 사용자는 그 차이를 모름
"우리만 이거 할 수 있어!" → 3개월 뒤 대기업이 똑같이 만듦
```

## 가설 vs 실험의 차이

### 가설만 세우는 팀

```
1주차: 가설 세우기
2주차: 설문조사 만들기
3주차: 설문조사 돌리기
4주차: 결과 분석
5주차: "가설이 맞았다!" 선언
6주차~: 6개월간 제품 개발
출시 후: "왜 안 되지?"
```

### 실험하는 팀

```
1주차: 가설 세우기
2주차: 최소한의 제품(MVP) 만들기
3주차: 실제 사용자에게 테스트
4주차: 결과 분석 + 가설 수정
5주차: 다시 테스트
...
빠른 반복
```

:::callout[important]
설문으로 검증한 가설과 돈을 받고 검증한 가설은 완전히 달라요.
:::

## 진짜 검증은 이렇게

### 1. 돈으로 검증하기

```
❌ "이런 서비스 있으면 쓰시겠어요?" → "네!"
✅ "지금 결제하시면 이용 가능해요" → "음... 다음에요"
```

사람들이 지갑을 여는 순간이 진짜 검증이에요.

### 2. 행동으로 검증하기

```
❌ 관심 표현: "알림 받을게요"
✅ 실제 행동: 웹사이트에서 10분 이상 머물렀다
```

### 3. 반복으로 검증하기

```
❌ 한 번 써봄: "괜찮네요"
✅ 계속 씀: 매주 3번 이상 접속
```

## 실패에서 배우기

### 실패했을 때 물어볼 것

1. **문제가 진짜 있었나?**
   - 설문이 아니라 행동으로 확인했나?

2. **솔루션이 맞았나?**
   - 더 좋은 방법은 없었나?

3. **타깃이 맞았나?**
   - 더 절실한 사람들이 있었나?

4. **실행은 잘 했나?**
   - 제품 완성도는 괜찮았나?

5. **타이밍은 맞았나?**
   - 너무 빠르거나 늦지 않았나?

### 피벗(Pivot) 고려하기

```
전체를 바꾸지 말고 하나씩 바꿔보기:

- 타깃 변경: B2C → B2B
- 솔루션 변경: 앱 → 웹
- 비즈니스 모델 변경: 구독 → 건당 결제
- 기능 축소: 10가지 → 1가지 핵심 기능
```

## 정리

:::callout[important]
가설이 맞았다고 성공이 보장되지 않아요.
설문 결과보다 실제 결제, 실제 재방문을 보세요.
:::

**핵심:**
- 말과 행동은 다르다는 걸 기억하세요
- 설문보다 실제 돈을 받아보세요
- 가설이 맞아도 실행이 중요해요
- 실패하면 왜 실패했는지 분석하세요
- 전체를 바꾸지 말고 하나씩 피벗하세요
