# 사용자가 늘면 서버가 터진다는 게 무슨 뜻이죠

## "서버 터졌어요!"

어느 날 갑자기 앱이 안 돼요. 개발팀에 물어보니 "서버가 터졌다"고 해요. 도대체 서버가 터진다는 게 무슨 뜻일까요? 진짜 폭발하는 건가요?

## 서버란 뭘까요?

### 식당으로 비유하면

```
사용자 = 손님
앱 = 메뉴판
서버 = 주방

손님이 메뉴판 보고 주문하면,
주방에서 요리해서 가져다줘요.
```

==서버는 사용자의 요청을 처리하는 컴퓨터예요.==

### 서버가 하는 일

- 로그인 정보 확인
- 데이터 저장하고 불러오기
- 이미지, 동영상 전달
- 결제 처리
- 알림 보내기

## "터진다"는 게 무슨 뜻?

### 진짜 폭발은 아니에요

```
물리적 폭발 ❌
시스템 과부하로 멈춤 ✅
```

:::callout[tip]
"터졌다" = 요청이 너무 많아서 처리를 못 하는 상태
:::

### 식당 비유로 다시 보면

```
평소: 주방에 요리사 3명, 손님 30명 → 괜찮아요
오늘: 주방에 요리사 3명, 손님 3,000명 → ???

결과:
- 주문 밀림 (응답 지연)
- 요리사 과로 (CPU 100%)
- 재료 부족 (메모리 부족)
- 결국 가게 문 닫음 (서버 다운)
```

## 왜 갑자기 사용자가 늘까요?

### 예상 가능한 상황

```
✓ 마케팅 캠페인 후
✓ 언론 보도 후
✓ 바이럴 됐을 때
✓ 이벤트 시작할 때
✓ 세일 기간
```

### 예상 못한 상황

```
✓ 유명인이 갑자기 언급
✓ 커뮤니티에서 화제
✓ 경쟁사 장애로 유입
```

:::callout[warning]
"좋은 문제"지만, 미리 준비 안 하면 기회를 날려요.
:::

## 개발팀이 쓰는 용어들

### 트래픽 (Traffic)

```
"트래픽이 폭주했어요"
= 사용자 요청이 엄청 많아졌어요
```

### 스케일링 (Scaling)

```
"스케일 업 해야 해요"
= 서버 성능을 높여야 해요

"스케일 아웃 해야 해요"
= 서버 대수를 늘려야 해요
```

식당 비유:
- 스케일 업 = 요리사를 더 잘하는 사람으로 교체
- 스케일 아웃 = 요리사를 더 고용

### 로드 밸런싱 (Load Balancing)

```
"로드 밸런서 설정 확인 중이에요"
= 트래픽을 여러 서버에 분산하는 장치 점검 중
```

식당 비유: 손님을 여러 테이블에 골고루 안내하는 직원

### 캐싱 (Caching)

```
"캐시 적용하면 버틸 수 있어요"
= 자주 요청되는 데이터를 미리 준비해두는 것
```

식당 비유: 인기 메뉴는 미리 만들어두기

### 오토 스케일링 (Auto Scaling)

```
"오토 스케일링 설정해놨어요"
= 트래픽에 따라 서버가 자동으로 늘어나요
```

식당 비유: 손님이 늘면 자동으로 요리사가 출근하는 시스템

## 디자이너가 도울 수 있는 것

### 1. 로딩 상태 디자인

```
서버 응답이 느릴 때 보여줄 화면:
- 로딩 스피너
- 스켈레톤 UI
- 진행률 표시
```

:::callout[tip]
사용자가 "뭔가 되고 있구나"라고 느끼게 해주세요.
:::

### 2. 에러 상태 디자인

```
서버가 완전히 멈췄을 때:
- 친근한 에러 메시지
- 재시도 버튼
- 대안 안내 (고객센터 등)
```

**나쁜 예:**
```
Error 503: Service Unavailable
```

**좋은 예:**
```
지금 접속이 많아요 😅
잠시 후 다시 시도해주세요.
[다시 시도] [고객센터 문의]
```

### 3. 이미지 최적화

```
❌ 10MB 원본 이미지 그대로
✅ 적절한 크기로 압축 (수백 KB)
```

이미지가 가벼울수록 서버 부담이 줄어요.

### 4. 불필요한 요청 줄이기

```
❌ 스크롤할 때마다 서버에 요청
✅ 필요할 때만 요청 (무한 스크롤 최적화)
```

## 트래픽 폭주 대비하기

### 출시 전 체크리스트

- [ ] 예상 사용자 수 계산했나요?
- [ ] 피크 시간대 대비했나요?
- [ ] 마케팅 일정과 공유했나요?
- [ ] 모니터링 시스템 있나요?
- [ ] 장애 발생 시 대응 매뉴얼 있나요?

### 마케팅과의 소통

:::callout[important]
마케팅 캠페인 일정을 개발팀에 미리 공유하세요!
:::

```
❌ 마케팅팀: "내일 유튜버 광고 나가요"
   개발팀: "어... 지금요?"
   결과: 서버 터짐

✅ 마케팅팀: "2주 뒤 유튜버 광고 예정이에요"
   개발팀: "서버 미리 늘려놓을게요"
   결과: 성공적인 유입
```

## 정리

:::callout[important]
서버가 터진다 = 요청이 너무 많아서 처리를 못 한다.
트래픽 증가가 예상되면 미리 개발팀에 알려주세요.
:::

**핵심:**
- 서버는 사용자 요청을 처리하는 컴퓨터예요
- "터진다"는 과부하로 멈춘다는 뜻이에요
- 마케팅 일정은 개발팀과 공유해야 해요
- 로딩/에러 상태 디자인이 중요해요
- 이미지 최적화로 서버 부담을 줄일 수 있어요
