# 첫 유저 100명의 행동이 중요한 이유

## "유저가 생겼어요! 근데... 뭘 봐야 하죠?"

드디어 서비스가 출시됐고, 첫 유저들이 들어오기 시작했어요. 100명... 1,000명... 숫자가 늘어나는 게 신기하죠. 그런데 문득 궁금해져요. "이 숫자만 보면 되는 건가?"

## 첫 100명이 특별한 이유

### 그들이 바로 '얼리어답터'

```
일반 사용자: 검증된 서비스만 써요
얼리어답터: 새로운 걸 먼저 써보고 싶어해요
```

==첫 100명은 불편함을 감수하고 당신의 서비스를 써보는 사람들이에요.==

이 사람들의 특징:
- 문제에 대한 고민이 깊어요
- 기꺼이 피드백을 줘요
- 서비스가 성장하면 함께 기뻐해줘요

### 숫자보다 행동이 중요해요

**나쁜 지표:**
```
✓ 가입자 수: 1,000명
✓ 앱 다운로드: 500회
```

**좋은 지표:**
```
✓ 가입 후 첫 주에 3번 이상 접속한 사람: ?명
✓ 핵심 기능을 사용한 사람: ?명
✓ 친구에게 추천한 사람: ?명
```

:::callout[important]
1,000명이 가입하고 모두 떠났다면 실패예요.
100명이 가입하고 80명이 남았다면 성공의 신호예요.
:::

## 봐야 할 핵심 지표들

### 1. 활성화 (Activation)

"첫 경험이 좋았나요?"

```
가입 → 첫 경험 → 이탈 or 계속

예시:
- 첫 게시물을 작성했나요?
- 프로필을 완성했나요?
- 첫 구매를 했나요?
```

:::callout[tip]
사용자가 "아, 이거 괜찮네!"라고 느끼는 순간을 찾으세요.
그게 바로 'Aha moment'예요.
:::

### 2. 리텐션 (Retention)

"다시 돌아오나요?"

```
Day 1 리텐션: 다음 날 다시 온 비율
Day 7 리텐션: 일주일 뒤에도 쓰는 비율
Day 30 리텐션: 한 달 뒤에도 쓰는 비율
```

==리텐션이 0%로 수렴하면, 아무리 많은 유저가 와도 결국 빈 서비스가 돼요.==

### 3. 참여도 (Engagement)

"얼마나 적극적으로 쓰나요?"

- 한 번 접속할 때 머무는 시간
- 핵심 기능 사용 횟수
- 댓글, 좋아요 등 상호작용

### 4. 추천 의향 (NPS)

"친구에게 추천하고 싶나요?"

```
0~6점: 비추천자 (Detractor)
7~8점: 중립
9~10점: 추천자 (Promoter)

NPS = 추천자 비율 - 비추천자 비율
```

## 첫 유저에게 배우는 방법

### 1. 직접 대화하기

```
❌ 설문조사 링크 보내기
✅ 1:1 통화나 만남 요청하기
```

첫 100명에게는 직접 연락하세요. 그들의 이야기에서 가장 중요한 인사이트가 나와요.

**물어볼 것들:**
- "어떻게 우리 서비스를 알게 됐어요?"
- "어떤 문제를 해결하려고 쓰셨어요?"
- "가장 불편했던 게 뭐예요?"
- "없으면 안 되는 기능이 뭐예요?"

### 2. 행동 데이터 보기

:::callout[tip]
사람들은 말과 행동이 다를 때가 많아요.
"자주 쓸 것 같아요"라고 말해도, 실제론 안 쓸 수 있죠.
:::

```
사용자가 말한 것: "이 기능 좋아요"
사용자의 행동: 그 기능을 한 번도 안 씀
```

### 3. 이탈 지점 찾기

```
가입 → 튜토리얼 → 첫 기능 사용 → ???
                              ↑
                         여기서 많이 나가네?
```

퍼널(Funnel)을 그려보세요. 어디서 사람들이 떠나는지 보여요.

## 실전 체크리스트

### 출시 첫 주에 확인할 것

- [ ] 가입 전환율 (방문자 중 가입한 비율)
- [ ] 활성화율 (가입자 중 핵심 기능 사용 비율)
- [ ] Day 1 리텐션
- [ ] 주요 이탈 지점

### 첫 달에 확인할 것

- [ ] Day 7, Day 30 리텐션
- [ ] 핵심 사용자 패턴 (많이 쓰는 사람들의 공통점)
- [ ] 추천 경로 (어디서 유입되나)
- [ ] NPS 점수

## 주의할 점

### 허영 지표(Vanity Metrics)에 속지 마세요

```
허영 지표:
- 총 가입자 수 (떠난 사람도 포함)
- 페이지뷰 (새로고침도 포함)
- 앱 다운로드 수 (삭제한 사람도 포함)

진짜 지표:
- 활성 사용자 수 (실제로 쓰는 사람)
- 리텐션율 (계속 쓰는 비율)
- 구매 전환율 (돈을 쓴 비율)
```

### 너무 빨리 판단하지 마세요

:::callout[warning]
100명의 데이터로 "우리 서비스 실패야"라고 결론 내리지 마세요.
통계적으로 의미 있으려면 최소한의 샘플이 필요해요.
:::

### 모든 피드백이 맞는 건 아니에요

```
10명 중 1명이 원하는 기능 → 바로 만들지 마세요
10명 중 8명이 불편해하는 것 → 바로 고치세요
```

## 정리

:::callout[important]
첫 100명은 숫자가 아니라 선생님이에요.
가입자 수보다 그들의 행동을 관찰하세요.
:::

**핵심:**
- 가입 수보다 리텐션이 중요해요
- 첫 유저들에게 직접 이야기를 들으세요
- 말보다 행동 데이터를 믿으세요
- 퍼널에서 이탈 지점을 찾으세요
- 허영 지표에 속지 마세요
