# 첫 유저 100명의 행동이 중요한 이유 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

첫 100명의 유저는 제품의 방향을 결정하는 가장 중요한 데이터 소스다. 이들의 행동과 피드백을 체계적으로 수집하고 분석해야 한다.

---

## 첫 100명이 중요한 이유

| 시점 | 유저 수 | 할 수 있는 것 |
|------|---------|--------------|
| 0-100명 | 소수 | 1:1 인터뷰, 직접 관찰, 빠른 실험 |
| 100-1,000명 | 중간 | 패턴 발견, A/B 테스트 시작 |
| 1,000명+ | 다수 | 통계적 유의미한 분석 가능 |

## 초기 유저 피드백 수집 체크리스트

### 피드백 채널
- [ ] 앱 내 피드백 버튼
- [ ] 이메일 문의
- [ ] 앱스토어 리뷰
- [ ] 고객 인터뷰 (화상/대면)
- [ ] 설문조사

### 피드백 수집 시점
- [ ] 가입 직후 (왜 가입했는지)
- [ ] 첫 사용 후 (첫인상)
- [ ] 일주일 후 (계속 쓸 의향)
- [ ] 이탈 시점 (왜 떠나는지)

### 질문 예시
| 시점 | 질문 |
|------|------|
| 가입 후 | "어떻게 알게 되셨나요?" |
| 사용 후 | "가장 유용한 기능은 무엇인가요?" |
| 이탈 시 | "무엇이 아쉬우셨나요?" |
| NPS | "친구에게 추천하시겠어요? (0-10)" |

## 피드백 분석 프레임워크

### 분류 기준
| 유형 | 설명 | 조치 |
|------|------|------|
| 버그 | 오작동, 에러 | 즉시 수정 |
| UX 개선 | 사용성 불편 | 우선순위 판단 |
| 기능 요청 | 새 기능 제안 | 로드맵 검토 |
| 칭찬 | 만족 표현 | 강점으로 기록 |
| 혼란 | 이해 안 됨 | 온보딩/UX 개선 |

### 우선순위 판단
- [ ] 몇 명이 같은 피드백을 주었는가?
- [ ] 핵심 유저가 언급한 피드백인가?
- [ ] 핵심 지표에 영향을 미치는가?
- [ ] 구현 난이도는 어떠한가?

## 실무 팁

:::callout[tip]
**"Mom Test" 원칙**

유저 인터뷰에서 진짜 정보를 얻으려면:
- "이 앱 어떠세요?" (X) → 좋다고 할 수밖에 없음
- "마지막으로 이 문제를 겪은 게 언제예요?" (O) → 실제 경험 기반

책 "The Mom Test" 참고
:::

:::callout[warning]
**10명의 피드백 ≠ 모든 유저의 의견**

초기 피드백의 함정:
- 소수의 목소리가 크게 들림
- 얼리 어답터 ≠ 일반 유저
- 요청한다고 다 만들면 안 됨

→ 데이터와 함께 판단하세요
:::
