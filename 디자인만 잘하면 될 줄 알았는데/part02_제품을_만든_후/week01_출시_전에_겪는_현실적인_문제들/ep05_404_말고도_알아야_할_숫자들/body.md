# 404 말고도 알아야 할 숫자들

## "404 에러가 났어요"

404는 많이 들어봤죠? "페이지를 찾을 수 없습니다"라는 뜻이에요. 그런데 500, 401, 503 같은 다른 숫자들도 있어요. 이게 다 뭘까요?

## HTTP 상태 코드란?

서버가 "요청을 어떻게 처리했는지" 알려주는 숫자예요.

:::callout[tip]
마치 택배 배송 상태와 비슷해요.
- 200: 배송 완료
- 404: 주소를 찾을 수 없음
- 500: 물류센터 문제
:::

## 숫자의 의미

### 첫 자리로 구분

| 번호대 | 의미 | 간단 설명 |
|--------|------|----------|
| 2xx | 성공 | 잘 됐어요! |
| 3xx | 리다이렉트 | 다른 곳으로 이동 |
| 4xx | 클라이언트 오류 | 요청이 잘못됨 |
| 5xx | 서버 오류 | 서버가 문제 |

## 자주 보는 상태 코드

### 성공 (2xx)

**200 OK**
- 요청 성공
- 가장 좋은 상태

**201 Created**
- 새로 만들어짐
- 예: 회원가입 완료, 글 작성 완료

### 클라이언트 오류 (4xx)

**400 Bad Request**
- 요청이 잘못됨
- 예: 필수 입력값 누락

**401 Unauthorized**
- 인증 필요
- 예: 로그인 안 한 상태로 접근

**403 Forbidden**
- 권한 없음
- 예: 관리자 페이지에 일반 유저가 접근

**404 Not Found**
- 페이지/리소스 없음
- 예: 삭제된 게시물, 잘못된 URL

**429 Too Many Requests**
- 요청이 너무 많음
- 예: 짧은 시간에 너무 많이 클릭

### 서버 오류 (5xx)

**500 Internal Server Error**
- 서버 내부 오류
- 개발자가 급하게 확인해야 함

**502 Bad Gateway**
- 게이트웨이 오류
- 서버 간 통신 문제

**503 Service Unavailable**
- 서비스 이용 불가
- 서버 점검 중이거나 과부하

**504 Gateway Timeout**
- 게이트웨이 시간 초과
- 응답이 너무 오래 걸림

## 디자이너가 알아야 하는 이유

### 1. 에러 화면 디자인

각 에러마다 다른 화면이 필요할 수 있어요.

| 코드 | 사용자에게 보여줄 메시지 |
|------|------------------------|
| 401 | "로그인이 필요해요" |
| 403 | "접근 권한이 없어요" |
| 404 | "페이지를 찾을 수 없어요" |
| 500 | "잠시 후 다시 시도해주세요" |

### 2. 원인 파악

```
사용자: "안 돼요!"
디자이너: "어떤 에러가 떴어요?"
사용자: "500이요"
디자이너: "서버 문제네요. 개발팀에 알릴게요"
```

### 3. 버그 리포트

에러 코드를 포함하면 원인 파악이 빨라요.

```
❌ "결제가 안 돼요"
✅ "결제 시 500 에러가 발생해요"
```

## 에러별 대응 가이드

### 4xx 에러 (클라이언트 문제)

**확인할 것:**
- URL이 정확한가?
- 로그인 상태인가?
- 권한이 있는 계정인가?
- 입력값이 올바른가?

**디자인 고려:**
- 사용자가 해결할 수 있는 안내 제공
- 예: "로그인 후 이용해주세요" + 로그인 버튼

### 5xx 에러 (서버 문제)

**확인할 것:**
- 개발팀에 바로 알리기
- 서버 점검 중인지 확인
- 모든 사용자에게 발생하는지

**디자인 고려:**
- 사용자 탓이 아님을 안내
- 재시도 버튼 제공
- 예: "일시적인 문제가 발생했어요. 잠시 후 다시 시도해주세요"

## 정리

:::callout[important]
HTTP 상태 코드는 에러의 원인을 알려주는 단서예요.
4xx는 요청 문제, 5xx는 서버 문제입니다.
에러 코드를 포함해서 리포트하면 해결이 빨라져요.
:::

**핵심:**
- 200: 성공
- 4xx: 클라이언트(사용자/요청) 문제
- 5xx: 서버 문제
- 에러 코드별로 다른 화면 디자인 필요
- 버그 리포트에 에러 코드 포함하기
