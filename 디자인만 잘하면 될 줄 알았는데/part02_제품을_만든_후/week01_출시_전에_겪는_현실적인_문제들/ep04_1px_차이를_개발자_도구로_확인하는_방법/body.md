# 1px 차이를 개발자 도구로 확인하는 방법

## "뭔가 달라 보이는데..."

디자인과 구현물을 비교했는데 뭔가 미묘하게 달라요. 느낌은 다른데 뭐가 다른지 정확히 모르겠어요. 이럴 때 개발자 도구를 쓰면 정확하게 확인할 수 있어요.

## 개발자 도구란?

모든 웹 브라우저에 내장된 도구예요. 웹페이지의 HTML, CSS, 네트워크 등을 검사할 수 있어요.

### 여는 방법

**Mac:**
- Chrome/Edge: `Cmd + Option + I`
- Safari: `Cmd + Option + I` (개발자 메뉴 활성화 필요)

**Windows:**
- Chrome/Edge: `F12` 또는 `Ctrl + Shift + I`

**또는:** 마우스 우클릭 → "검사" 또는 "요소 검사"

## 디자이너가 확인할 수 있는 것들

### 1. 크기 (Width, Height)

요소를 선택하면 우측에서 크기를 확인할 수 있어요.

```
Box Model:
┌─────────────────┐
│     margin      │
│  ┌───────────┐  │
│  │  padding  │  │
│  │ ┌───────┐ │  │
│  │ │content│ │  │
│  │ └───────┘ │  │
│  └───────────┘  │
└─────────────────┘
```

### 2. 여백 (Margin, Padding)

- **Margin**: 요소 바깥 여백 (오렌지색)
- **Padding**: 요소 안쪽 여백 (초록색)

### 3. 색상

Computed 탭에서 정확한 색상 코드 확인 가능

```
color: #333333
background-color: #FFFFFF
```

### 4. 폰트

```
font-family: Pretendard
font-size: 16px
font-weight: 600
line-height: 24px
```

## 실전 사용법

### Step 1: 의심되는 요소 선택

1. 개발자 도구 열기
2. 왼쪽 상단 화살표 아이콘 클릭 (요소 선택 모드)
3. 확인하고 싶은 요소 클릭

### Step 2: 스타일 확인

우측 패널에서:
- **Styles**: 적용된 CSS 규칙
- **Computed**: 최종 계산된 값
- **Box Model**: 크기와 여백

### Step 3: 디자인과 비교

```
디자인: padding 24px
개발자 도구: padding 20px
→ 4px 차이 발견!
```

## 자주 쓰는 확인 포인트

### 여백 확인

| 확인 항목 | 확인 방법 |
|----------|----------|
| 요소 간 간격 | margin 값 |
| 내부 여백 | padding 값 |
| 전체 크기 | width/height + padding |

### 폰트 확인

| 확인 항목 | CSS 속성 |
|----------|----------|
| 글꼴 | font-family |
| 크기 | font-size |
| 굵기 | font-weight |
| 행간 | line-height |
| 자간 | letter-spacing |

### 색상 확인

| 확인 항목 | CSS 속성 |
|----------|----------|
| 글자색 | color |
| 배경색 | background-color |
| 테두리 | border-color |

## 디자이너를 위한 팁

### 1. 스크린샷 비교

개발자 도구에서 확인한 값과 Figma 값을 나란히 놓고 비교하세요.

### 2. 정확한 값으로 피드백

```
❌ "여백이 좁아 보여요"
✅ "여백이 20px인데 디자인은 24px예요"
```

### 3. 반응형도 확인

개발자 도구에서 기기 모드를 바꿔서 다양한 화면 크기 테스트 가능

```
상단 기기 아이콘 클릭 → iPhone, iPad 등 선택
```

## 자주 발생하는 차이

### 1px 차이가 나는 이유

- border 포함 여부
- 반올림 처리
- 레티나 디스플레이 대응

### 색상이 다른 이유

- 투명도(opacity) 적용
- RGB vs HEX 변환 오차
- 컬러 프로파일 차이

### 폰트가 다른 이유

- 폰트 파일 미설치
- 웹폰트 로딩 실패
- 시스템 폰트 fallback

## 정리

:::callout[important]
개발자 도구를 사용하면 "느낌"이 아닌 "정확한 수치"로 차이를 파악할 수 있어요.
여백, 크기, 색상, 폰트를 정확히 확인하고 피드백하세요.
:::

**핵심:**
- 개발자 도구: F12 또는 Cmd+Option+I
- 요소 선택 후 Styles/Computed 탭 확인
- "느낌이 다르다" 대신 정확한 수치로 피드백
- 디자인과 실제 값을 나란히 비교
