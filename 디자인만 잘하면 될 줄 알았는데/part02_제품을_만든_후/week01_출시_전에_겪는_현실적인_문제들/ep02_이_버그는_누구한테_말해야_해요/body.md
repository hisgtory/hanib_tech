# 이 버그는 누구한테 말해야 해요

## "발견했는데 어디다 말하지?"

QA 중에 이상한 점을 발견했어요. 근데 이게 프론트엔드 문제인지, 백엔드 문제인지, 아니면 기획 문제인지 모르겠어요. 누구한테 말해야 할까요?

## 버그의 종류별 담당자

### 1. 화면이 이상해요 → 프론트엔드

**이런 경우:**
- 버튼 위치가 이상해요
- 색상이 디자인과 달라요
- 레이아웃이 깨져요
- 애니메이션이 안 돼요

**담당:** 프론트엔드 개발자

### 2. 데이터가 이상해요 → 백엔드

**이런 경우:**
- 저장한 내용이 안 불러와져요
- 결제했는데 기록이 없어요
- 목록이 안 나와요
- 로그인이 안 돼요

**담당:** 백엔드 개발자

### 3. 기능 자체가 이상해요 → PM/기획자

**이런 경우:**
- 기획서에 있는 버튼이 없어요
- 플로우가 기획과 달라요
- 이 기능이 있어야 하는지 모르겠어요
- 요구사항이 불명확해요

**담당:** PM 또는 기획자

### 4. 앱이 죽어요/느려요 → 개발팀 전체

**이런 경우:**
- 앱이 강제 종료돼요
- 화면이 멈춰요
- 로딩이 너무 오래 걸려요
- 서버가 응답 안 해요

**담당:** 개발팀 전체 (원인 파악 필요)

## 모르겠으면 일단 QA팀에게

:::callout[tip]
담당을 모르겠으면 QA팀이나 PM에게 말하세요.
그들이 적절한 담당자에게 전달해줄 거예요.
:::

**전달 경로:**
```
발견자 → QA팀/PM → 담당 개발자
```

## 버그 리포트 잘 쓰는 법

### 필수 정보

1. **무엇이** 문제인가요?
2. **어디서** 발생했나요? (화면, 기능)
3. **어떻게** 재현하나요? (단계별)
4. **환경은** 어떤가요? (기기, 브라우저, OS)

### 좋은 버그 리포트 예시

```
❌ 안 좋은 예:
"로그인이 안 돼요"

✅ 좋은 예:
[현상] 로그인 버튼 클릭 시 에러 메시지 표시
[재현 방법]
1. 로그인 화면 진입
2. 이메일/비밀번호 입력
3. 로그인 버튼 클릭
4. "서버 오류" 메시지 표시
[환경] iPhone 15, iOS 17, Safari
[비고] Chrome에서는 정상 동작
```

### 스크린샷/녹화 첨부

==백 마디 말보다 한 장의 스크린샷이 낫습니다.==

- 문제 상황 캡처
- 가능하면 화면 녹화
- 에러 메시지 전체 캡처

## 버그 추적 도구

### 회사에서 쓰는 도구들

| 도구 | 특징 |
|------|------|
| Jira | 대기업에서 많이 사용 |
| Linear | 스타트업에서 인기 |
| Asana | PM 도구와 통합 |
| Notion | 간단한 팀에서 사용 |
| Slack | 긴급 버그 공유 |

### 어디에 올려야 할까?

팀마다 달라요. 입사할 때 물어보세요:
- "버그 리포트는 어디에 올려요?"
- "긴급 버그는 어떻게 공유해요?"

## 긴급도 판단

### 지금 당장! (Critical)

- 서비스 전체가 안 됨
- 결제가 안 됨
- 개인정보 노출
- 데이터 손실

→ **슬랙으로 바로 알리기**

### 빨리 (High)

- 핵심 기능 장애
- 많은 사용자 영향

→ **버그 트래커 + 담당자 멘션**

### 여유 있게 (Medium/Low)

- 사소한 UI 문제
- 드물게 발생
- 우회 방법 있음

→ **버그 트래커에 등록**

## 정리

:::callout[important]
버그를 발견하면 증상을 보고 담당을 추측해보고,
모르겠으면 QA팀이나 PM에게 전달하세요.
스크린샷과 재현 방법을 포함하면 해결이 빨라집니다.
:::

**핵심:**
- 화면 문제 → 프론트엔드
- 데이터 문제 → 백엔드
- 기능 문제 → PM/기획자
- 모르겠으면 → QA팀/PM
- 잘 정리된 버그 리포트가 해결을 빠르게 함
