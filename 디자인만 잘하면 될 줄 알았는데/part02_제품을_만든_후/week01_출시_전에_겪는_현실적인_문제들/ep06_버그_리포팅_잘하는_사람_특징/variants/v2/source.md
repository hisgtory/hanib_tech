# 버그 리포팅 잘하는 사람 특징 - 실무 체크리스트 버전

<!-- variant: v2, type: 실무 체크리스트 버전 -->

## 핵심 요약

좋은 버그 리포트는 개발자가 바로 문제를 재현하고 수정할 수 있게 한다. 필수 정보를 빠짐없이, 정확하게 전달하는 것이 핵심이다.

---

## 버그 리포트 필수 항목 체크리스트

### 기본 정보 (필수)
- [ ] 버그 제목 (한 줄 요약)
- [ ] 발견 일시
- [ ] 발견자
- [ ] 심각도 (Critical / High / Medium / Low)

### 환경 정보 (필수)
- [ ] 기기 (iPhone 14 Pro, Galaxy S23 등)
- [ ] OS 버전 (iOS 17.1, Android 14 등)
- [ ] 앱/웹 버전
- [ ] 브라우저 (웹인 경우)
- [ ] 네트워크 환경

### 재현 정보 (필수)
- [ ] 재현 단계 (Step by step)
- [ ] 기대 결과
- [ ] 실제 결과
- [ ] 재현율 (항상/가끔/1회)

### 첨부 자료 (권장)
- [ ] 스크린샷
- [ ] 화면 녹화
- [ ] 에러 메시지 전문
- [ ] 콘솔 로그

---

## 버그 심각도 분류 기준

| 심각도 | 기준 | 예시 |
|--------|------|------|
| **Critical** | 서비스 이용 불가 | 앱 크래시, 결제 실패, 데이터 손실 |
| **High** | 주요 기능 장애 | 로그인 불가, 핵심 기능 오작동 |
| **Medium** | 기능 일부 장애 | 특정 조건에서 버튼 미작동 |
| **Low** | 사소한 문제 | UI 깨짐, 오타, 미세한 정렬 |

## 좋은 버그 리포트 예시

### Before (나쁜 예)
```
제목: 로그인 안 됨
내용: 로그인이 안 돼요
```

### After (좋은 예)
```
제목: [iOS] 소셜 로그인 - 카카오 로그인 시 무한 로딩

환경:
- iPhone 14 Pro, iOS 17.1
- 앱 버전 2.3.1
- WiFi 연결

재현 단계:
1. 앱 실행
2. 로그인 화면에서 "카카오로 시작하기" 탭
3. 카카오 앱으로 전환 후 로그인 완료
4. 다시 앱으로 돌아옴

기대 결과:
- 메인 화면으로 이동

실제 결과:
- 로딩 스피너가 계속 돌아감 (1분 이상)
- 에러 메시지 없음

재현율: 10회 중 8회 발생

첨부: 화면 녹화 영상

비고: 일반 이메일 로그인은 정상 작동
```

## 버그 리포트 작성 체크리스트

### 작성 전 확인
- [ ] 이미 보고된 버그인지 확인했는가?
- [ ] 최신 버전에서도 발생하는가?
- [ ] 캐시/쿠키 삭제 후에도 발생하는가?
- [ ] 다른 기기에서도 발생하는가?

### 작성 시 주의
- [ ] 한 리포트에 한 버그만 담았는가?
- [ ] 주관적 표현 대신 객관적 사실을 적었는가?
- [ ] 재현 단계가 누락 없이 순서대로 적혔는가?
- [ ] 스크린샷에 민감 정보는 가렸는가?

### 작성 후 확인
- [ ] 다른 사람이 이 리포트만 보고 재현할 수 있는가?
- [ ] 담당자에게 알림을 보냈는가?
- [ ] 우선순위 태그를 달았는가?

## 버그 분류 체크리스트

### 버그 유형
- [ ] 기능 버그 (예상대로 동작 안 함)
- [ ] UI 버그 (디자인과 다름)
- [ ] 성능 버그 (느림, 멈춤)
- [ ] 호환성 버그 (특정 환경에서만 발생)
- [ ] 보안 버그 (데이터 노출, 권한 문제)

### 발생 조건
- [ ] 항상 발생
- [ ] 특정 조건에서 발생
- [ ] 간헐적 발생
- [ ] 1회성 발생

## 버그 추적 워크플로우

### 상태 정의
| 상태 | 의미 |
|------|------|
| Open | 새로 등록됨 |
| In Progress | 수정 중 |
| In Review | 수정 완료, 검토 중 |
| QA | QA 테스트 중 |
| Resolved | 수정 완료 |
| Closed | 확인 완료 |
| Won't Fix | 수정 안 함 (의도된 동작) |

### 담당자별 역할
- **발견자**: 리포트 작성, 재현 확인
- **PM**: 우선순위 결정, 일정 조율
- **개발자**: 원인 분석, 수정
- **QA**: 수정 확인, 회귀 테스트

## 실무 팁

:::callout[tip]
**버그 리포트 템플릿을 만들어두세요**

팀에서 공통으로 사용할 템플릿을 만들어두면:
- 필수 정보 누락 방지
- 리포트 품질 일관성 유지
- 신규 입사자도 바로 사용 가능
:::

:::callout[warning]
**감정적 표현은 피하세요**

"이게 말이 돼요?", "또 이러네요" 같은 표현은:
- 문제 해결에 도움이 안 됨
- 팀 분위기 저하
- 객관적 판단 방해

→ 사실만 담담하게 적으세요
:::
