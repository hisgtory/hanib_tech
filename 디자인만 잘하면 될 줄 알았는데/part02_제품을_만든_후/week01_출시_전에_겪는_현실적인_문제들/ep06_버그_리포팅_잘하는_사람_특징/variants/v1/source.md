# 버그 리포팅 잘하는 사람 특징

## 스토리텔링 버전

:::conversation
**N**: 한입님, 저 버그 발견했는데 개발자분이 재현이 안 된다고 하셔요...

**한입**: 어떻게 리포트하셨어요?

**N**: "결제 버튼 눌렀는데 안 돼요"라고 했어요.

**한입**: 아... 그러면 개발자분이 찾기 어려우실 거예요. 범인 잡으려면 목격자 진술이 정확해야 하잖아요!

**N**: 그럼 어떻게 말해야 해요?
:::

### 좋은 버그 리포트의 구조

:::conversation
**한입**: 버그 리포트는 이 순서로 작성하면 좋아요.

1. **뭘 하려고 했는지** (기대 동작)
2. **실제로 뭐가 됐는지** (실제 동작)
3. **어떤 순서로 했는지** (재현 단계)
4. **어떤 환경이었는지** (환경 정보)

**N**: 예시로 보여주실 수 있어요?
:::

**나쁜 버그 리포트:**
```
결제 안 돼요
```

**좋은 버그 리포트:**
```
[현상]
결제 버튼 클릭 시 무한 로딩 발생

[재현 방법]
1. 상품 상세 페이지 진입
2. "장바구니 담기" 클릭
3. 장바구니에서 "결제하기" 클릭
4. 결제 수단 "카카오페이" 선택
5. "결제하기" 버튼 클릭
→ 무한 로딩 (5분 이상 대기해도 진행 안 됨)

[환경]
- 기기: iPhone 14 Pro
- OS: iOS 17.2
- 앱 버전: 2.3.1
- 네트워크: WiFi

[첨부]
- 스크린샷 1장
- 화면 녹화 영상
```

### 재현 단계가 제일 중요해요

:::conversation
**한입**: 개발자가 버그를 고치려면 먼저 그 버그를 직접 봐야 해요. 재현이 안 되면 고칠 수가 없어요.

**N**: 그럼 어떻게 재현 단계를 잘 쓸 수 있어요?

**한입**: 처음부터 다시 해보면서 모든 클릭을 기록하세요. 당연한 것도요!

예를 들어 "로그인 후"가 아니라:
1. 앱 실행
2. "로그인" 버튼 클릭
3. 이메일 입력: test@example.com
4. 비밀번호 입력: ****
5. "로그인" 버튼 클릭

이렇게 상세하게요!
:::

### 환경 정보도 꼭 포함하세요

| 항목 | 왜 중요한지 |
|------|-------------|
| 기기 | 특정 기기에서만 발생할 수 있어요 |
| OS 버전 | 버전마다 동작이 달라요 |
| 앱 버전 | 이미 고쳐진 버전일 수 있어요 |
| 브라우저 | 크롬에서만 발생하는 버그도 있어요 |
| 네트워크 | 느린 네트워크에서만 발생할 수 있어요 |

:::callout[tip]
스크린샷보다 화면 녹화가 10배 유용해요!
특히 "가끔 발생하는" 버그는 영상이 필수예요.
:::

### 프로처럼 리포트하기

:::conversation
**N**: 더 프로처럼 하고 싶은데요?

**한입**: 몇 가지 팁을 더 드릴게요!

1. **항상 발생 vs 가끔 발생** - "10번 중 3번 정도 발생"처럼 빈도를 알려주세요
2. **언제부터 발생** - "오늘 업데이트 후부터" 같은 정보가 있으면 범위가 좁혀져요
3. **비교 정보** - "아이폰에서는 되는데 갤럭시에서 안 돼요" 같은 비교도 유용해요
4. **콘솔 에러** - 개발자 도구 열어서 에러 메시지 캡처해주면 금상첨화!
:::

### 좋은 버그 리포터 = 좋은 협업자

:::callout[important]
개발자 입장에서 좋은 버그 리포트를 받으면:
- 원인 파악 시간 50% 감소
- 불필요한 질문 왕복 없음
- 빠른 수정 → 빠른 배포

결국 팀 전체의 시간을 아끼는 거예요!
:::

이 세 가지면 개발자가 10배 빨리 고칠 수 있어요!
:::
