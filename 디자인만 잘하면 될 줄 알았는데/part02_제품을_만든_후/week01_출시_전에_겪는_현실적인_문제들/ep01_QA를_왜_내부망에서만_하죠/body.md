# QA를 왜 내부망에서만 하죠

## "왜 회사에서만 테스트해요?"

"QA는 VPN 연결해야 해요." "내부망에서만 볼 수 있어요." 왜 집에서 편하게 확인하면 안 될까요?

## QA가 뭔가요?

QA는 **Quality Assurance**의 약자예요. 말 그대로 "품질 보증"이에요.

출시 전에 버그가 없는지, 기획대로 동작하는지 확인하는 과정이에요.

## 왜 내부망에서만 할까?

### 1. 아직 완성되지 않은 제품이라서

QA 중인 제품은:
- 버그가 있을 수 있어요
- 테스트 데이터가 들어있어요
- 아직 공개하면 안 되는 기능이 있어요

**외부에 노출되면:**
- 경쟁사가 미리 볼 수 있음
- 사용자가 버그 있는 제품을 봄
- 테스트 데이터가 유출됨

### 2. 보안 때문에

:::callout[warning]
QA 서버에는 민감한 정보가 있을 수 있어요.
:::

- 테스트용 결제 정보
- 가짜 사용자 데이터
- 개발 중인 신규 기능

이런 정보가 외부로 나가면 보안 문제가 생겨요.

### 3. 환경 분리가 필요해서

**서버 환경의 종류:**

| 환경 | 용도 | 접근 범위 |
|------|------|----------|
| 개발(Dev) | 개발자 작업용 | 개발자만 |
| QA/스테이징 | 테스트용 | 내부 직원 |
| 운영(Production) | 실제 서비스 | 모든 사용자 |

환경을 분리해야 테스트 중 실수가 실제 서비스에 영향을 안 줘요.

## 내부망이란?

### VPN 없이는 접근 불가

```
외부 인터넷 ← 방화벽 → 내부망(회사 네트워크)
                         ↑
                      QA 서버
```

내부망은 회사 건물 안이거나, VPN으로 연결해야 접근할 수 있어요.

### 왜 이렇게 복잡하게?

- 해커가 쉽게 접근 못하게
- 내부 정보 유출 방지
- 테스트 환경과 실제 환경 분리

## 디자이너가 QA에 참여할 때

### 준비물

1. **VPN 설정**: IT팀에 요청
2. **QA 서버 주소**: PM이나 개발팀에게 받기
3. **테스트 계정**: 별도의 테스트용 계정

### 주의사항

==QA 환경에서 본 것은 아직 출시 전이에요.==

- 스크린샷 외부 공유 금지
- SNS에 올리지 않기
- 경쟁사에 정보 유출 주의

### QA에서 확인할 것들

**디자이너 관점:**
- 디자인대로 구현되었는지
- 여백, 색상, 폰트 등 스타일
- 반응형 레이아웃
- 애니메이션 동작
- 에러 화면, 빈 화면

## 자주 겪는 상황

### "집에서 QA 확인하고 싶어요"

```
방법 1: VPN 연결
방법 2: 회사로 출근
```

VPN 설정이 안 되어 있다면 IT팀에 요청하세요.

### "QA 서버가 안 들어가져요"

**확인할 것들:**
- VPN 연결 상태
- QA 서버 점검 중인지
- URL이 정확한지

### "QA에서 본 거랑 실서버가 달라요"

배포 전이라 그래요. QA에서 확인한 내용이 실서버에 반영되려면 배포가 필요해요.

## 정리

:::callout[important]
QA를 내부망에서만 하는 이유는 보안과 환경 분리 때문이에요.
미완성 제품, 테스트 데이터, 신규 기능을 보호하기 위함입니다.
:::

**핵심:**
- QA = 출시 전 품질 검증
- 내부망 = 회사 내부 네트워크 (VPN 필요)
- 보안과 환경 분리를 위해 제한됨
- 디자이너도 VPN 설정하면 QA 참여 가능
