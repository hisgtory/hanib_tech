# HTTP Status Codes - hanib_tech 콘텐츠 시리즈

> *"200, 404, 500, ... HTTP 응답 모음.zip"*

## 🎯 콘텐츠 개요

이 시리즈는 개발자가 꼭 알아야 할 HTTP 상태 코드들을 친근하고 이해하기 쉽게 설명한 한입씨 테크의 대표적인 교육 콘텐츠입니다. 총 9개의 카드로 구성되어 있으며, 각 카드는 특정 상태 코드에 대한 명확한 설명과 실제 사용 예시를 제공합니다.

## 📚 콘텐츠 구성 (9개 카드)

### 1️⃣ 인트로: HTTP Status Code 컬렉션
**메인 메시지**: "200, 404, 500, ... HTTP 응답 모음.zip"

**핵심 내용**:
- 404 Not found
- 400 Bad Request  
- 200 OK
- 201 Created

**스토리텔링 방식**: 
- `.zip` 파일 메타포를 사용하여 호기심 유발
- 익숙한 디지털 용어로 친근함 조성
- 검은 배경에 노란색/흰색 텍스트로 시각적 임팩트 극대화

---

### 2️⃣ HTTP Status Code란?
**메인 메시지**: "우리가 알고있는 404, 500 등의 코드를 HTTP Status Code라고 해요"

**핵심 내용**:
- HTTP를 통한 서버-클라이언트 통신 과정 설명
- 서버가 데이터와 함께 status code 전달
- 요청(Request) → 응답(Response) 구조

**시각적 요소**:
```
[클라이언트 N] ←→ [서버]
    요청 →     게시글, 이미지 등등 데이터
    ← 응답     데이터 + status code: 200
```

**교육적 접근**:
- 캐릭터 N을 활용한 친근한 설명
- 구체적인 예시 (게시글, 이미지)
- 통신 과정의 시각적 도식화

---

### 3️⃣ 200 OK 👍
**메인 메시지**: "요청에 대해 성공적으로 응답하였다는 뜻이에요"

**핵심 내용**:
- 성공적인 요청 처리를 의미
- 클라이언트 반응: "잘 처리됐구나😋"
- UI 렌더링 또는 안내 문구 노출 후 다음 단계 진행

**실제 사용 예시**:
```
[게시물을 성공적으로 수정했어요]
```

**톤앤매너**:
- 긍정적이고 친근한 톤
- 이모지 활용으로 감정 표현
- 사용자 경험 관점에서의 설명

---

### 4️⃣ 201 Created ✨
**메인 메시지**: "웹 서버가 성공적으로 새로운 리소스를 생성했음을 나타내요"

**핵심 내용**:
- 새로운 리소스 생성 성공
- 게시물 작성, 댓글 작성 시 사용
- 200과 구별되는 특별한 의미

**실제 사용 예시**:
```
[댓글을 작성했습니다.]
```

**차별점**:
- 단순 성공(200)과 생성 성공(201)의 구분
- 구체적인 사용 케이스 제시
- 반짝임 이모지(✨)로 '새로 만들어짐' 강조

---

### 5️⃣ 400 Bad Request 🧯
**메인 메시지**: "처리할 수 없는 잘못된 요청에 대해 400으로 응답해요"

**핵심 내용**:
- 클라이언트 측 요청 오류
- 구체적 예시:
  - 시작일이 종료일보다 큰 경우
  - 최대 글자 수 초과
- 서버의 수정 요청 안내 + 클라이언트의 화면 렌더링

**실제 사용 예시**:
```
EMAIL 입력 필드: hanib_tech@123456.....
ERROR: 올바른 이메일 주소를 입력해주세요.
```

**교육적 가치**:
- 에러의 원인과 해결 방안 제시
- 사용자 인터페이스와 연결된 설명
- 개발자-사용자 관점 모두 고려

---

### 6️⃣ 403 Forbidden 🚫
**메인 메시지**: "권한 없음을 뜻해요"

**핵심 내용**:
- 권한 부족으로 인한 접근 차단
- 일상적 경험과 연결: "URL은 공유 받았는데, 클릭해보니 권한이 없으니 요청하라는 화면"
- 조회 권한, 삭제 권한 등 다양한 상황

**실제 사용 예시**:
```
Google 프레젠테이션
액세스 권한 필요
[열쇠구멍 이미지]
메일(선택사항): [    ]
[액세스 요청] 버튼
```

**스토리텔링 특징**:
- 모든 사람이 경험해본 상황으로 설명
- Google Drive 예시로 친숙함 조성
- 실제 UI 스크린샷 활용

---

### 7️⃣ 404 Not Found ❌  
**메인 메시지**: "URL이 잘못되어 존재하지 않는 페이지거나 리소스가 삭제되어 없는 경우예요"

**핵심 내용**:
- 가장 널리 알려진 에러 코드
- 발생 원인:
  - 잘못된 URL
  - 삭제된 리소스
- 보안적 활용: 403 대신 404로 페이지 존재 여부 숨김

**실제 사용 예시**:
```
Notion
페이지를 찾을 수 없음
이 페이지가 존재하지 않거나 페이지에 대한 
액세스 권한이 없습니다.
[내 콘텐츠로 돌아가기]
```

**고급 팁**:
- 보안/프라이버시를 위한 404 활용법
- 403 vs 404의 전략적 선택

---

### 8️⃣ 500 Internal Server Error 🧯
**메인 메시지**: "서버에서 에러가 발생한 경우 실행중이던 코드가 더 이상 진행되지 않고 500을 응답해요"

**핵심 내용**:
- 서버 측 예상치 못한 오류
- 개발자들이 의도하지 않은 응답 코드
- 개발자들의 노력: 500 에러 발생률 최소화

**인터랙티브 요소**:
- "500뿐만 아니라 501, 502 등 5xx 응답 코드를 발견하게 된다면 저에게 알려주세요!"
- 캐릭터 한입이 "..." 말풍선으로 대기 중

**차별점**:
- 다른 코드들과의 본질적 차이점 강조
- 개발자 관점에서의 해석
- 커뮤니티 참여 유도

---

### 9️⃣ 마무리: 실용적 관점
**메인 메시지**: "설명하지 못한 코드가 아직 많이 남아있어요!"

**핵심 내용**:
- 프로토콜 명시 vs 실제 사용의 차이
- 개발팀 간 협의를 통한 코드 선택
- 실무에서는 가장 흔히 사용되는 코드들 위주

**실무적 통찰**:
```
캐릭터 한입: "우리팀은 200, 404만 사용할까요? ^^"
```

**교육적 마무리**:
- 이론과 실무의 균형
- 팀 단위의 의사결정 중요성
- 실용주의적 접근법 강조
