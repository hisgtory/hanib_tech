# 1-1. 우리팀을 소개합니다

소프트웨어 제품을 만들기 위한 최소 인원으로 구성된 팀은 다음과 같아요.

- **Product Owner**: 기획자라고도 불리는 PO는 제품의 방향성을 정의하고 상세한 내용을 기획해요. 문제 정의, 가설, 해결책 제시, 구체적인 정책, 가설 검증 지표 정의 등을 해요. 노션, 컨플루언스를 주로 사용해요.

- **Product Designer**: UX를 고려해서 PO의 기획서를 바탕으로 앱의 UI를 만들어요. 이전에는 기획자가 와이어프레임까지 다 잡아서 전달했다면, 요즘은 PD가 문서로 구성된 기획서를 바탕으로 와이어프레임부터 설계해서 실제 UI까지 만들어요. 피그마를 주로 사용해요.

- **웹 프론트엔드 개발자**: 우리 제품에 웹사이트의 UI를 구성하고 서버와 데이터를 주고 받아 온라인 기능을 구현 해요. 키워드: HTML,CSS,Javascript,Typescript,React

- **앱 개발자**: 우리 제품의 앱을 개발해요. 웹 프론트엔드와 동일하게 UI를 구성하고 서버로부터 데이터를 받아와 온라인 기능을 구현해요. 안드로이드와 iOS의 개발은 꽤나 달라서 분야가 구분되어 있고 다른 플랫폼의 앱 개발을 위해서는 학습 시간이 필요해요. 키워드: Kotlin, Java, Swift

- **서버 개발자**: UI에 필요한 데이터를 DB에 저장하고, 꺼내볼 수 있게 하는 기능을 제공해요. 흔히 말하는 '알고리즘'을 구현해요.

물론, 당연하게도 팀의 구성은 유동적이에요. 제품과 상황에 따라 모든 파트가 존재하지 않기도 해요.

우리 제품이 앱기반이라면 웹 프론트엔드 개발자가 없을 수 있어요.

커뮤니티와 같은 온라인 기능이 없이 혼자 사용하는 간단한 일기앱이라면 서버 개발자 없이 개발하기도 해요.

<callout>
이 최소한의 인원 구성으로 이뤄진 팀을 특공대라는 뜻의 '스쿼드' 라고 해요.

초기 스타트업의 구성은 당연히 이렇겠지만, 스타트업이 커져도 이런 최소한의 구성으로 된 스쿼드를 여러개 만들어서 각각이 더 작은 영역에서 깊은 몰입을 통한 개선을 하도록 만들어요.
</callout>

개발팀, 기획팀, 디자인팀으로 구성하는 전통적인 방식보다는검색실 > 검색 스쿼드, 검색 결과 스쿼드를 만들어서 해당 스쿼드에 각 파트의 구성원들을 채워넣어요.

거기에 왜 그렇게 많은 사람이 필요해?라고 생각할 수도 있는데요.

하나의 주제와 목표에 대해 깊은 몰입과 논의가

검색 스쿼드에서 자동완성 영역에 키워드 뿐만 아니라 상품, 광고, 아티클에 대한 추천을 할수도 있고, 검색 자체에 AI를 도입하는 시도도 할 수 있죠.

각 스쿼드는 각자의 목표를 달성하기 위해 가설을 세우고, 실험을 하고, 지표를 통해 배우고, 새로운 시도를 하는 형태로 일해요.
