# Episode 22-1: 창업, 디자이너가 만드는 유니콘의 비밀

## 🎬 Scene: 퇴사를 결심한 순간

```
금요일 밤 11시, 또 다시 무의미한 수정 요청

상사: "뭔가 2% 부족해요. 다시 해주세요."
디자이너: (속으로) "내가 만들고 싶은 건 이게 아닌데..."

집으로 가는 지하철에서:
"내가 직접 만들면 어떨까?"
"Airbnb도 디자이너가 창업했잖아?"
"나도 할 수 있을 것 같은데..."

6개월 후:
- 월 매출 3,000만원
- MAU 10만명
- 시드 투자 5억원
- 팀원 5명

"회사 다닐 때보다 100배 힘들지만
 100배 행복해요. 내 제품이니까요."
```

**디자이너 창업자는 특별한 무기를 가지고 있습니다.**
**사용자 경험을 설계하는 능력, 그것이 바로 경쟁력입니다.**

## Part 1: 디자이너 창업의 강점과 약점

### 💪 디자이너의 창업 강점

```
강점 1: 사용자 중심 사고
- 문제 정의 능력
- 공감 능력
- 솔루션 시각화

강점 2: 빠른 프로토타이핑
- MVP 직접 제작
- 빠른 검증
- 비용 절감

강점 3: 브랜딩 능력
- 차별화된 아이덴티티
- 일관된 경험
- 감성적 연결

강점 4: 스토리텔링
- 비전 전달
- 투자자 설득
- 팀 모집

성공 사례:
Airbnb (Brian Chesky) - RISD 출신
Pinterest (Evan Sharp) - 건축 디자인
Canva (Melanie Perkins) - 그래픽 디자이너
Kickstarter (Charles Adler) - 디자이너
```

### 😰 극복해야 할 약점

```
약점 & 해결책:

1. 기술 구현력
문제: "개발을 못해요"
해결: 
- No-code 도구 활용
- 기술 공동창업자 찾기
- 아웃소싱 → 인하우스

2. 비즈니스 이해
문제: "숫자를 몰라요"
해결:
- 기초 회계 공부
- 멘토 찾기
- 온라인 MBA

3. 영업/마케팅
문제: "세일즈가 어색해요"
해결:
- 제품이 스스로 팔리게
- 콘텐츠 마케팅
- 커뮤니티 구축

4. 의사결정 속도
문제: "완벽주의 성향"
해결:
- 70% 룰
- 빠른 실행과 개선
- Done is better than perfect
```

## Part 2: 아이디어에서 제품까지

### 💡 아이디어 검증 프로세스

```
Step 1: 문제 정의 (1주)
"무엇을 해결하려고 하는가?"

도구: Problem Canvas
┌─────────────────────────┐
│ 문제: 포트폴리오 관리 어려움 │
│ 타겟: 주니어 디자이너      │
│ 현재 솔루션: PDF, 웹사이트  │
│ 페인포인트: 업데이트 번거로움│
└─────────────────────────┘

Step 2: 솔루션 스케치 (1주)
"어떻게 해결할 것인가?"
- 10개 아이디어
- 3개로 압축
- 1개 선택

Step 3: 프로토타입 (2주)
Figma → 클릭 가능한 프로토타입
목표: 핵심 기능만 구현

Step 4: 사용자 테스트 (1주)
- 타겟 10명 인터뷰
- 페인포인트 확인
- 지불 의사 확인

Step 5: Go/No-Go 결정
✅ Go 신호:
- 7/10명 이상 "꼭 필요"
- 5/10명 이상 "돈 낼 의향"
- 명확한 차별점
```

### 🚀 MVP 제작 전략

```
디자이너의 MVP 도구:

No-Code 스택:
Frontend: Framer, Webflow
Backend: Bubble, Xano
Database: Airtable
Payment: Stripe, Gumroad
Analytics: Google Analytics

2주 MVP 플랜:
Day 1-3: 디자인 (Figma)
Day 4-7: 개발 (No-code)
Day 8-10: 테스트
Day 11-12: 결제 연동
Day 13-14: 런칭 준비

비용: 50만원 이하
시간: 80시간
결과: 작동하는 제품
```

```

- 페이스북 그룹

- 디스코드
- 오픈 카톡

3. 콘텐츠 마케팅

SEO 키워드:
- 롱테일 키워드
- 문제 중심 검색어
- How-to 콘텐츠
```

## Part 4: 투자 유치

### 💸 시드 투자 준비

```
투자 단계별 준비:

Pre-Seed (1-5억원):
- 팀 (공동창업자)
- 프로토타입
- 초기 고객 (100명+)
- 6개월 런웨이

Seed (5-20억원):
- PMF 징후
- MRR $5,000+
- 성장률 20%+/월
- 12개월 계획

IR Deck 구조 (10장):
1. 문제 (Pain Point)
2. 솔루션
3. 시장 규모 (TAM/SAM/SOM)
4. 제품 (데모)
5. 비즈니스 모델
6. 경쟁 우위
7. 팀
8. Traction
9. Go-to-Market
10. Ask & Use of Funds
```

### 🎯 투자자 미팅

```
투자자가 보는 것:

1. Founder-Market Fit
"왜 당신이 이 문제를 해결해야 하나?"
→ 개인 스토리 + 도메인 전문성

2. 시장 크기
"얼마나 큰 회사가 될 수 있나?"
→ TAM $10B+ 증명

3. 차별화
"왜 이기나?"
→ 디자인 = 해자

4. 팀
"함께 할 수 있나?"
→ 공동창업자 필수

5. Traction
"증거가 있나?"
→ 매출 or 사용자 성장

피칭 팁:
- 3분 안에 핵심 전달
- 숫자로 증명
- 데모 필수
- 비전 제시
```

## Part 5: 팀 빌딩

### 👥 공동창업자 찾기

```
이상적인 조합:

디자이너 (당신) + 개발자 + 비즈니스
= 완벽한 삼각형

찾는 방법:
1. 네트워크 활용
   - 전 직장 동료
   - 학교 동문
   - 커뮤니티

2. 플랫폼
   - CoFoundersLab
   - AngelList
   - Y Combinator

3. 해커톤/밋업
   - 스타트업 위크엔드
   - 해커톤 참가
   - 네트워킹 이벤트

지분 배분:
균등 분배 원칙
CEO: 33.4%
CTO: 33.3%
CDO: 33.3%

Vesting: 4년, 1년 cliff
```

### 🎨 초기 팀 구성

```
첫 10명 채용 순서:

1-2. 공동창업자
3. 풀스택 개발자
4. 마케터
5. 프로덕트 매니저
6. 시니어 개발자
7. 디자이너
8. 세일즈
9. CS 매니저
10. 데이터 분석가

채용 팁:
- 문화 핏 > 실력
- 주니어 > 시니어 (초기)
- 열정 > 경력
- 스톡옵션 활용

문화 만들기:
- 미션/비전 명확히
- 핵심 가치 정의
- 투명한 소통
- 빠른 실행
- 실패 허용
```

## Part 6: 생존과 성장

### 📈 성장 지표 관리

```
핵심 지표 대시보드:

North Star Metric:
MAU or MRR (비즈니스 모델에 따라)

Weekly 추적:
- 신규 가입
- 활성 사용자
- 이탈률
- NPS
- 매출

월간 분석:
- CAC (고객 획득 비용)
- LTV (생애 가치)
- Burn Rate
- Runway
- Unit Economics

건강한 지표:
- MoM Growth: 20%+
- Churn: <5%
- LTV/CAC: >3
- NPS: >30
```

```
피벗 신호:

- 자금 3개월 이하
- 팀 이탈

피벗 유형:
1. Customer Pivot (고객 변경)
2. Problem Pivot (문제 변경)
3. Solution Pivot (솔루션 변경)
4. Revenue Model Pivot

Slack: 게임 회사 → 협업 툴
Pinterest: 쇼핑 앱 → 영감 보드

피벗 프로세스:
1. 데이터 분석
```
