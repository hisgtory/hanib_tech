# Episode 3-6: React를 빼고 개발하면 무슨 일이 일어날까?

React는 현재 프론트엔드 개발의 대표 주자로 자리 잡았지만, React 없이도 훌륭한 웹 애플리케이션을 만들 수 있습니다. 이번 에피소드에서는 React를 사용하지 않고 개발할 때 어떤 일이 일어나는지, 그리고 어떤 대안들이 있는지 살펴보겠습니다.

## 🎯 왜 이런 질문을 해야 할까요?

프론트엔드 개발자라면 누구나 한 번쯤 "React가 정말 필수일까?"라는 의문을 품어봤을 것입니다. 특히 간단한 프로젝트를 시작할 때, 복잡한 빌드 시스템과 수많은 의존성을 보며 "이게 정말 필요한가?"라는 생각이 들 때가 있죠.

## 1. Vanilla JavaScript로 돌아가기

### 장점: 순수함의 매력 ✨

- 빌드 스텝 없음: 파일을 만들고 바로 브라우저에서 열면 끝! 하루종일 webpack과 씨름할 필요 없습니다.
- 학습 곡선 완화: JavaScript만 알면 바로 시작 가능! JSX, hooks, 상태 관리 패턴 등을 배울 필요가 없습니다.
- 가벼운 파일 크기: React + ReactDOM은 기본적으로 40KB 정도. 작은 프로젝트에는 부담이 될 수 있습니다.
- 완전한 통제: 모든 것을 직접 다룰 수 있어 세밀한 조정이 가능합니다.

### 단점: 현실의 벽 😓

- DOM 조작의 복잡성: document.createElement, appendChild... 이런 코드를 반복하다 보면 React의 소중함을 깨닫게 됩니다.
- 상태 관리의 악몽: 복잡한 상태를 관리하려면 직접 시스템을 구축해야 합니다.
- 콤포넌트 재사용성: 코드를 모듈화하고 재사용하기 어렵습니다.
- 테스트의 어려움: UI 테스트가 React보다 훨씬 복잡해집니다.

## 2. 대안 프레임워크들의 세계 🌐

### Vue.js: 부드러운 진입 장벽 🌱

Vue는 React와 Angular의 장점을 결합한 프레임워크로, 특히 초보자에게 친화적입니다.

- 템플릿 기반 문법: HTML과 비슷해 직관적
- 점진적 도입 가능: 기존 프로젝트에 부분적으로 적용 가능
- 풍부한 생태계: Vue Router, Vuex 등 공식 라이브러리

### Svelte: 컴파일 타임의 마법 ✨

Svelte는 빌드 타임에 모든 것을 컴파일해서 가장 빠른 성능을 자랑합니다.

- 가상 DOM 없음: 런타임 오버헤드가 거의 없음
- 훨씬 작은 번들 크기: React의 1/10 수준
- 내장된 상태 관리: 복잡한 설정 없이 바로 사용

### Angular: 대규모 어플리케이션의 친구 🏗️

Google이 만든 Angular는 대규모 엔터프라이즈 애플리케이션에 최적화되어 있습니다.

- 완전한 프레임워크: 라우팅, HTTP 클라이언트, 폼 처리 등 모든 것이 포함
- TypeScript 기본 지원: 타입 안정성으로 대규모 프로젝트에 유리
- 강력한 CLI: 코드 생성, 테스트, 빌드 등을 자동화

## 3. 진보된 접근법들 🚀

### Web Components: 미래의 표준 🌐

Web Components는 프레임워크에 의존하지 않는 웹 표준 기술입니다.

- Custom Elements: 나만의 HTML 태그 만들기
- Shadow DOM: 스타일과 마크업 캡슐화
- 프레임워크 중립적: 어떤 프레임워크와도 함께 사용 가능

### Static Site Generators (SSG): 빠른 성능의 비밀 ⚡

Jekyll, Hugo, Eleventy 같은 SSG는 빌드 타임에 HTML을 생성합니다.

- 최고의 성능: 서버 사이드 렌더링 없음
- 보안성: 서버 코드가 없어 해킹 위험 감소
- SEO 친화적: 모든 콘텐츠가 HTML로 제공

## 4. 언제 React를 포기해야 할까? 🤔

### React가 과한 경우들:

- 📝 간단한 랜딩 페이지나 정적 사이트
- 🎨 프로토타입이나 실험적인 프로젝트
- 💰 예산이 부족하고 빠른 개발이 필요한 경우
- 📉 성능이 극도로 중요한 경우

### React가 필수인 경우들:

- 🛠️ 복잡한 상태 관리가 필요한 대규모 SPA
- 👥 큰 팀에서 협업해야 하는 프로젝트
- 🔄 콤포넌트 재사용이 중요한 경우
- 🌐 React Native로 모바일 앱도 개발할 계획

## 5. 실전 팁: React 없이 개발하기 💡

### 1. 모듈 시스템 활용하기

ES6 모듈을 사용해 코드를 조직화하세요. Webpack이나 Rollup 같은 번들러를 사용하면 React 없이도 모듈화된 코드를 작성할 수 있습니다.

### 2. 템플릿 엔진 고려하기

Handlebars, Mustache 같은 템플릿 엔진은 서버 사이드와 클라이언트 사이드 모두에서 사용할 수 있어 유용합니다.

### 3. 상태 관리 패턴 직접 구현

Observer 패턴이나 Pub/Sub 패턴을 사용해 간단한 상태 관리 시스템을 만들 수 있습니다.

## 마무리: React는 도구일 뿐 🔧

React는 훌륭한 도구이지만, 모든 문제의 해결책은 아닙니다. 프로젝트의 성격과 요구사항에 따라 적절한 도구를 선택하는 것이 중요합니다.
