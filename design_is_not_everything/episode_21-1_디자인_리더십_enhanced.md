# Episode 21-1: 디자인 리더십, 시니어를 넘어 리더가 되는 길

## 🎬 Scene: 두 시니어 디자이너의 갈림길

```
5년차 디자이너 A:
"저는 디자인만 잘하면 되는 줄 알았어요"
- 뛰어난 실력
- 혼자 작업 선호
- 피드백 거부
- 정치 무관심
→ 7년 후: 여전히 시니어 IC

5년차 디자이너 B:
"디자인은 팀 스포츠라는 걸 깨달았어요"
- 실력 + 소통
- 주니어 멘토링
- 적극적 협업
- 영향력 확대
→ 7년 후: VP of Design

CEO: "B를 승진시킨 이유? 
      혼자 잘하는 사람 10명보다
      10명을 잘하게 만드는 사람 1명이 더 가치있거든요."
```

**리더십은 직책이 아니라 영향력입니다.**

## Part 1: IC에서 리더로의 전환

### 🎯 Individual Contributor vs Leader

```
IC (개인 기여자):
- 내 작업 완성
- 품질 책임
- 실행 중심
- 깊이 있는 전문성
- 개인 성과

Leader (리더):
- 팀 성과 달성
- 전략 수립
- 의사결정
- 넓은 시야
- 팀 성장

전환기 혼란:
"디자인할 시간이 없어요"
"회의만 하는 것 같아요"
"실력이 녹슬까 봐 불안해요"

→ 정상입니다. 역할이 바뀐 것입니다.
```

### 📈 리더십 레벨

```
Level 1: Senior IC
- 자기 일 잘함
- 품질 높은 결과물
영향력: 본인

Level 2: Lead
- 프로젝트 리드
- 주니어 멘토링
영향력: 2-3명

Level 3: Manager
- 팀 관리
- 채용/평가
영향력: 5-8명

Level 4: Senior Manager
- 다수 팀 관리
- 전략 수립
영향력: 10-20명

Level 5: Director
- 조직 설계
- 비전 제시
영향력: 20-50명

Level 6: VP/Head
- 회사 전략
- C-Level 협업
영향력: 50명+
```

## Part 2: 핵심 리더십 스킬

### 🗣 커뮤니케이션 마스터리

```
상향 커뮤니케이션 (임원에게):

❌ Bad:
"디자인 시스템이 필요해요.
다른 회사도 다 하고 있어요."

✅ Good:
"디자인 시스템 구축으로
개발 속도 40% 향상,
연간 5,000만원 절감 가능합니다.
3개월 투자로 1년 내 ROI 300% 예상됩니다."

하향 커뮤니케이션 (팀원에게):

❌ Bad:
"위에서 시켰으니까 해야 해요"

✅ Good:
"이 프로젝트가 왜 중요한지 설명할게요.
우리가 이걸 성공시키면..."

수평 커뮤니케이션 (동료에게):

❌ Bad:
"우리 팀 일이 더 중요해요"

✅ Good:
"어떻게 하면 양쪽 다 win-win할 수 있을까요?"
```

### 🌱 팀원 성장 지원

```
1:1 미팅 구조 (주 1회, 30분):

Check-in (5분):
"이번 주 어떠셨어요?"
"가장 즐거웠던/힘들었던 일은?"

Progress (10분):
"진행 중인 프로젝트 상황은?"
"블로커가 있나요?"

Growth (10분):
"어떤 스킬을 개발하고 싶어요?"
"다음 레벨로 가려면 뭐가 필요할까요?"

Feedback (5분):
"제가 더 잘 도와드릴 수 있는 게 있나요?"
"팀이나 회사에 제안하고 싶은 게 있나요?"

기록:
- 약속한 것 추적
- 성장 기록
- 피드백 문서화
```

## Part 3: 전략적 사고

### 🎯 Design Strategy 수립

```
비즈니스 목표 연결:

회사 목표: 매출 50% 성장
↓
제품 목표: 사용자 2배 증가
↓
디자인 목표: 온보딩 개선으로 전환율 30% 상승
↓
실행 계획:
1. 온보딩 플로우 리디자인
2. A/B 테스트
3. 데이터 분석
4. 점진적 개선

OKR 설정:
Objective: 최고의 사용자 경험 제공
KR1: NPS 40 → 60
KR2: 온보딩 완료율 50% → 75%
KR3: 일일 활성 사용자 10만 → 20만
```

### 📊 데이터 기반 의사결정

```
의견 vs 데이터:

의견: "이 디자인이 더 예뻐요"
데이터: "A안 전환율 12%, B안 전환율 8%"
결정: B안도 예쁘지만 A안 채택

설득력 있는 제안:
1. 문제 정의 (숫자로)
2. 가설 수립
3. 실험 설계
4. 결과 측정
5. 인사이트 도출

프레젠테이션:
"현재 이탈률 73%입니다. (문제)
복잡한 UI가 원인으로 보입니다. (가설)
단순화된 버전을 테스트했더니 (실험)
이탈률이 45%로 감소했습니다. (결과)
전면 적용 시 월 매출 2억 증가 예상됩니다. (임팩트)"
```

## Part 4: 팀 빌딩과 문화

### 👥 드림팀 만들기

```
채용 전략:

포트폴리오 < 팀 핏
기술 < 태도
경력 < 잠재력

인터뷰 질문:
"가장 실패한 프로젝트는?"
→ 실패에서 배우는 태도

"동료와 의견 충돌 시?"
→ 협업 스타일

"5년 후 목표는?"
→ 성장 의지

팀 구성:
- Maker: 실행력 뛰어남
- Thinker: 전략적 사고
- Connector: 커뮤니케이션
- Innovator: 창의적 시도

= 다양성이 만드는 시너지
```

### 🎨 디자인 문화 구축

```
건강한 디자인 문화:

1. 심리적 안전감
- 실패 허용
- 의견 존중
- 실험 장려

2. 투명한 소통
- 의사결정 공유
- 진행상황 오픈
- 피드백 문화

3. 지속적 학습
- 세미나/워크샵
- 컨퍼런스 참가
- 스터디 그룹

4. 축하 문화
- 작은 성공도 축하
- 공개 칭찬
- 팀 회식/이벤트

문화 만들기 활동:
- Design Critique (주 1회)
- Design Sprint (분기 1회)
- Team Lunch & Learn (월 1회)
- Portfolio Day (반기 1회)
```

## Part 5: 정치와 영향력

### 🎭 조직 정치 네비게이션

```
정치 ≠ 나쁜 것
정치 = 영향력 + 관계

Stakeholder 관리:

CEO/임원:
- 비즈니스 임팩트 강조
- 숫자로 말하기
- 리스크 관리

PM:
- 공동 목표 설정
- 트레이드오프 협의
- 정기 싱크

개발:
- 기술 제약 이해
- 단계적 구현
- 존중과 신뢰

마케팅:
- 브랜드 일관성
- 캠페인 지원
- 데이터 공유

영향력 확대 전략:
1. 가시성 확보 (성과 공유)
2. 네트워킹 (커피 챗)
3. 신뢰 구축 (약속 이행)
4. Win-Win (상생 제안)
```

### 🎯 갈등 관리

```
흔한 갈등 상황:

"디자인 vs 개발 우선순위"
해결: 사용자 가치 기준 합의

"리소스 부족"
해결: 우선순위 명확화, 단계적 접근

"의견 충돌"
해결: 데이터 기반 결정, A/B 테스트

갈등 해결 5단계:
1. Listen (경청)
2. Empathize (공감)
3. Apologize (필요시 사과)
4. Resolve (해결책 제시)
5. Follow-up (후속 조치)

중재자 역할:
"양쪽 입장 다 이해됩니다.
공통 목표를 다시 확인하고
어떻게 하면 둘 다 얻을 수 있을지
함께 찾아보면 어떨까요?"
```
