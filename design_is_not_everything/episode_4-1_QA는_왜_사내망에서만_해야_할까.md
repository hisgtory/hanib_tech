# Episode 4-1: QA는 왜 사내망에서만 해야 할까?

개발자가 열심히 만든 서비스가 드디어 테스트 단계에 들어섰습니다! 🎉 그런데 QA팀에서 "사내망에서만 테스트할 수 있어요"라고 하네요. 왜 그럴까요? 오늘은 QA가 사내망에서 진행되는 이유와 그 중요성에 대해 알아보겠습니다.

## 🔒 사내망이란 무엇인가요?

사내망(Internal Network)은 회사 내부에서만 접근할 수 있는 네트워크 환경을 말합니다. 마치 우리 집 안에서만 사용할 수 있는 와이파이처럼, 회사 건물이나 VPN을 통해서만 접속할 수 있는 안전한 공간이죠.

일반 인터넷과 달리 사내망은:

- 외부에서 접근할 수 없어요 (해커들도 못 들어옵니다!)
- 회사의 중요한 데이터와 시스템이 안전하게 보호됩니다
- 특정 IP 대역만 사용하여 접속 권한을 관리할 수 있어요

## 🤔 QA는 왜 사내망에서만 가능한가요?

QA(Quality Assurance) 단계에서 테스트하는 서비스는 아직 완성되지 않은 '베타 버전'입니다. 이 단계에서는 여러 가지 민감한 정보와 불안정한 요소들이 포함되어 있어요.

### 1. 🔐 보안상의 이유

- **아직 공개되지 않은 신기능**이 포함되어 있어요
- **실제 고객 데이터**를 사용해 테스트하는 경우가 있습니다
- **보안 취약점**이 아직 패치되지 않았을 수 있어요
- **개발 중인 API 키**나 **인증 정보**가 노출될 위험이 있습니다

### 2. 💼 비즈니스적 이유

- 경쟁사가 **신제품 정보**를 미리 알아챌 수 있어요
- **마케팅 전략**이나 **출시 일정**이 유출될 수 있습니다
- **가격 정책**이나 **프로모션 계획**이 노출될 위험이 있어요

### 3. 🔧 기술적 이유

- **개발 환경 설정**이 외부에 노출되면 안 돼요
- **내부 시스템과의 연동**이 필요한 경우가 많아요
- **테스트 데이터베이스**에 외부에서 접근하면 위험해요

## 🏭 사내망 QA 환경의 구성

사내망에서 QA를 진행할 때는 보통 다음과 같은 환경이 구성됩니다:

### QA 서버 환경

- **Staging 서버**: 실제 프로덕션과 비슷한 환경
- **QA 데이터베이스**: 테스트용 데이터가 들어있는 DB
- **테스트 API 서버**: 외부 연동 테스트를 위한 목(Mock) 서버

### 접근 방법

- **VPN 접속**: 외부에서 사내망에 안전하게 접속
- **사무실 내 PC**: 회사 내부에서 직접 접속
- **특정 IP 대역**: 허가된 IP에서만 접근 가능

## 💡 QA 환경의 장점

사내망에서 QA를 진행하면 여러 가지 장점이 있어요:

- **안전한 테스트**: 외부 위협으로부터 안전합니다
- **실제와 비슷한 환경**: 프로덕션과 유사한 환경에서 테스트 가능
- **내부 시스템 연동**: ERP, CRM 등 내부 시스템과 연동 테스트
- **통제된 환경**: 테스트 환경을 체계적으로 관리

## 🎯 실제 사례: 왕온라인커머스의 QA

옛날 제가 일했던 온라인 커머스 회사에서는 블랙프라이데이 이벤트를 준비하면서 이런 일이 있었어요:

할인 쿠폰 시스템을 개발하고 QA를 진행하는데, 실수로 QA 서버를 외부에 잘못 노출한 적이 있었습니다. 다행히 빨리 발견해서 바로 차단했지만, 만약 경쟁사가 이를 발견했다면 블랙프라이데이 전략이 통째로 노출될 뻔했죠.

그 이후로는 더욱 엄격하게 사내망 접근 통제를 하게 되었고, VPN 설정도 더 강화했습니다.

## 📝 마무리

QA를 사내망에서만 진행하는 것은 단순히 규칙이나 관례가 아니라, 회사의 소중한 자산을 보호하고 안전한 서비스 개발을 위한 필수적인 조치입니다.

개발자로서 불편할 수도 있지만, 이런 보안 조치들이 결국 우리가 만든 서비스와 회사를 지키는 방패가 되는 것임을 기억해야 합니다. 👍

### 💡 팀: QA를 할 때 꼭 기억하세요!

- VPN 설정을 미리 확인하세요
- 테스트 환경 URL을 외부에 공유하지 마세요
- QA 데이터를 스크린샷 찍을 때 민감한 정보가 포함되지 않도록 주의하세요
- 테스트 종료 후 VPN 연결을 해제하는 것도 잊지 마세요
