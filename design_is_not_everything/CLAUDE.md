## 📱 hanib_tech 콘텐츠 제작 가이드

### 1. Instagram 카드뉴스 제작 전략

#### 각 Week별 카드뉴스 구성 (10장 기준)

**Week 1 - 우리팀을 소개합니다 예시:**

1. **Hook (1장)**: "개발팀에 왜 이렇게 많은 사람이 필요할까요?"
2. **문제제기 (2-3장)**: 각 역할별 설명 (PO, Designer, Developer)
3. **핵심개념 (4-7장)**: 스쿼드의 개념과 구성 이유
4. **실무적용 (8-9장)**: 대기업에서 스쿼드를 운영하는 방식
5. **Payoff (10장)**: "작은 팀이 큰 성과를 내는 비밀은 바로 이것!"

#### 카드별 텍스트 가이드라인:
- **1-2장**: 호기심 유발하는 질문
- **3-7장**: 핵심 개념을 일상 비유로 설명
- **8-9장**: 실제 사례나 적용 방법
- **10장**: 액션 아이템이나 다음 편 예고

### 2. 서적 콘텐츠 확장 전략

#### 각 Episode의 서적 구성 (5-10페이지)

**구조:**
1. **도입부** (1페이지): Instagram에서 다룬 내용을 더 깊게
2. **본론** (3-6페이지): 
   - 실제 사례 연구 2-3개
   - 단계별 설명
   - 흔한 실수와 해결방법
3. **TMI 박스** (중간중간): 재미있는 개발 이야기
4. **Try This at Work** (1페이지): 실무 적용 가이드
5. **다음 연결** (1페이지): 다음 챕터 연결고리

#### 비개발자를 위한 언어 변환 예시:

**Technical → hanib_tech Style:**
- "REST API" → "카페 주문 시스템처럼 정해진 방식으로 소통하는 방법"
- "프론트엔드/백엔드" → "고객이 보는 부분 / 뒤에서 일하는 부분"
- "스쿼드" → "각자 다른 전문분야를 가진 사람들이 하나의 목표를 향해 뭉친 특공대"

### 3. 콘텐츠 톤앤매너 가이드

#### 핵심 원칙:
1. **비개발자도 이해할 수 있게**: 전문용어는 반드시 일상 비유와 함께
2. **호기심 자극**: 각 콘텐츠는 "왜?"로 시작
3. **실무 연결**: "회사에서 이런 상황 본 적 있죠?" 식으로 연결
4. **긍정적 마무리**: 불안감보다는 "이제 알았으니 소통이 더 쉬워질 거예요"

#### 자주 사용할 표현들:
- "아, 그래서 개발자가 그렇게 말했구나!"
- "마케터인 저도 이제 이해했어요"
- "디자이너와 개발자 사이에서 이런 일이..."
- "기획자가 놓치기 쉬운 포인트는..."

### 4. 하이라이트 활용 전략

모든 `<hl>...</hl>` 태그 내용은:

### 5. 시리즈 연속성 유지 방법

- 시리즈 전체 맵 제시 (현재 위치 표시)

### 6. 측정 지표 제안

**Instagram 성과 측정:**
- 저장 수 (실무에서 참고하려는 의도)
- 공유 수 (동료에게 설명할 때 사용)

**서적 성과 측정:**
- 챕터별 완독률
- 실무 적용 후기
