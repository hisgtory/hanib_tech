## 🎬 Opening: 그 날의 연봉 협상

### 연말 평가 시즌, HR 미팅룸

```
HR: "3년차인데 연봉 인상을 많이 요구하시네요?"
Designer A: "저는 3년 동안 열심히..."
HR: "하지만 임팩트가..."

[다른 미팅룸]
Designer B (2년차): "제가 주도한 디자인 시스템으로 개발 시간 40% 단축했고, 
                     사용자 만족도 15% 상승, 디자인 일관성으로 CS 문의 30% 감소했습니다.
                     구체적인 데이터는 여기 있습니다."
HR: "인상 폭 조정해드리겠습니다."
```

**핵심 차이**: 연차 vs 임팩트

오늘은 IT 회사에서 디자이너가 진짜 성장하는 방법을 알려드릴게요!

## 📊 Part 1: IT 회사 디자이너 레벨 시스템

### 실제 기업들의 디자이너 레벨 구조

#### 🎯 레벨별 역할과 책임

```yaml
Junior Designer (L1-L2) | 0-2년차
  역할: 실행과 학습
  - 정해진 태스크 수행
  - 디자인 가이드라인 따르기
  - 피드백 반영과 개선
  연봉: 3,500-4,500만원

Mid-level Designer (L3-L4) | 2-5년차  
  역할: 독립적 문제 해결
  - 프로젝트 오너십
  - 주니어 가이드
  - 프로세스 개선 제안
  연봉: 4,500-6,500만원

Senior Designer (L5-L6) | 5-8년차
  역할: 전략과 리더십
  - 디자인 전략 수립
  - 크로스 펑션 협업 리드
  - 디자인 시스템 구축
  연봉: 6,500-9,000만원

Lead/Principal Designer (L7+) | 8년차+
  역할: 비전과 조직 성장
  - 프로덕트 비전 정의
  - 팀 빌딩과 문화 구축
  - 비즈니스 임팩트 창출
  연봉: 9,000만원-1.5억+
```

### 레벨업의 진짜 기준

#### ❌ 잘못된 생각
```
"3년 했으니까 시니어 아니야?"
"포트폴리오 예쁘면 되는 거 아니야?"
"야근 많이 하면 인정받겠지?"
```

#### ✅ 실제 평가 기준
```markdown
## Impact (임팩트)
- 비즈니스 지표 개선
- 사용자 경험 향상 수치
- 팀 생산성 증가

## Scope (영향 범위)
- Task → Feature → Product → Company
- 개인 → 팀 → 조직 → 회사 전체

## Autonomy (자율성)
- 지시 받음 → 가이드 필요 → 독립적 → 다른 사람 리드
```

## 🚀 Part 2: 레벨별 성장 전략

### Level 1-2: Junior Designer 생존기

#### 📚 핵심 학습 영역

**1. 개발 친화적 디자인**
```markdown
## 꼭 배워야 할 것들
- [ ] HTML/CSS 기본 이해
- [ ] 개발자 도구 사용법
- [ ] Git 기본 (최소한 읽기)
- [ ] API가 뭔지 이해
- [ ] 컴포넌트 사고

## 왜 중요한가?
"이거 구현 가능한가요?" → "이렇게 하면 구현 쉬울 것 같아요"
```

**2. 커뮤니케이션 스킬**
```
❌ Bad: "예쁘게 해주세요"
✅ Good: "브랜드 톤앤매너에 맞춰 따뜻한 느낌으로"

❌ Bad: "이상해요"
✅ Good: "여백이 불균형해서 시각적 흐름이 어색해요"
```

**3. 실무 도구 마스터**
```markdown
## Essential Tools
- Figma: 컴포넌트, 오토레이아웃, 변수
- Slack: 스레드, 멘션, 채널 활용
- Jira: 티켓 작성, 스프린트 이해
- Notion: 문서화, 위키 작성
```

#### 💼 Junior 시절 실제 업무 사례

**Case: 버튼 컴포넌트 개선 프로젝트**

```markdown
## Before (실패 사례)
"버튼 디자인 개선했어요" 
→ 개발자: "기존 코드 다 수정해야 해요"
→ PM: "일정 지연됩니다"
→ 결과: 롤백

## After (성공 사례)
1. 현재 버튼 사용 현황 조사
2. 개발자와 구현 방법 논의
3. 단계적 마이그레이션 계획
4. 가이드 문서 작성
→ 결과: 성공적 적용, 팀 신뢰 획득
```

### Level 3-4: Mid-level Designer 도약기

#### 🎯 프로젝트 오너십 갖기

**오너십의 진짜 의미**
```yaml
Before: "PM이 요청한 화면 디자인했어요"

After: 
  - 왜 이 기능이 필요한지 이해
  - 사용자 리서치 직접 수행
  - 대안 3가지 제시
  - 개발 난이도 고려한 우선순위
  - 성공 지표 정의 참여
  - 런칭 후 데이터 분석
```

#### 📊 데이터 기반 디자인

**실제 A/B 테스트 사례**
```markdown
## CTA 버튼 최적화 프로젝트

### 가설
"버튼 색상을 더 선명하게 하면 클릭률 상승"

### 실험 설계
- Control: 기존 파란색 (#3B82F6)
- Variant A: 진한 파란색 (#2563EB)
- Variant B: 초록색 (#10B981)

### 결과
- Control: 2.3% CTR
- Variant A: 2.1% CTR (-8.7%)
- Variant B: 3.1% CTR (+34.8%) 🎯

### 인사이트
"더 진한색이 아니라 대비가 중요했음"
→ 전체 CTA 가이드라인 업데이트
```

#### 🤝 크로스 펑션 협업

**개발팀과 디자인 스프린트**

```markdown
## 2주 스프린트 일정

Week 1:
월: 요구사항 리뷰 (PM + Dev + Design)
화: 와이어프레임 + 기술 검토
수: 디자인 초안 + API 스펙 논의
목: 디자인 리뷰 + 개발 착수
금: 디자인 QA 대기

Week 2:
월: 1차 구현 리뷰
화-수: 디자인 QA + 수정
목: 최종 QA
금: 배포 + 회고
```

### Level 5-6: Senior Designer 전문가

#### 🏗 디자인 시스템 구축

**실제 디자인 시스템 구축 과정**

```markdown
## Phase 1: 현황 분석 (2주)
- 버튼 스타일 27가지 발견
- 색상 정의 없이 52가지 사용
- 폰트 크기 18가지 난립

## Phase 2: 정리와 표준화 (4주)
- Core Token 정의 (색상, 타이포, 여백)
- Component 표준화 (30개 핵심 컴포넌트)
- Documentation 작성

## Phase 3: 마이그레이션 (8주)
- 개발팀과 단계별 적용 계획
- 페이지별 순차 적용
- QA와 버그 수정

## 결과
- 디자인 시간 50% 단축
- 개발 시간 40% 단축
- 디자인 일관성 점수 95%
- ROI: 연간 5억원 절감
```

#### 📈 비즈니스 임팩트 창출

**전환율 개선 프로젝트**
```markdown
## 문제 정의
- 장바구니 → 결제 전환율 15% (업계 평균 25%)

## 원인 분석
1. 복잡한 결제 단계 (7단계)
2. 불명확한 배송비 정책
3. 신뢰 신호 부족

## 디자인 솔루션
1. 단계 축소: 7 → 3단계
2. 배송비 실시간 계산 표시
3. 보안 뱃지, 고객 리뷰 추가

## 결과
- 전환율: 15% → 24% (+60%)
- 월 매출 증가: 3억원
- 고객 만족도: 4.2 → 4.6
```

#### 👥 주니어 멘토링

**효과적인 멘토링 방법**
```markdown
## 디자인 리뷰 피드백 레벨

### Level 1: 지시형 (비추천)
"이 색상을 #3B82F6로 바꿔주세요"

### Level 2: 설명형 (기본)
"대비가 부족해서 #3B82F6가 더 나을 것 같아요"

"WCAG 2.1 기준으로 대비율이 3:1 미만이에요.

### Level 4: 코칭형 (최고)
"이 색상 선택의 의도가 뭔가요?
접근성 체크는 해보셨나요?
다른 대안은 어떤 게 있을까요?"
```

```
    ↓ No
```

## 🚨 Part 7: 피해야 할 함정들

### 커리어 성장의 함정

#### 함정 1: "포트폴리오만 예쁘면 돼"
```
현실: 비즈니스 임팩트 없는 예쁜 디자인 < 못생겼지만 매출 올린 디자인
해법: 모든 프로젝트에 성과 지표 포함
```

#### 함정 2: "연차 차면 자동 승진"
```
현실: 10년차 주니어도 있고, 3년차 시니어도 있음
해법: 연차보다 임팩트와 역량에 집중
```

#### 함정 3: "혼자 잘하면 인정받겠지"

```markdown
## Warning Signs
```
