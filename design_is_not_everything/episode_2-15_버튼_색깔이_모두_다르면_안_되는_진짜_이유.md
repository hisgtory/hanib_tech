# Episode 2-15: 버튼 색깔이 모두 다르면 안 되는 진짜 이유 [Exclusive]

💡 이 콘텐츠는 한입테크 Exclusive 시리즈입니다. 실무에서 바로 적용 가능한 심화 내용을 담았습니다.

## 🎯 핵심 요약

- 일관된 버튼 색상은 사용자의 인지 부하를 줄여줍니다
- 색상별 역할을 명확히 정의하면 사용자 행동을 자연스럽게 유도할 수 있습니다
- 디자인 시스템을 통해 개발 효율성과 브랜드 일관성을 동시에 확보할 수 있습니다

## 🤔 왜 이게 문제가 될까?

어느 날, 신입 디자이너가 프로젝트를 진행하며 각 페이지마다 "예쁜" 버튼을 디자인했습니다. 회원가입 페이지엔 밝은 초록색, 상품 구매엔 진한 파란색, 설정 페이지엔 보라색... 각각은 아름다웠지만, 전체적으로 보니 문제가 있었죠.

### 1. 인지 부하와 학습 부담

사용자는 페이지를 이동할 때마다 "이 페이지에서 중요한 버튼은 무슨 색이지?"라고 다시 학습해야 합니다. 이는 매우 피곱한 경험이죠.

### 2. 브랜드 일관성 훼손

"어? 이 앱이 맞나? 다른 앱 아닌가?" 사용자는 혼란스러워합니다. 브랜드의 신뢰도가 떨어지는 순간입니다.

### 3. 개발자의 고통

"이 페이지에서 주요 버튼은 #FF5733이고, 저 페이지는 #3366FF이고..." 개발자들은 각 페이지마다 다른 색상 코드를 관리해야 합니다. 유지보수가 악몽이 됩니다.

## 💡 해결책: 색상 시스템 구축하기

### 단계 1: 기본 색상 역할 정의

- **Primary Action** (주 행동): 브랜드 켈러, 주로 파란색 계열
- **Secondary Action** (보조 행동): 회색 또는 테두리만 있는 버튼
- **Destructive Action** (파괴적 행동): 빨간색, 삭제/취소 등
- **Success/Confirm** (성공/확인): 초록색, 완료/성공 상태

### 단계 2: 계층적 색상 시스템 만들기

각 역할별로 5-7단계의 명도를 준비합니다:

- Primary-100: #E3F2FD (배경색)
- Primary-300: #64B5F6 (호버 상태)
- Primary-500: #2196F3 (기본 상태)
- Primary-700: #1976D2 (누른 상태)

### 단계 3: 접근성 검증

WCAG 2.1 기준에 따라 명도 대비를 검사합니다:

- 일반 텍스트: 4.5:1 이상
- 큰 텍스트: 3:1 이상
- UI 컴포넌트: 3:1 이상

## 🛠️ 실무 적용 팁

### 1. 디자인 토큰 활용하기

```css
:root {
  --color-primary: #2196F3;
  --color-secondary: #757575;
  --color-danger: #F44336;
  --color-success: #4CAF50;
}

.btn-primary {
  background-color: var(--color-primary);
}
```

### 2. 컴포넌트 라이브러리 구축

React, Vue, Angular 등에서 공통 버튼 컴포넌트를 만들어 사용합니다:

```jsx
<Button variant="primary">저장하기</Button>
<Button variant="secondary">취소</Button>
<Button variant="danger">삭제</Button>
```

### 3. 대표적인 디자인 시스템 예시

**Material Design (Google)**

- Primary: 브랜드 대표색
- Secondary: 보조 액션
- Error: 빨간색 계열

**Human Interface Guidelines (Apple)**

- Blue: 기본 인터랙티브 요소
- Red: 파괴적 액션
- Green: 긍정적 액션

## 🎯 실전 체크리스트

- ☑️ 각 버튼의 역할이 명확하게 정의되어 있나요?
- ☑️ 색상 시스템이 문서화되어 있나요?
- ☑️ 접근성 기준을 충족하나요?
- ☑️ 개발자가 쉽게 사용할 수 있는 컴포넌트가 있나요?
- ☑️ 색맹 사용자를 고려했나요?

## 📈 효과 측정하기

### 정량적 지표

- **태스크 완료 시간**: 일관된 색상 시스템은 사용자의 학습 부담을 줄여 15-20% 더 빠른 태스크 완료를 가능하게 합니다
- **오류율 감소**: 명확한 색상 체계는 사용자 오류를 30% 감소시킵니다
- **개발 속도**: 컴포넌트 재사용으로 40% 빠른 개발

### 정성적 효과

- 브랜드 신뢰도 향상
- 사용자 만족도 증가
- 팀 협업 효율성 향상

## 💡 마무리

버튼 색깔이 모두 다르면 안 되는 진짜 이유는 단순합니다. **사용자 경험의 일관성**입니다. 사용자는 한 번 학습한 패턴을 계속 사용하고 싶어합니다. 이러한 일관성은 사용자의 인지 부하를 줄이고, 브랜드 신뢰도를 높이며, 개발 효율성을 향상시킵니다.

좋은 디자인은 예쁜 것이 아니라 **사용하기 편한 것**입니다. 오늘부터 체계적인 색상 시스템을 구축해보세요!
