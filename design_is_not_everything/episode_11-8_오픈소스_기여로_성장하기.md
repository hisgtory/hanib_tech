## 🏰 Chapter 1: 오픈소스 던전의 정체

### 🌟 오픈소스란 무엇인가?

**테크선배**: "오픈소스는 전 세계 개발자들이 함께 만들어가는 거대한 협력 프로젝트야!"

> 📱 **일상 비유**
> 오픈소스 = 온 동네가 함께 키우는 공동 정원
> - 누구나 씨앗을 심을 수 있고 (기여)
> - 열매를 따먹을 수 있고 (사용)
> - 정원을 가꿀 수 있어! (개선)

**한빛**: "와... 그럼 제가 쓰는 앱들도?"

**테크선배**: "샤라락~ 맞아! React, Vue, Python... 대부분이 오픈소스야!"

### ⚡ 오픈소스 던전의 특징

```
🏆 경험치 획득 시스템
├── 🔰 Lv.1: 이슈 리포팅 (버그 발견하기)
├── ⚔️ Lv.3: 문서 기여 (가이드 작성)
├── 🛡️ Lv.5: 코드 리뷰 (동료 도움)
├── 🗡️ Lv.7: 기능 개발 (새로운 능력 추가)
└── 👑 Lv.10: 메인테이너 (던전 관리자)
```

---

## 🎮 Chapter 2: 첫 던전 입장하기

### 🗺️ 초보자용 던전 찾기

**한빛**: "어떤 던전부터 도전해야 할까요?"

**테크선배**: "쿠와앙! 먼저 'Good First Issue' 태그가 붙은 퀘스트를 찾아보자!"

> 🎯 **초보자 친화적 던전들**
> - 📚 문서 번역 프로젝트
> - 🎨 디자인 시스템
> - 📖 튜토리얼 개선
> - 🐛 간단한 버그 수정

### 🔍 던전 정보 수집하기

```bash
# 던전 입장 준비 (Repository Fork)
git clone https://github.com/awesome-project/repo.git
cd repo

# 던전 지도 읽기 (README.md)
# 규칙서 확인하기 (CONTRIBUTING.md)
# 퀘스트 목록 보기 (Issues)
```

**TMI 박스 💡**
> **전설의 첫 기여 영웅담**
> 
> 리누스 토르발스가 Linux를 오픈소스로 공개했을 때:
> "이건 그냥 취미 프로젝트야, 크고 전문적인 것은 아니야"
> 
> → 지금은 전 세계 서버의 95%가 Linux! 
> 작은 기여 하나가 세상을 바꿀 수 있어요! ⚡

## ⚔️ Chapter 3: 첫 번째 보스전 - PR 만들기

### 🎯 미션 수행하기

**한빛**: "드디어 코드를 수정했어요!"

**테크선배**: "좋아! 이제 Pull Request라는 보스전에 도전할 시간이야!"

> 🏹 **PR은 이런 거야**
> - 마케팅팀이 기획서를 검토 요청하는 것
> - 디자이너가 시안을 피드백 받는 것
> - 개발자가 "이 코드 어때요?" 하고 물어보는 것

### 💫 PR 작성의 기술

```markdown
# 보스전 신청서 (PR Template)

## 🎯 무엇을 했나요?
버튼 색상을 파란색으로 변경했습니다.

## 🔍 왜 필요한가요?
사용자 테스트에서 기존 빨간색이 경고로 인식된다는 피드백

## 📸 스크린샷
[Before/After 이미지]

## ✅ 체크리스트
- [x] 테스트 완료
- [x] 문서 업데이트
- [x] 디자인 가이드 준수
```

**테크선배**: "쿠와앙! 이렇게 친절하게 설명하면 메인테이너도 기뻐할 거야!"
