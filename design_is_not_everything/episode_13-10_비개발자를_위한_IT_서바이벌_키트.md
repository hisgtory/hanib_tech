## 🎁 **키트 구성품 전체 목록**

### 📖 **1. 필수 IT 용어 사전** (메인 아이템)
**🏷️ 효능**: 개발자와의 대화에서 길을 잃지 않게 해줍니다

**📝 사용법**:
- **API**: "프로그램들이 서로 대화하는 규칙"이라고 기억하세요
- **데이터베이스**: "엑셀의 진화형, 훨씬 똑똑한 창고"
- **서버**: "24시간 일하는 컴퓨터"
- **클라우드**: "남의 컴퓨터에 내 데이터 저장하기"
- **프론트엔드**: "사용자가 보고 만지는 부분"
- **백엔드**: "뒤에서 묵묵히 일하는 부분"

**💡 활용 팁**: 
처음 듣는 용어가 나오면 "OO은 쉽게 말하면 어떤 건가요?"라고 물어보세요. 
개발자들도 설명하는 걸 좋아합니다!

---

### 💬 **2. 개발자와 소통 매뉴얼** (핵심 아이템)
**🏷️ 효능**: 서로 다른 언어를 쓰는 두 종족 간의 번역기 역할

**📋 상황별 대화 가이드**:

**🟢 요청할 때**:
- ❌ "간단한 거니까 금요일까지 해주세요"
- ✅ "이 작업에 보통 어느 정도 시간이 걸리나요? 우선순위를 어떻게 정하면 좋을까요?"

**🟡 진행상황 확인할 때**:
- ❌ "왜 아직 안 됐어요?"
- ✅ "현재 진행상황은 어떤가요? 제가 도울 수 있는 부분이 있을까요?"

**🔴 문제 발생 시**:
- ❌ "버그가 있어요! 빨리 고쳐주세요!"
- ✅ "이런 문제가 발생했는데, 어떤 정보를 더 드리면 도움이 될까요?"

---

### 🛠️ **3. 도구 사용 가이드** (실용 아이템)
**🏷️ 효능**: 개발자들이 쓰는 도구를 이해하고 활용할 수 있게 해줍니다

**🔧 기본 도구 키트**:

**Git/GitHub**
- 무엇인가: "코드의 타임머신이자 협업 공간"
- 언제 쓰는가: 코드 변경 사항을 추적하고 여러 사람이 함께 작업할 때
- 당신이 알아야 할 것: "커밋 = 저장", "브랜치 = 평행우주", "머지 = 합치기"

**Jira/Trello/Notion**
- 무엇인가: "할 일 관리의 진화형"
- 언제 쓰는가: 프로젝트 진행상황을 투명하게 관리할 때
- 당신이 할 수 있는 것: 태스크 상태 확인, 우선순위 논의, 진행상황 체크

**Slack/Discord**
- 무엇인가: "팀을 위한 카카오톡"
- 효과적 사용법: 채널별 목적 이해, 스레드 활용, 적절한 알림 설정

---

### 🚨 **4. 위기 대처 시나리오** (응급처치 키트)
**🏷️ 효능**: IT 관련 응급상황에서 당황하지 않고 대처할 수 있게 해줍니다

**🔴 레드 알럿 상황들**:

**시나리오 1: "서버가 다운됐어요!"**
- 즉시 행동: 개발팀에 알리기, 영향 범위 파악하기
- 소통 방법: "언제부터 문제가 시작됐는지", "어떤 현상이 나타나는지" 구체적으로 설명
- 대외 대응: 미리 준비된 공지 템플릿 활용

**시나리오 2: "데이터가 날아갔어요!"**
- 즉시 행동: 추가 작업 중단, 백업 상황 확인
- 질문하기: "언제부터 문제가 있었는지", "백업에서 복구 가능한지"
- 예방책: 정기적인 백업 정책 수립

**시나리오 3: "해커 공격을 받은 것 같아요!"**
- 즉시 행동: 보안팀/개발팀 연락, 관련 시스템 접근 중단
- 증거 보전: 스크린샷, 로그 기록 등 보관
- 대응 절차: 미리 정해진 보안 인시던트 대응 절차 따르기

---

### 📈 **5. 성장 로드맵과 체크리스트** (업그레이드 키트)
**🏷️ 효능**: IT 리터러시를 단계적으로 향상시킬 수 있는 가이드

**🌱 Level 1: 생존자** (1-3개월)
- [ ] 기본 IT 용어 30개 이해하기
- [ ] 개발자와 5분 이상 대화 가능하기
- [ ] 간단한 요청사항을 구체적으로 전달하기
- [ ] Git의 기본 개념 이해하기

**🌿 Level 2: 적응자** (3-6개월)
- [ ] 프로젝트 관리 도구 능숙하게 사용하기
- [ ] 개발 프로세스의 전체 흐름 이해하기
- [ ] 기술적 제약사항과 비즈니스 요구사항의 균형점 찾기
- [ ] 간단한 A/B 테스트 설계하고 분석하기

**🌳 Level 3: 협력자** (6개월 이후)
- [ ] 개발자들과 기술적 논의에 참여하기
- [ ] 데이터를 기반으로 한 의사결정 내리기
- [ ] 새로운 기술 트렌드 파악하고 비즈니스 기회 연결하기
- [ ] 비개발자 동료들에게 IT 지식 전파하기

## 🎁 **보너스 아이템 (TMI 박스)**

### 🔍 **개발자 생태계 관찰 가이드**
개발자들의 행동 패턴을 이해하면 더 효과적으로 소통할 수 있어요:

- **오전 9시 전**: 커피 필수, 복잡한 요청 금지
- **점심 직후**: 졸릴 수 있음, 간단한 확인 업무 적합
- **금요일 오후**: 새로운 배포나 큰 변경 작업 지양
- **월요일 오전**: 주말 동안 생각해온 아이디어들 쏟아내는 시간

### 📚 **숨겨진 개발자 언어 사전**
- "금방 할게요" = 최소 2-3일
- "간단해요" = 생각보다 복잡할 수 있음
- "확인해보겠습니다" = 디버깅 시간 필요
- "이론적으로는 가능합니다" = 실제로는 어려움

### 🎯 **효과적인 피드백 주는 법**
- 구체적 상황 설명하기
- 기대했던 동작 vs 실제 동작 비교하기
- 스크린샷이나 화면 녹화 첨부하기
- 브라우저, 디바이스 정보 포함하기
