# Episode 4-1: QA는 왜 사내망에서만 해야 할까?

## Hook: 왜 개발팀은 QA를 숨기고 싶어할까?

"QA 서버 주소 알려달라고 하면 개발자들이 왜 이렇게 망설일까요?" 비개발자라면 한 번쯤 궁금했을 이 질문의 답을 찾아보겠습니다.

## QA(Quality Assurance)란 무엇인가?

QA는 제품이나 서비스가 출시되기 전에 품질을 검증하는 과정입니다. 쉽게 말해서 "정말 잘 만들어졌는지 확인하는 단계"라고 생각하면 됩니다.

- **기능 테스트**: 모든 버튼이 제대로 작동하는가?
- **성능 테스트**: 많은 사용자가 동시에 접속해도 괜찮은가?
- **보안 테스트**: 해킹이나 데이터 유출 위험은 없는가?
- **사용성 테스트**: 실제 사용자가 쉽게 사용할 수 있는가?

## 왜 사내망에서만 QA를 할까?

### 1. 보안 이유 💻

**미완성된 기능의 노출 방지**
QA 단계의 제품은 아직 완성되지 않은 상태입니다. 버그가 있을 수 있고, 보안 취약점이 존재할 수 있습니다. 만약 이런 상태의 서비스가 외부에 노출된다면?

- 해커들이 취약점을 찾아 공격할 수 있습니다
- 경쟁사가 우리의 새로운 기능을 미리 알 수 있습니다
- 개인정보가 유출될 위험이 있습니다

**실제 사례**: 한 스타트업이 QA 서버를 실수로 공개 설정해두었다가, 아직 출시되지 않은 신기능이 언론에 보도된 사건이 있었습니다.

### 2. 비즈니스 이유 💼

**브랜드 보호**
QA 단계의 서비스는 사용자 경험이 완성되지 않은 상태입니다. 만약 고객이나 투자자가 이런 모습을 보게 된다면 회사에 대한 신뢰가 떨어질 수 있습니다.

**조기 유출 방지**
- 신제품 정보가 미리 알려져 마케팅 효과가 떨어질 수 있습니다
- 경쟁사가 비슷한 기능을 먼저 출시할 수 있습니다
- 투자자나 파트너십 협상에 영향을 줄 수 있습니다

### 3. 기술적 이유 ⚙️

**서버 부하 관리**
QA 서버는 보통 실제 서비스 서버보다 사양이 낮습니다. 많은 사람이 접속하면 서버가 다운될 수 있어서 정상적인 테스트가 어려워집니다.

**데이터 관리**
- QA에서는 테스트용 가짜 데이터를 사용합니다
- 실제 사용자 데이터와 섞이면 안 됩니다
- 테스트 중 데이터가 삭제되거나 변경될 수 있습니다

## 사내망 QA 환경은 어떻게 구성되나요?

### VPN (Virtual Private Network) 접속
```
개발자 컴퓨터 → VPN → 회사 네트워크 → QA 서버
```

### 사내 IP 제한
특정 IP 주소에서만 접속할 수 있도록 설정합니다:
```
허용된 IP: 192.168.1.xxx (사무실 네트워크)
차단된 IP: 그 외 모든 외부 IP
```

### 인증 시스템
- 회사 계정으로만 로그인 가능
- 2단계 인증 필수
- 접속 로그 모두 기록

## 사내망 QA의 장점들

### ✅ 철저한 보안
- 외부 해킹 시도 원천 차단
- 내부 직원만 접근 가능
- 모든 접속 기록 추적 가능

### ✅ 안정적인 테스트 환경
- 예측 가능한 트래픽
- 통제된 사용자 그룹
- 일정한 네트워크 환경

### ✅ 피드백 품질 향상
- 제품을 잘 아는 직원들이 테스트
- 구체적이고 전문적인 피드백
- 빠른 커뮤니케이션 가능

### ✅ 비즈니스 기밀 보호
- 신기능 조기 유출 방지
- 경쟁사 정보 수집 차단
- 브랜드 이미지 보호

## 실제 사례: 한 핀테크 회사의 QA 프로세스

**배경**: 새로운 투자 상품 출시 준비 중

**QA 환경 구성**:
1. **사내 전용 네트워크**: 오직 사무실에서만 접속 가능
2. **임시 계정 생성**: 테스트용 가짜 사용자 데이터 사용
3. **단계별 테스트**: 
   - 1주차: 개발팀 내부 테스트
   - 2주차: 기획팀과 디자인팀 참여
   - 3주차: 경영진 및 마케팅팀 리뷰

**결과**: 
- 출시 전 17개의 주요 버그 발견 및 수정
- 보안 취약점 3개 사전 차단
- 경쟁사에 기능 유출 없이 성공적 론칭

## 마무리: QA는 제품의 마지막 관문

QA를 사내망에서만 하는 것은 개발자들의 고집이 아니라, 제품과 회사를 보호하기 위한 필수적인 보안 조치입니다. 

**기억해야 할 포인트**:
- QA는 미완성 제품을 테스트하는 단계
- 보안, 비즈니스, 기술적 이유로 사내망 사용이 필수
- 안전한 환경에서 더 정확한 테스트 가능
- 최종 사용자에게는 완성된 제품만 공개

다음 에피소드에서는 QA 과정에서 버그를 발견했을 때 누가 처리해야 하는지, 어떻게 효과적으로 리포트하는지에 대해 알아보겠습니다!
