
## Part 1: 개발자의 세계 이해하기

```

"작동하게 → 안정적으로 → 효율적으로"

교집합 찾기:
"좋은 UX = 안정적이고 빠른 서비스"
```

### 😤 개발자를 화나게 하는 말 TOP 10

```
1. "그냥 버튼 하나 추가하는 건데..."
   → 실제: DB 스키마 변경, API 수정, 테스트, 배포

2. "인스타그램처럼 만들어주세요"
   → 인스타그램 개발 인력: 500명+

3. "픽셀 퍼펙트하게 해주세요"
   → 브라우저별, 기기별 차이 무시

4. "이거 5분이면 되지 않아요?"
   → 5분 = 구현 + 테스트 + 리뷰 + 배포 = 3시간

5. "코드 몇 줄만 바꾸면 되잖아요"
   → 나비효과를 아시나요?

6. "왜 이렇게 오래 걸려요?"
   → 설명하는데만 30분

7. "다른 앱에는 있던데요"
   → 그 앱 개발 기간: 3년

8. "백엔드는 신경 안 써도 되죠?"
   → 프론트만 예쁘면 뭐해요

9. "배포하고 나서 수정하면 되죠"
   → 핫픽스 = 야근

10. "개발자님들이 알아서 해주세요"
    → 책임 회피 NO
```

## Part 2: 개발 언어 마스터하기

### 💻 필수 개발 용어 사전

#### Frontend 용어
```
HTML/CSS/JS = 뼈대/옷/행동
React/Vue = 프레임워크 (레고 세트)
Component = 재사용 가능한 조각
State = 상태 (열림/닫힘)
Props = 전달 값
Redux = 상태 관리 도구
Responsive = 반응형 (화면 크기 대응)
SPA = 한 페이지 앱
PWA = 앱처럼 작동하는 웹
```

#### Backend 용어
```
API = 데이터 주고받는 규칙
Database = 데이터 저장소
Server = 요청 처리하는 컴퓨터
REST = API 설계 방식
GraphQL = 필요한 데이터만 요청
CRUD = 생성/읽기/수정/삭제
Authentication = 누구인지 확인
Authorization = 권한 확인
Cache = 임시 저장 (빠른 로딩)
```

#### 개발 프로세스 용어
```
Git = 버전 관리
Branch = 작업 공간 분리
Commit = 저장
Push/Pull = 업로드/다운로드
Merge = 합치기
PR (Pull Request) = 코드 리뷰 요청
CI/CD = 자동 테스트/배포
Sprint = 2주 개발 주기
Refactoring = 코드 개선
Tech Debt = 나중에 고쳐야 할 것
```

## Part 6: 갈등 해결하기

### 💥 자주 발생하는 갈등과 해결

#### 갈등 1: "디자인대로 안 됐어요"

```
상황:
디자이너: "간격이 20px인데 30px로 됐네요"
개발자: "그게 그거 아닌가요?"

해결:
1. Design QA 세션 정례화
2. 픽셀 차이의 비즈니스 영향 설명
3. 중요도에 따른 우선순위 설정
4. 디자인 시스템으로 자동화

결과:
"중요한 것과 아닌 것 구분하니 서로 편해졌어요"
```

#### 갈등 2: "기술적으로 불가능해요"

```
상황:
개발자: "이건 구현 불가능해요"
디자이너: "왜요? 다른 앱은 되던데"

해결:
1. "불가능"의 정확한 이유 파악
   - 진짜 불가능 (5%)
   - 시간이 많이 걸림 (60%)
   - 방법을 모름 (35%)

2. 대안 찾기
   - 유사한 효과
   - 단계적 구현
   - 외부 라이브러리

3. Trade-off 논의
   "이 기능에 2주 투자 vs 다른 기능 3개"

결과:
"완벽하진 않지만 만족스러운 중간점 찾음"
```
