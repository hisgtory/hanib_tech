## 🎂 클라우드 서비스의 3단계

### Pizza as a Service로 이해하기
```
🍕 피자로 비유하는 클라우드 서비스

1. 전통적 방식 (On-Premise)
   = 집에서 직접 피자 만들기
   • 밀가루, 토핑 직접 구매 ✓
   • 오븐 구매 및 관리 ✓
   • 레시피 개발 ✓
   • 조리 및 서빙 ✓

2. IaaS (Infrastructure as a Service)
   = 오븐과 주방만 빌리기
   • 주방 시설 제공 ✓
   • 재료는 직접 준비
   • 조리는 직접
   예: AWS EC2, Google Compute Engine

3. PaaS (Platform as a Service)
   = 도우와 오븐 제공, 토핑만 올리기
   • 기본 도우 제공 ✓
   • 오븐 제공 ✓
   • 토핑만 선택
   예: Heroku, Google App Engine

4. SaaS (Software as a Service)
   = 완성된 피자 배달
   • 주문만 하면 끝 ✓
   • 먹기만 하면 됨
   예: Gmail, Notion, Slack
```

### CDN (Content Delivery Network)
```
🌍 CDN이 왜 필요할까?

문제 상황:
서버 위치: 서울
사용자 위치: 뉴욕
→ 거리: 11,000km
→ 지연시간: 150ms
→ 이미지 100개 = 15초 대기

CDN 해결책:
1. 전 세계 거점에 복사본 저장
2. 가장 가까운 곳에서 전송
3. 지연시간: 150ms → 10ms
4. 로딩 시간: 15초 → 1초

주요 CDN 서비스:
• CloudFlare (무료 플랜 有)
• AWS CloudFront
• Akamai
```

### 비용 계산기 활용
```javascript
// 간단한 클라우드 비용 예측기
function calculateMonthlyCost(config) {
  const costs = {
    // 서버 비용 (시간당)
    server: {
      small: 0.02,   // t3.micro
      medium: 0.04,  // t3.small
      large: 0.08    // t3.medium
    },
    // 스토리지 (GB당)
    storage: 0.023,
    // 트래픽 (GB당)
    traffic: 0.09
  };
  
  const monthlyCost = 
    (costs.server[config.size] * 24 * 30) +
    (costs.storage * config.storage) +
    (costs.traffic * config.traffic);
    
  return {
    total: monthlyCost,
    yearly: monthlyCost * 12,
    breakdown: {
      server: costs.server[config.size] * 24 * 30,
      storage: costs.storage * config.storage,
      traffic: costs.traffic * config.traffic
    }
  };
}

// 사용 예시
const myApp = calculateMonthlyCost({
  size: 'medium',
  storage: 100,  // GB
  traffic: 500   // GB
});
console.log(`월 비용: $${myApp.total}`);
// 월 비용: $76.3
```

### 마이크로서비스 아키텍처
```
🔧 모놀리스 vs 마이크로서비스

전통적 방식 (모놀리스):
┌─────────────────┐

클라우드 네이티브 (마이크로서비스):
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│로그인│ │상품  │ │결제  │ │배송  │
│ 📦   │ │ 📦   │ │ 📦   │ │ 📦   │
└──────┘ └──────┘ └──────┘ └──────┘
    ↓        ↓        ↓        ↓
         API Gateway
장점: 독립적 배포, 확장, 관리
```

5. **미래는 하이브리드**: 온프레미스와 클라우드의 조합
