## 🔬 병리학: 4가지 주요 사망 원인

### 1. 기술적 불가능 (Technical Impossibility)
```
🔍 사례: 실시간 얼굴 인식 + AR 필터 + 라이브 스트리밍
문제점: 모바일 CPU/GPU 한계 초과
증상: 앱 크래시, 기기 발열, 배터리 방전
```

**실제 케이스**: 2023년 A사 라이브 커머스 앱
- 계획: 실시간 뷰티 필터 + HD 스트리밍
- 현실: 아이폰 12 이하 기기에서 작동 불가
- 결과: 사용자층 60% 이탈

### 2. 성능 한계 (Performance Constraints)
```
📊 위험 지표
- 로딩 시간 3초 초과
- 메모리 사용량 500MB 초과
- 배터리 소모량 시간당 20% 초과
```

### 3. 플랫폼 제약 (Platform Limitations)
```
🍎 iOS 함정들
- 앱스토어 리뷰 가이드라인 위반
- 사파리 브라우저 기능 제한
- 메모리 관리 정책

🤖 안드로이드 지뢰밭
- 기기별 화면 크기 파편화
- OS 버전별 API 차이
- 제조사별 커스터마이징 영향
```

### 4. 외부 의존성 (Third-party Dependencies)
```
⚡ API 지뢰 목록
- 결제 시스템: PG사별 지원 기능 차이
- 지도 서비스: 일일 호출량 제한
- 소셜 로그인: 정책 변경 리스크
- 푸시 알림: 플랫폼별 제약사항
```

#### 2. 실제 팀 대화 시나리오 부족
의료 비유는 좋지만, 실제 팀에서 벌어지는 구체적인 대화나 갈등 상황의 예시가 더 필요함.

**개선 방안:**
```
### 실제 기술 검토 회의 시나리오
디자이너: "이 효과는 사용자 경험에 꼭 필요해요"
개발자: "성능 문제가 있을 것 같은데..."
기획자: "비즈니스적으로는 중요한 기능이에요"
→ 건설적인 해결책 도출 과정의 실제 대화
```

```
- 팀 내 학습 문화 구축 방안

```
