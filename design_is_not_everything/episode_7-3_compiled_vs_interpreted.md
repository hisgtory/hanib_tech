# Episode 7-3. 컴파일 언어가 빠른 대신 불편한 이유

"컴파일 언어가 더 빠르다"는 말을 들어보셨을 텐데요. 그럼 왜 모든 언어가 컴파일 언어가 아닐까요?

## ⚡ 컴파일 vs 인터프리터의 차이

### 컴파일 언어 (C, C++, Go, Rust)
- **미리 번역**: 코드를 실행 전에 기계어로 변환
- **빠른 실행**: 이미 번역된 코드를 바로 실행
- **느린 개발**: 코드 변경 시마다 컴파일 필요

### 인터프리터 언어 (Python, JavaScript, Ruby)  
- **실시간 번역**: 코드를 실행하면서 한 줄씩 번역
- **느린 실행**: 번역과 실행이 동시에 일어남
- **빠른 개발**: 코드 변경 즉시 실행 가능

<callout>
🍕 **피자 배달로 비유하면**
- 컴파일: 미리 만들어둔 냉동피자 (빠른 섭취, 오래 걸린 준비)
- 인터프리터: 주문 후 만드는 신선한 피자 (느린 섭취, 빠른 주문)
</callout>

## 🚀 컴파일 언어의 장점

1. **뛰어난 성능**: 게임 엔진, 운영체제 등
2. **메모리 효율성**: 시스템 리소스 최적화
3. **배포 편의성**: 실행 파일 하나로 배포

## 😅 컴파일 언어의 단점

1. **긴 컴파일 시간**: 대규모 프로젝트는 몇 분~몇 시간
2. **플랫폼 의존성**: OS별로 다른 실행 파일 필요
3. **개발 속도 저하**: "코드 수정 → 컴파일 → 테스트" 반복
