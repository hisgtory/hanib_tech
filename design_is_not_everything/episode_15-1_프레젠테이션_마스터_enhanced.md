# Episode 15-1: 프레젠테이션, 디자인을 팔아라

## 🎬 Scene: 같은 디자인, 다른 결과

```
오후 2시, 임원 프레젠테이션

디자이너 A:
"저... 이게 디자인인데요... 어... 보시면..."
(클릭 클릭 클릭)
"아 잠깐만요... 파일이..."
임원: "다음에 다시 보죠"
결과: 기각 ❌

디자이너 B:
"3초만 주세요. (간단한 시연)
우리 사용자의 가장 큰 고통은 복잡한 결제 과정입니다.
이 디자인으로 클릭 수를 7번에서 2번으로 줄였고,
예상 전환율 상승은 34%, 월 매출 2억 증가 예상됩니다."
임원: "바로 진행하세요!"
결과: 즉시 승인 ✅
```

**좋은 디자인도 못된 발표로 죽습니다.**
**프레젠테이션은 디자인의 완성입니다.**

## Part 1: 프레젠테이션의 심리학

### 🧠 의사결정권자의 뇌 이해하기

```
임원/클라이언트의 관심사:

1순위: ROI (투자 대비 수익)
2순위: 리스크 (실패 가능성)
3순위: 타임라인 (언제까지)
4순위: 리소스 (비용/인력)
...
10순위: 디자인 디테일

깨달음:
"그들은 픽셀이 아닌 비즈니스를 본다"
```

### ⏱ 첫 30초의 법칙

```
연구 결과:
- 첫인상 형성: 7초
- 집중력 유지: 8초 (금붕어보다 짧음)
- 의사결정: 30초

30초 구성:
[0-7초] Hook: 충격적 사실/질문
[8-15초] Problem: 핵심 문제
[16-23초] Solution: 해결책
[24-30초] Impact: 기대 효과

예시:
"우리는 매일 1억원을 버리고 있습니다. (3초)
장바구니 이탈률 73%. (5초)
새 체크아웃 디자인으로 (5초)
이탈률을 40%로 낮출 수 있습니다. (5초)
월 30억 매출 증가 예상됩니다." (7초)
```

## Part 2: 스토리텔링 구조

### 📖 STAR 스토리 구조

```
S - Situation (상황 설정)
T - Task (해결 과제)
A - Action (디자인 솔루션)
R - Result (예상 결과)
```

#### 실전 적용 예시

```
S (상황): 
"신규 가입자의 60%가 온보딩에서 이탈합니다.
사용자 인터뷰 결과, '너무 복잡하다'가 1위였습니다."
[슬라이드: 이탈 그래프]

T (과제):
"온보딩을 단순화하여 
완료율을 40%에서 70%로 올려야 합니다."
[슬라이드: 목표 수치]

A (액션):
"5단계를 2단계로, 필수 정보만 수집,
Progressive Disclosure 적용했습니다."
[슬라이드: Before/After]

R (결과):
"예상 완료율 72%, 
월 신규 가입자 5,000명 증가,
LTV 15억 증가 예상됩니다."
[슬라이드: 비즈니스 임팩트]
```

### 🎭 감정 곡선 디자인

```
효과적인 발표의 감정 흐름:

불안 → 공감 → 희망 → 확신
  ↓      ↓      ↓      ↓
문제   "맞아"  해결책  "하자!"

시간별 구성:
0-20%: 문제 (불안 조성)
20-30%: 공감 (같은 편)
30-70%: 솔루션 (희망 제시)
70-90%: 증명 (확신 부여)
90-100%: CTA (행동 유도)
```

## Part 3: 슬라이드 디자인 공식

### 🎨 임팩트 있는 슬라이드 구조

```
슬라이드 타입별 템플릿:

1. 표지 슬라이드
┌──────────────────┐
│                  │
│   프로젝트명      │
│   한 줄 요약      │
│                  │
│   발표자 | 날짜   │
└──────────────────┘

2. 문제 정의
┌──────────────────┐
│  😰 큰 숫자       │
│                  │
│  "73% 이탈"      │
│                  │
│  작은 설명        │
└──────────────────┘

3. 솔루션 제시
┌──────────────────┐
│  Before | After  │
│  [이미지] [이미지] │
│                  │
│  핵심 개선 3가지   │
└──────────────────┘

4. 데이터 증명
┌──────────────────┐
│     📊           │
│  [그래프/차트]    │
│                  │
│  핵심 수치 1개    │
└──────────────────┘
```

### 📊 데이터 시각화 규칙

```
❌ 하지 마세요:
- 복잡한 표
- 작은 글씨
- 10개 이상 데이터
- 3D 차트
- 무지개 색상

✅ 이렇게 하세요:
- 핵심 숫자 1-3개
- 큰 폰트 (최소 24pt)
- 단순 그래프
- 2-3가지 색상
- 애니메이션 최소화

데이터 표현 공식:
"X%의 사용자가 Y 때문에 Z한다"
→ 스토리가 있는 데이터
```

## Part 4: 발표 기술

### 🎤 목소리와 제스처

```
음성 테크닉:

1. 속도 변화
- 중요 부분: 천천히
- 전환 부분: 빠르게
- 숫자: 또박또박

2. 볼륨 조절
- 문제: 크게
- 설명: 보통
- 강조: 작게 → 크게

3. 멈춤 활용
"이탈률이... (2초 멈춤) 73%입니다"
→ 주목도 200% 상승

제스처 가이드:
- 숫자: 손가락 표시
- 비교: 양손 대비
- 과정: 좌→우 이동
- 강조: 주먹 쥐기
- 개방: 팔 벌리기
```

### 👀 아이컨택과 포지셔닝

```
시선 분배 (10명 기준):
- 의사결정자: 40%
- 영향력자: 30%
- 지지자: 20%
- 나머지: 10%

포지셔닝:
┌────────────┐
│   스크린    │
│            │
└────────────┘
      ↑
    2-3걸음
      ↑
    발표자
      
- 스크린 가리지 않기
- 레이저 포인터 < 손짓
- 이동은 전환 시에만
```

## Part 5: Q&A 마스터하기

### 💬 예상 질문 준비

```
필수 준비 질문 10:

1. "비용은 얼마나 드나요?"
→ 개발 2주, 디자인 완료, ROI 3개월

2. "리스크는 뭐가 있나요?"
→ A/B 테스트로 검증, 롤백 가능

3. "왜 지금 해야 하나요?"
→ 경쟁사 이미 적용, 매일 300만원 손실

4. "다른 방법은 없나요?"
→ 3가지 안 검토, 이것이 최적

5. "사용자가 싫어하면?"
→ 베타 테스트 긍정 85%

6. "개발 가능한가요?"
→ 개발팀 사전 검토 완료

7. "이전에도 시도했는데?"
→ 그때와 다른 점 3가지

8. "성공 지표는?"
→ 전환율, DAU, NPS

9. "실패하면?"
→ Plan B 준비됨

10. "레퍼런스 있나요?"
→ 성공 사례 3개 준비
```

### 🛡 어려운 질문 대처법

```
공격적 질문:
Q: "이런 걸 디자인이라고 했나?"

대처:
1. 감정 인정: "날카로운 지적 감사합니다"
2. 구체화: "어떤 부분이 우려되시는지요?"
3. 데이터: "사용자 테스트 결과는..."
4. 대안: "다른 안도 준비했습니다"

모르는 질문:
"좋은 질문입니다. 
정확한 답변을 위해 확인 후 
오늘 내로 이메일 드리겠습니다."
(절대 아는 척 금지)
```

## Part 6: 도구별 프레젠테이션

### 💻 도구별 최적화

```
1. Keynote/PowerPoint
장점: 안정적, 애니메이션
단점: 정적
팁: 빌드 효과로 스토리 전개

2. Figma Present
장점: 실시간 수정, 프로토타입
단점: 인터넷 필요
팁: 오프라인 백업 필수

3. Pitch
장점: 협업, 분석
단점: 유료
팁: 팀 발표시 최적

4. Google Slides
장점: 무료, 공유 쉬움
단점: 기능 제한
팁: 링크 공유로 팔로업

5. Prototype
장점: 인터랙티브
단점: 맥락 설명 어려움
팁: 슬라이드와 병행
```

### 🎬 온/오프라인 발표 차이

```
오프라인 발표:
- 비언어적 소통 활용
- 즉각적 반응 확인
- 기술 이슈 최소
- 프린트 자료 준비

온라인 발표:
- 화면 공유 체크
- 음소거 확인
- 인터랙티브 도구 활용
- 녹화 활용
- 채팅 모니터링

하이브리드:
- 온라인 우선 설계
- 현장 참석자 케어
- 두 버전 준비
```
