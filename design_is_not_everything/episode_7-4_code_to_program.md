# Episode 7-4. 코드 100줄이 프로그램 1만 줄이 되는 비밀

개발자가 작성한 간단한 코드 100줄이 어떻게 거대한 프로그램이 될까요? 그 비밀을 알아봅시다.

## 🧩 라이브러리와 프레임워크의 마법

개발자가 작성하는 코드는 빙산의 일각입니다:

```python
import requests
response = requests.get('https://api.github.com')
print(response.json())
```

이 3줄의 코드 뒤에는:
- HTTP 프로토콜 구현 코드 수천 줄
- JSON 파싱 라이브러리 코드 수백 줄  
- 네트워크 통신 라이브러리 코드 수천 줄

<callout>
🏗️ **건물 건축으로 비유하면**
개발자는 설계도만 그리고, 실제 건축은 미리 만들어진 부품(라이브러리)들을 조립해서 완성합니다.
</callout>

## 📦 의존성의 눈덩이 효과

하나의 라이브러리가 또 다른 라이브러리들을 필요로 합니다:

```
내 코드 (100줄)
└── Express.js (웹 프레임워크)
    ├── Body-parser (요청 파싱)
    ├── Cookie-parser (쿠키 처리)  
    └── Static (정적 파일 서빙)
        ├── Send (파일 전송)
        └── Mime (파일 타입 감지)
```

## 🎯 컴파일러의 최적화

컴파일러는 코드를 최적화하면서 크기를 늘립니다:
- **인라인 함수**: 자주 쓰는 함수를 직접 삽입
- **루프 전개**: 반복문을 펼쳐서 속도 향상
- **에러 처리**: 각종 예외 상황 대응 코드 추가

<hl>개발자는 "무엇을" 만들지 정의하고, 컴퓨터가 "어떻게" 만들지 결정합니다!</hl>
