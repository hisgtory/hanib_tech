## 🐛 Part 5: QA와 버그 트래킹

#### 🎨 디자인 설명
```
[순환 플로우차트]

   [발견] → [리포트] → [확인] → [할당]
      ↑                            ↓

상태별 색상:
- New: 빨강

- Reopened: 주황
```

#### 📝 사용 위치
- QA 프로세스 이해

### 5.2 버그 우선순위 매트릭스

#### 🎨 디자인 설명
```
[2x2 매트릭스]

영향도 High │ P1 Critical │ P2 Major    │
           │     🔴      │     🟠      │
           ├──────────────┼──────────────┤
      Low  │ P3 Minor    │ P4 Trivial  │
           │     🟡      │     🟢      │
           └──────────────┴──────────────┘
             Low         High
             ← 발생 빈도 →

예시 포함:
P1: 결제 불가

P3: UI 깨짐
```

## 📐 제작 사양

### 인쇄용 (책)
- 해상도: 300dpi
- 색상: CMYK
- 크기: 페이지에 맞춤
- 여백: 상하좌우 15mm

### 디지털용 (웹/앱)
- 해상도: 72dpi (@1x), 144dpi (@2x)
- 색상: RGB/HEX
- 포맷: PNG (다이어그램), SVG (아이콘)
- 반응형 대응
