# Episode 16-1: 사이드 프로젝트, 성장의 지름길이자 수익의 원천

## 🎬 Scene: 금요일 밤, 두 디자이너의 선택

```
디자이너 A (넷플릭스 시청 중):

월급: 400만원
성장: 정체
1년 후: 여전히 주니어

디자이너 B (사이드 프로젝트 중):
"이번 주말엔 내 앱 런칭이다!"
월급: 400만원
사이드 수익: 300만원/월
1년 후: 
- 앱스토어 1위
- 연수익 5,000만원
- 구글 스카우트 제안

"똑같이 시작했는데 왜 이렇게 달라졌을까?"
```

**사이드 프로젝트는 취미가 아니라 투자입니다.**

## Part 1: 왜 사이드 프로젝트인가?

### 💎 사이드 프로젝트의 숨겨진 가치

```
본업에서 얻는 것:
- 월급 (고정)
- 경험 (제한적)
- 네트워크 (회사 내)
- 성장 (느림)

사이드에서 얻는 것:
- 추가 수익 (무제한)
- 다양한 경험
- 글로벌 네트워크
- 폭발적 성장
- 포트폴리오
- 개인 브랜드
- 창업 기회
- 투자 유치 가능성
```

### 📈 성공 사례들

```
Unsplash (2013)
시작: 사진 10장 공유 사이트
현재: 월 20억뷰, $63M 투자

Notion (2013)
시작: 개인 노트 앱
현재: 기업가치 10조원

Instagram (2010)
시작: 위치 기반 체크인 앱
현재: 메타에 1조원 인수

공통점:
- 퇴근 후/주말 프로젝트
- 작은 문제 해결
- 꾸준한 개선
- 커뮤니티 형성
```

## Part 2: 아이디어 찾기

### 🎯 문제 발견 프레임워크

```
1. 일상 불편 관찰
"매일 겪는 작은 불편"
→ 알람 앱, 할일 관리, 날씨

2. 본업 문제 해결
"회사에서 불편한 것"
→ 디자인 플러그인, 협업 도구

3. 취미 영역 개선
"좋아하는 분야의 빈틈"
→ 운동 트래커, 독서 관리

4. 트렌드 라이딩
"뜨는 분야 + 내 전문성"
→ AI 도구, Web3 서비스

문제 검증 체크리스트:
□ 나도 겪는 문제인가?
□ 주변에 3명 이상 공감하는가?
□ 기존 솔루션이 불만족스러운가?
□ 돈 내고 해결하고 싶은가?
```

### 💡 아이디어 구체화

```
아이디어 캔버스:

┌─────────────────────────────┐
│ 문제: 디자이너 포트폴리오 관리 │
├─────────────────────────────┤
│ 타겟: 주니어 디자이너         │
│ 솔루션: 원클릭 포트폴리오      │
│ 차별화: AI 자동 큐레이션       │
│ 수익: 프리미엄 템플릿         │
│ MVP: 3개 템플릿 + 기본 기능    │
└─────────────────────────────┘

검증 방법:
1. 랜딩 페이지 (1일)
2. 이메일 수집 (1주)
3. 베타 테스터 모집 (2주)
4. MVP 개발 (1개월)
```
