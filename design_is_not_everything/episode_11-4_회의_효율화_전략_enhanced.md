```
09:00-10:00 주간 킥오프
```

**회의는 필요악이 아니라 전략적 도구입니다.**
**제대로 하면 시간을 아끼고 성과를 높입니다.**

## Part 1: 회의 문화 진단

### 📊 당신의 회의 건강도 체크

```
□ 회의 시작/종료 시간이 명확하다
□ 참석자 모두의 역할이 분명하다
□ 회의 전 자료를 미리 공유한다
□ 액션 아이템이 명확히 정해진다
□ 회의록이 24시간 내 공유된다
□ 불필요한 참석자는 없다
□ 의사결정이 회의에서 이뤄진다
□ 후속 조치가 추적된다

점수:
0-2개: 회의 지옥 🔥
3-5개: 개선 필요 ⚠️
6-7개: 양호 ✅
8개: 완벽! 🌟
```

### 😵 나쁜 회의의 특징

```
The Meeting from Hell:

"안녕하세요~ 오늘 뭐 얘기하기로 했죠?"
(아무도 모름)

"일단 시작해볼까요?"
(30분간 잡담)

"그래서 결론이 뭐죠?"
(아무도 모름)

"다음에 다시 논의하죠"
(무한 반복)

결과:
- 2시간 낭비
- 결정 사항 0
- 참석자 10명 × 2시간 = 20시간 소멸
```

## Part 2: 회의 유형별 최적화

### 🎯 목적별 회의 프레임워크

#### 1. 브레인스토밍 회의
```
시간: 45분
참석: 5-7명
준비: 문제 정의

구조:
[0-5분] 문제 공유
[5-20분] 개인 아이디어 (무음)
[20-35분] 공유 및 그룹핑
[35-40분] 투표
[40-45분] 다음 단계

규칙:
✅ "Yes, and..." 원칙
❌ "No, but..." 금지
✅ 질보다 양
❌ 비판 금지
```

#### 2. 의사결정 회의
```
시간: 30분
참석: 결정권자 + 2-3명
준비: 옵션 정리

구조:
[0-5분] 상황 브리핑
[5-10분] 옵션 제시
[10-20분] 토론
[20-25분] 결정
[25-30분] 액션 플랜

필수 산출물:
- 최종 결정
- 담당자
- 데드라인
- 성공 지표
```

#### 3. 디자인 크리틱
```
시간: 20분/프로젝트
참석: 디자이너 + 관련자
준비: 프로토타입

구조:
[0-3분] 컨텍스트 설명
[3-5분] 디자인 시연
[5-15분] 피드백
[15-18분] 우선순위
[18-20분] 다음 단계

피드백 규칙:
"I like..." (긍정)
"I wish..." (개선)
"What if..." (제안)
```

## Part 3: 효율적인 회의 진행법

### 🎪 퍼실리테이션 기술

```
회의 진행자의 역할:

1. 시작 (2분)
"오늘 목표는 X 결정입니다.
종료 시간은 10:30입니다."

2. 진행 중
- 시간 관리: "5분 남았습니다"
- 참여 유도: "○○님 의견은?"
- 궤도 수정: "본론으로 돌아가죠"
- 요약: "지금까지 나온 의견은..."

3. 마무리 (3분)
"결정 사항: A를 B까지 C가 완료
다음 회의: 필요시 소집"
```

### ⏰ 시간 관리 테크닉

```
25분 회의 법칙:
30분 회의 → 25분으로
60분 회의 → 50분으로

이유:
- 집중력 유지
- 버퍼 타임 확보
- 연속 회의 방지

타임박싱:
소개: 2분
논의: 15분
결정: 5분
정리: 3분
= 25분

타이머 활용:
🍅 뽀모도로 타이머
⏰ Time Timer (시각적)
📱 회의 전용 앱
```
