# HTTP Status Codes 시리즈 스크립트

## 📝 시리즈 개요
- **주제**: HTTP 응답 상태 코드 완전 정복
- **구성**: 9개 카드 + 아웃트로
- **대상**: 개발 입문자 및 웹 서비스 관련 업무자
- **핵심 메시지**: "상태 코드는 어렵지만 실무에서는 팀이 선택한 것만 사용한다"

---

## 🎬 카드별 상세 스크립트

### 1️⃣ HTTP_ 1.png (인트로)
**화면 구성**: 검은 배경, 노란색 제목
**메인 카피**: "200, 404, 500, ... HTTP 응답 모음.zip"

**내레이션 스크립트**:
```
"웹 서핑을 하다 보면 한 번쯤 만나는 숫자들이 있죠.
404 Not found, 400 Bad Request, 200 OK...
이런 코드들이 도대체 뭘까요?
오늘은 HTTP 상태 코드라는 .zip 파일을 함께 풀어보겠습니다!"
```

**키포인트**: 
- 일상적 경험으로 시작
- .zip 메타포로 호기심 유발
- 친근하고 궁금증을 자아내는 톤

---

### 2️⃣ HTTP_ 2.png (정의)
**화면 구성**: 회색 배경, 클라이언트-서버 다이어그램
**메인 카피**: "우리가 알고있는 404, 500 등의 코드를 HTTP Status Code라고 해요"

**내레이션 스크립트**:
```
"우리는 매일 HTTP를 통해 서버와 소통하고 있어요.
웹페이지를 보거나 앱을 사용할 때마다 말이죠.
이때 서버는 우리가 요청한 데이터와 함께 
'어떤 상황인지'를 알려주는 코드를 보내줍니다.
바로 이게 HTTP Status Code예요!"
```

**키포인트**:
- HTTP의 기본 동작 원리 설명
- 클라이언트-서버 통신의 시각화
- 상태 코드의 역할과 의미 명확화

---

### 3️⃣ HTTP_ 3.png (200 OK)
**화면 구성**: 노란 배경, 성공 메시지 UI 예시
**메인 카피**: "200 OK 👍"

**내레이션 스크립트**:
```
"200 OK는 '성공했어요!'를 뜻하는 코드입니다.
서버가 여러분의 요청을 잘 처리했다는 뜻이죠.
이 코드를 받으면 클라이언트는 '잘 처리됐구나😋' 하고
화면을 업데이트하거나 성공 메시지를 보여줍니다.
게시글을 올렸을 때 '성공적으로 등록되었습니다' 같은 메시지를 보신 적 있죠?"
```

**키포인트**:
- 가장 기본적인 성공 응답
- 사용자 경험과의 연결
- 구체적인 사용 사례 제시

---

### 4️⃣ HTTP_ 4.png (201 Created)
**화면 구성**: 노란 배경, 생성 완료 UI 예시
**메인 카피**: "201 Created ✨"

**내레이션 스크립트**:
```
"201 Created는 200보다 더 구체적인 성공 코드예요.
'새로운 것을 성공적으로 만들었어요!'라는 뜻이죠.
게시글을 작성하거나 댓글을 달았을 때 사용됩니다.
200이 일반적인 성공이라면, 201은 '새로 생성된' 특별한 성공이에요!"
```

**키포인트**:
- 200과의 차이점 명확화
- 생성 작업의 특별함 강조
- 실무에서의 구체적 사용 상황

---

### 5️⃣ HTTP_ 5.png (400 Bad Request)
**화면 구성**: 노란 배경, 이메일 입력 오류 UI
**메인 카피**: "400 Bad Request 🧯"

**내레이션 스크립트**:
```
"400 Bad Request는 '잘못된 요청이에요'라는 뜻입니다.
사용자가 뭔가 잘못 입력했을 때 나타나죠.
예를 들어, 이메일 형식이 틀렸거나 필수 항목을 빼먹었을 때요.
서버는 '어디가 잘못됐는지'도 함께 알려주고,
클라이언트는 그걸 화면에 빨간색으로 표시해줍니다."
```

**키포인트**:
- 클라이언트 측 오류임을 강조
- 구체적인 오류 상황 예시
- 사용자 피드백의 중요성

---

### 6️⃣ HTTP_ 6.png (403 Forbidden)
**화면 구성**: 노란 배경, Google Drive 접근 권한 요청 화면
**메인 카피**: "403 Forbidden 🚫"

**내레이션 스크립트**:
```
"403 Forbidden은 '권한이 없어요'라는 뜻입니다.
URL을 받아서 클릭했는데 '접근 권한을 요청하세요'라는 
화면을 보신 적 있으시죠? 바로 그게 403이에요.
파일은 존재하지만, 여러분이 볼 권한이 없다는 뜻입니다."
```

**키포인트**:
- 모든 사람이 경험한 상황 활용
- Google Drive 예시로 친숙함 조성
- 권한과 인증의 개념 이해

---

### 7️⃣ HTTP_ 7.png (404 Not Found)
**화면 구성**: 노란 배경, Notion "페이지를 찾을 수 없음" 화면
**메인 카피**: "404 Not Found ❌"

**내레이션 스크립트**:
```
"404 Not Found! 인터넷을 하는 사람이라면 누구나 본 적 있는 코드죠.
URL이 잘못됐거나 페이지가 삭제되었을 때 나타납니다.
그런데 재미있는 점이 있어요.
가끔은 권한이 없는 페이지에 대해서도 404를 보여줘요.
페이지가 존재한다는 사실 자체를 숨기기 위해서죠!"
```

**키포인트**:
- 가장 유명한 에러 코드
- 일반적인 원인과 특별한 활용법
- 보안 관점에서의 고급 팁

---

### 8️⃣ HTTP_ 8.png (500 Internal Server Error)
**화면 구성**: 노란 배경, 한입 캐릭터와 대화
**메인 카피**: "500 Internal Server Error 🧯"

**내레이션 스크립트**:
```
"500 Internal Server Error는 서버에 문제가 생겼다는 뜻이에요.
다른 코드들과 달리 이건 개발자들이 의도하지 않은 에러입니다.
그래서 개발자들은 이 에러가 최대한 안 생기도록 열심히 노력해요.
만약 500대 에러를 발견하시면 개발팀에 제보해주세요! 
정말 도움이 됩니다."
```

**키포인트**:
- 서버 측 예상치 못한 오류
- 개발자 관점에서의 해석
- 커뮤니티 참여 유도

---

### 9️⃣ HTTP_ 9.png (실무 관점)
**화면 구성**: 회색 배경, 한입 캐릭터의 현실적 조언
**메인 카피**: "설명하지 못한 코드가 아직 많이 남아있어요!"

**내레이션 스크립트**:
```
"HTTP 상태 코드는 정말 많아요. 하지만 모든 코드를 다 쓰지는 않아요.
실무에서는 팀끼리 상의해서 필요한 코드만 골라서 사용합니다.
'우리 팀은 200, 400, 404만 쓸까요?' 이런 식으로요.
오늘 다룬 코드들은 가장 많이 사용되는 것들이니까
일단 이것들부터 잘 알아두시면 됩니다!"
```

**키포인트**:
- 이론과 실무의 차이 인정
- 팀 단위 의사결정의 중요성
- 실용주의적 접근법 강조
