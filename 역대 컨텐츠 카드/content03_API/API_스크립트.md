# API 기초 개념 시리즈 스크립트

## 📝 시리즈 개요
- **주제**: API가 뭔지 몰라서 힘든 사람들을 위한 친절한 설명
- **구성**: 7개 카드 + 아웃트로
- **대상**: 개발 입문자, 비개발자 중 API 이해가 필요한 사람
- **핵심 메시지**: "API는 어려운 게 정상이다. 하지만 우리는 이미 매일 사용하고 있다."

---

## 🎬 카드별 상세 스크립트

### 1️⃣ API_1.png (인트로)
**화면 구성**: 노란 배경, 현실적인 개발팀 대화 말풍선
**대화 내용**: 
- "아직 API가 일부분만 나온 상태라.."
- "외부 API 연동이.."
- "..아아 넹!"

**내레이션 스크립트**:
```
"개발팀 회의에서 이런 대화, 들어보신 적 있으신가요?
'API가 일부분만 나온 상태라..', '외부 API 연동이..'
듣고 있으면 뭔가 중요한 것 같은데, 도대체 API가 뭘까요?
오늘은 이 궁금증을 함께 풀어보겠습니다!"
```

**키포인트**: 
- 실제 개발 현장의 대화로 시작
- 비개발자도 경험했을 법한 상황
- 공감대 형성 후 자연스러운 학습 동기 유발

### 2️⃣ API_2.png (정의와 첫 혼란)
**화면 구성**: 회색 배경, 개발자와 고양이 캐릭터 대화
**메인 카피**: "API는 Application Programming Interface의 약자로 두 소프트웨어가 서로 통신할 수 있게하는 메커니즘입니다."

**내레이션 스크립트**:
```
"API는 Application Programming Interface의 줄임말이에요.
두 소프트웨어가 서로 대화할 수 있게 해주는 메커니즘입니다.
...음, 그래서요? 여전히 어렵죠?
괜찮아요! 처음엔 다들 이래요.
공식 정의만 들어서는 이해하기 어려운 게 당연합니다."
```

**키포인트**:
- 정확한 정의 제시 후 즉시 어려움 인정
- 학습자의 혼란을 정상적인 것으로 받아들임
- 심리적 안전감 제공

### 3️⃣ API_3.png (핵심 가치 제시)
**화면 구성**: 노란 배경, 활짝 웃는 캐릭터
**메인 카피**: "API란 데이터와 소프트웨어간의 상호작용!"

**내레이션 스크립트**:
```
"좀 더 쉽게 설명해드릴게요!
API는 앱과 서버가 대화하는 방법이에요.
인스타그램을 예로 들어볼까요?
앱 개발자는 어떻게 하면 서버에서 피드를 가져올 수 있을지,
팔로우 관계를 업데이트할 수 있을지 알아야 해요.
바로 이때 API가 '이렇게 요청하면 됩니다!'라고 알려주는 거예요."
```

**키포인트**:
- 추상적 개념을 구체적 상황으로 설명
- 모든 사람이 아는 인스타그램 예시 활용
- 실제 앱 개발 맥락에서의 필요성 설명

---

### 4️⃣ API_4.png (API 문서화)
**화면 구성**: 노란 배경, Swagger UI 스크린샷과 개발 도구들
**메인 카피**: "수많은 API를 문서로 관리!"

**내레이션 스크립트**:
```
"서버 개발자는 정말 많은 API를 만들어야 해요.
좋아요 API, 팔로우 API, 피드 조회 API...
이 모든 걸 어떻게 관리할까요?
바로 문서로 정리해서 앱 개발자에게 제공하는 거예요!
보통은 Swagger라는 도구를 사용해서 코드에서 자동으로 문서를 만들어줍니다."
```

**키포인트**:
- API의 복잡성과 문서화 필요성
- 실무에서 사용하는 도구 소개
- 개발팀 간 협업 방식 이해

### 5️⃣ API_5.png (소셜 로그인 사례)
**화면 구성**: 노란 배경, Facebook 로그인 코드와 JSON 응답
**메인 카피**: "우리의 일상에서 볼 수 있는 수많은 API - 1"

**내레이션 스크립트**:
```
"API가 어디에 쓰이는지 궁금하시죠?
여러분이 매일 사용하고 있는 것들이에요!
'Facebook으로 로그인', '구글로 로그인' 버튼을 보세요.
이런 소셜 로그인을 만들려면 각 플랫폼의 API 문서를 보고
정해진 방식대로 구현해야 합니다.
생각보다 복잡하죠? 그래서 개발자들이 여러 플랫폼을 지원하는 걸 어려워해요."
```

**키포인트**:
- 일상적 경험과 API의 연결
- 실제 코드와 응답 예시로 현실감 제공
- 개발의 복잡성에 대한 이해 증진

---

### 6️⃣ API_6.png (결제 시스템 사례)
**화면 구성**: 노란 배경, 결제 서비스 로고들 (토스페이먼츠, 네이버페이, 카카오페이 등)
**메인 카피**: "우리의 일상에서 볼 수 있는 수많은 API - 2"

**내레이션 스크립트**:
```
"온라인에서 결제할 때도 API를 사용해요.
토스페이먼츠, 네이버페이, 카카오페이...
각각 다른 회사에서 제공하는 결제 서비스들이죠.
이것들을 연동하려면 각 회사의 API 문서를 공부해야 해요.
그래서 개발자들은 늘 새로운 API를 배우고 있답니다.
지도 API, SMS 발송 API 등등... 정말 다양해요!"
```

**키포인트**:
- 결제라는 필수적 기능과 API
- 여러 서비스 연동의 복잡성
- 개발자의 지속적 학습 필요성 이해

### 7️⃣ API_7.png (친근한 마무리)
**화면 구성**: 회색 배경, 고양이와 개발자 캐릭터가 함께
**메인 카피**: "알아들기 어려웠던 API, 조금 이해되시나요?"

**내레이션 스크립트**:
```
"처음엔 어려웠던 API, 이제 조금 감이 오시나요?
완벽하게 이해하지 못해도 괜찮아요.
중요한 건 API가 우리 일상 속에 이미 깊숙이 들어와 있다는 거예요.
로그인할 때, 결제할 때, 지도를 볼 때...
모든 순간에 API가 열심히 일하고 있답니다.
개발자가 되고 싶다면 언젠가는 더 깊이 배우게 될 거고,
아니더라도 이제 개발팀 회의에서 API 얘기가 나와도 당황하지 마세요!"
```

**키포인트**:
- 완벽한 이해를 강요하지 않는 부드러운 마무리
- 일상과의 연결로 친근함 증대
- 향후 학습에 대한 동기 부여
- 실무에서의 자신감 향상

- "어려워요.." 라는 솔직한 반응 인정
- 완벽한 이해보다 심리적 안전감 우선
- "조금 이해되시나요?" 겸손한 확인

### 현실성
- 실제 개발 현장 대화로 시작

- 개발자와 비개발자 모두를 대상

- 점진적 이해 추구
