# Refactoring 시리즈 스크립트

## 📝 시리즈 개요
- **주제**: 리팩터링이 뭔지 모르는 사람들을 위한 현실적 가이드
- **구성**: 8개 카드 + 아웃트로
- **대상**: 개발팀과 협업하는 모든 직군, 개발 입문자
- **핵심 메시지**: "리팩터링은 선택이 아니라 필수다. 하지만 언제, 어떻게 할지는 상황에 따라 달라진다."

---

## 🎬 카드별 상세 스크립트

### 1️⃣ Refactoring_1.png (인트로 - 현실적 대화)
**화면 구성**: 검은 배경, 실제 개발팀 대화 상황
**대화 내용**:
- "네! 근데 저희 그런 기능이 있었나요?"
- "..? 리팩터링 한다니까요..?"
- ".. 네??"

**내레이션 스크립트**:
```
"회사에서 이런 대화 들어보신 적 있나요?
'이번 스프린트는 리팩터링을 좀 할게요'
'리팩터링이요? 그게 뭐죠?'
개발팀에서 자주 나오는 단어인데, 정작 뭔지는 잘 모르겠는 '리팩터링'
오늘은 이 신비한 단어의 정체를 파헤쳐보겠습니다!"
```

**키포인트**:
- 실제 업무 상황에서의 소통 문제
- 비개발자도 겪을 수 있는 상황
- 유머러스하면서도 현실적인 접근

### 2️⃣ Refactoring_2.png (정의와 시각적 설명)
**화면 구성**: 회색 배경, Before/After 블록 정리 작업 시각화
**메인 카피**: "리팩터링은 코드를 개선하는 작업이에요"

**내레이션 스크립트**:
```
"리팩터링은 코드를 개선하는 작업이에요.
쉽게 말해 '코드 정리'라고 보시면 됩니다.
여러분이 방을 정리하는 것처럼, 개발자들도 코드를 정리해요.
중요한 건, 기능은 그대로 두고 내부 구조만 깔끔하게 만드는 거예요.
사용자 입장에서는 달라진 게 없지만, 개발자들이 일하기는 훨씬 편해져요.
오래된 라이브러리를 최신 버전으로 업데이트하는 것도 리팩터링이에요!"
```

**키포인트**:
- "방 정리" 비유로 친근함 조성
- 기능 변화 없이 구조 개선이라는 핵심 강조
- 사용자와 개발자 관점 모두 고려

---

### 3️⃣ Refactoring_3.png (필요성 설명)
**화면 구성**: 노란 배경, 생산성과 안정성 구분 설명
**메인 카피**: "Q. 리팩터링 왜 해야 하는 거예요?"

**내레이션 스크립트**:
```
"왜 리팩터링을 해야 할까요?
시간이 지나면서 쌓인 레거시는 개발자들의 생산성과 서비스의 안정성을 떨어뜨려요.

생산성이 떨어지는 예를 들면:
'하나만 바꾸면 다 바뀔 줄 알았는데, 이상하게 일부분만 바뀌네?'

안정성이 떨어지는 예는:
'하나를 고쳤더니 다른 곳에서 버그가 생겼어요!'

이런 상황들이 반복되면 개발이 점점 어려워져요.
그래서 코드의 품질을 다시 끌어올리기 위해 리팩터링이 필요한 거예요."
```

**키포인트**:
- 구체적인 문제 상황 제시
- 생산성과 안정성이라는 두 축으로 정리
- 개발 현장에서 실제로 겪는 어려움 공감

---

### 4️⃣ Refactoring_4.png (시기에 대한 현실적 조언)
**화면 구성**: 노란 배경, Q&A 형식
**메인 카피**: "Q. 신규 기능 개발할 때마다 조금씩 개선하면 안되는 건가요?"

**내레이션 스크립트**:
```
"좋은 질문이에요! 네, 맞습니다!
보통 개발자들은 코드를 수정할 때마다 아쉬운 부분들을 조금씩 같이 고쳐요.
이렇게 일상적으로 하는 소규모 개선은 정말 좋은 습관이에요.

하지만 가끔은 대규모 정리가 필요할 때가 있어요.
수정해야 할 범위가 너무 크면 시간도 오래 걸리고, 
리팩터링 후에는 별도로 테스트도 해야 하거든요.
이런 경우에는 따로 일정을 잡아서 집중적으로 작업합니다."
```

**키포인트**:
- 일상적 개선과 대규모 리팩터링 구분
- 현실적인 시간과 리소스 고려
- 테스트의 중요성 언급

---

### 5️⃣ Refactoring_5.png (위험성에 대한 솔직한 설명)
**화면 구성**: 회색 배경, 고양이와 개발자의 대화
**대화 내용**:
- "아 잠, 대규모 공사이다보니 리팩터링 후 처음보는 버그들이 발견될 수 있어요."
- 고양이: "버그가 수정되는게 아니라 더 생길 수도 있다고요?"

**내레이션 스크립트**:
```
"리팩터링의 위험성도 솔직히 말씀드릴게요.
대규모 리팩터링을 하다 보면 예상치 못한 버그가 생길 수 있어요.
'어? 이건 원래 이렇게 동작하는 거였나?' 하면서 새로운 문제를 발견하기도 하죠.

그래서 리팩터링 후에는 반드시 QA를 진행해야 해요.
여유가 있는 팀이라면 미리 테스트 코드를 작성해두면, 
리팩터링 전후의 동작이 똑같은지 확인할 수 있어서 훨씬 안전합니다."
```

**키포인트**:
- 리팩터링의 위험성을 숨기지 않고 솔직히 공유
- QA의 필수성 강조
- 테스트 코드의 가치와 현실적 제약 인정

---

### 6️⃣ Refactoring_6.png (시기 결정 가이드라인)
**화면 구성**: 노란 배경, 체크리스트 형식
**메인 카피**: "Q. 리팩터링은 언제 해야하고 얼마나 걸리나요?"

**내레이션 스크립트**:
```
"언제 리팩터링을 해야 할까요? 이런 신호들을 주의 깊게 봐주세요:

첫 번째, 버그가 많거나 개발 속도가 느려진 것 같다면
두 번째, 최근에 신규 기능을 급하게 출시했다면

이럴 때가 리팩터링이 필요한 시점일지도 몰라요!

기간은 프로젝트마다 달라요.
작은 프로젝트는 한 주에서 몇 주 정도,
대규모 프로젝트는 분기 단위로 진행되기도 해요.
정확한 예측은 어렵지만, 이 정도 규모로 생각해보시면 됩니다."
```

**키포인트**:
- 구체적이고 실용적인 판단 기준 제시
- 프로젝트 규모별 현실적인 기간 제시
- 정확한 예측의 어려움도 인정

---

### 7️⃣ Refactoring_7.png (팀 문화와 협업)
**화면 구성**: 노란 배경, 고양이와 개발자 대화
**질문**: "개발자분들이 리팩터링 하는 동안 다른 업무를 먼저 진행하고 있으면 되는건가요?"

**내레이션 스크립트**:
```
"좋은 질문이에요! 네, 맞습니다.
리팩터링 기간 동안에는 다른 개발 작업이 중단돼요.
그동안 디자이너나 PM은 다음 스프린트 준비를 하시면 됩니다.
OKR 계획을 세우거나, 유저 리서치를 진행하거나, 
각자의 업무를 하고 계시면 돼요.

리팩터링은 개발팀만의 일처럼 보이지만, 
사실은 팀 전체를 위한 투자예요.
나중에 모든 팀원이 더 빠르고 안정적으로 일할 수 있게 되거든요."
```

**키포인트**:
- 다른 직군의 역할과 협업 방식 설명
- 리팩터링이 전체 팀을 위한 투자임을 강조
- 각자의 업무에 집중할 수 있다는 안내

---

### 8️⃣ Refactoring_8.png (격려와 실천 독려)
**화면 구성**: 회색 배경, 한입 캐릭터의 격려 메시지
**메인 카피**: "우리가 방을 몰아서 치우듯 코드도 정리하는 시간이 필요해요"

**내레이션 스크립트**:
```
"우리가 주기적으로 방을 정리하듯, 코드도 정리하는 시간이 필요해요.
당장 눈에 보이는 성과는 없어 보일 수 있지만,
장기적으로는 팀 전체를 위해 꼭 필요한 일이에요.

만약 동료가 명확한 동기와 목표를 가지고 
리팩터링 기간을 제안한다면 적극 지지해주세요!
깨끗한 코드는 결국 더 빠른 개발과 안정적인 서비스로 이어집니다.
팀 전체가 함께 만들어가는 더 나은 업무 환경이에요."
```

**키포인트**:
- 일상적 비유(방 정리)로 친근함 조성
- 장기적 관점의 중요성 강조
- 팀원 간 상호 지지와 이해 독려
- 궁극적인 가치와 목표 제시

## 🎯 이 시리즈의 특별함

### 균형잡힌 관점
- 리팩터링의 필요성과 위험성을 모두 다룸
- 이상론과 현실적 제약 사이의 균형
- 개발자와 비개발자 모두의 관점 고려

### 현실적 접근
- 완벽한 해답보다는 상황별 가이드라인 제시
- 팀 상황에 따른 유연한 접근법 인정
- 솔직한 어려움과 한계 공유

### 팀 문화 지향
- 개인 스킬보다는 팀 전체의 협업 강조
- 상호 이해와 지지의 중요성 역설
- 장기적 관점에서의 팀 성장 추구

### 감정적 배려
- 각 직군의 입장과 어려움 인정
- 판단보다는 이해와 공감 우선
- 격려와 동기부여로 마무리

이 시리즈는 기술적 주제를 다루면서도 팀 문화와 소통에 중점을 둔 hanib_tech의 독특한 접근법이 가장 잘 드러나는 콘텐츠입니다. 개발 지식을 넘어서 조직 문화 개선에까지 기여하는 교육 콘텐츠의 모범 사례라고 할 수 있습니다.
